# 2.2 대입문 — `a = 1`, `a = b`

## 개요

**대입문**은 `이름 = 표현식` 형태다. 오른쪽 **표현식**을 먼저 계산하고, 그 결과를 왼쪽 **이름**에 붙인다. `a = 1`, `a = b`, `a = a + 1`처럼 이름에 값을 붙이는 동작의 의미를 익히는 것이 이 단원의 목표다.

---

## 핵심 개념

- **형식**: `이름 = 표현식`. 등호 왼쪽에는 (지금 단계에서는) 하나의 이름만 둔다.
- **실행 순서**: 오른쪽 표현식을 먼저 계산한 뒤, 그 결과를 왼쪽 이름에 붙인다.
- **대입은 문(statement)**: `a = 1`은 값으로 평가되는 표현식이 아니라, 실행만 하는 문이다. 따라서 `if a = 1:`처럼 조건 자리에 쓸 수 없다.
- **연쇄 대입**: `a = b = 0`처럼 같은 값을 여러 이름에 한 번에 붙일 수 있다.

---

## 코드 예제

```python
# 오른쪽을 계산한 뒤 왼쪽 이름에 붙인다.
a = 1
b = 2
a = a + 1   # 오른쪽: a의 값 1 + 1 → 2. 그 다음 a에 2를 붙인다.
print(a)    # 2
```

```python
# a = b: b에 붙어 있는 값을 a라는 이름에도 붙인다.
x = 10
y = x
print(y)   # 10
```

```python
# 연쇄 대입: 같은 값을 여러 이름에 붙인다.
a = b = 0
print(a, b)   # 0 0
```

---

## 주의사항 (Pitfalls)

- **등호의 의미**: `=`는 "같다"가 아니라 **대입**이다. 같음 비교는 `==`를 쓴다(Ch4 등에서 다룸).
- **조건문에 대입 사용 불가**: `if x = 1:`은 문법 오류. 대입은 문이므로 조건 자리에는 넣을 수 없다.
- **좌변은 이름**: 지금 단계에서는 `a`, `b`처럼 단일 식별자만 왼쪽에 둔다. `a + 1 = 2`처럼 식을 왼쪽에 두면 안 된다.

---

## Deep Insight

대입은 "이름에 값을 붙이는" 동작이다. `a = a + 1`은 "a가 가리키는 값에 1을 더한 결과를 다시 a라는 이름에 붙인다"로 이해하면 된다. 오른쪽의 `a`는 대입 전의 값으로 계산되고, 그 다음 왼쪽 이름이 새 값과 연결된다. 대입은 문이기 때문에 다른 언어의 할당 표현식(`:=` 등)과 구분해서 다룰 수 있다.

---

## 요약

- 대입문: `이름 = 표현식`. 오른쪽을 계산한 결과를 왼쪽 이름에 붙인다.
- `a = 1`, `a = b`, `a = a + 1`의 의미를 구분할 수 있어야 한다.
- 대입은 문이므로 조건 등 표현식 자리에 쓸 수 없다. 연쇄 대입 `a = b = 0`은 가능하다.
