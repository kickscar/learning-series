# 4.1 조건문

## 개요

**분기**는 조건에 따라 실행할 코드를 나누는 것이다. `if`·`elif`·`else`로 여러 갈래를 만들고, **삼항 연산자** `a if c else b`로 짧은 분기를 한 줄에 쓸 수 있다. `and`·`or`는 **short-circuit**으로 앞만 보고 결과가 정해지면 뒤를 평가하지 않으며, 그 성질을 이용해 `x = y or default` 같은 기본값 패턴도 쓴다(단, Falsy인 유효한 값이 있으면 주의). 읽기 좋은 분기 작성과, `or`/`and`로 기본값을 주는 패턴(그 한계 포함)을 익히는 것이 이 단원의 목표다.

---

## 핵심 개념

### if / elif / else

`if 조건:` 다음에 들여쓴 블록은 **조건이 참일 때만** 실행된다. `else:`는 앞의 `if`(및 연결된 `elif`)가 모두 거짓일 때 실행된다. `elif 조건:`은 "그렇지 않고, 이 조건이 참이면"을 의미하며, 여러 개를 이어서 다중 분기를 만든다. 조건은 Truthy/Falsy로 평가되므로, `if x:`는 `if bool(x):`와 같다. `elif`와 `else`는 선택 사항이며, `if`만 단독으로 쓸 수 있다.

```python
x = 0
if x > 0:
    print("positive")
elif x < 0:
    print("negative")
else:
    print("zero")
```

실행 결과: `zero`

### 삼항 연산자 (조건 표현식)

**조건이 참이면 a, 거짓이면 b**를 값으로 쓰고 싶을 때는 **삼항 연산자** `a if c else b`를 쓴다. 먼저 `c`가 평가되고, 참이면 전체 식의 값이 `a`, 거짓이면 `b`가 된다. `a`와 `b` 중 하나만 평가되므로, "조건에 따라 둘 중 하나를 선택하는 값"을 짧게 쓸 때 유용하다. 문이 아니라 **표현식**이므로 변수 대입이나 함수 인자 안에 넣을 수 있다.

```python
x = 3
label = "odd" if x % 2 else "even"
print(label)
print("ok" if x > 0 else "ng")
```

실행 결과: `odd` / `ok`

### and / or의 short-circuit

`and`는 **앞이 거짓이면 뒤를 평가하지 않고** 앞의 값을 그대로 반환한다. `or`는 **앞이 참이면 뒤를 평가하지 않고** 앞의 값을 그대로 반환한다. 이를 **short-circuit**이라고 한다. 따라서 "앞만 보고 결과가 정해지면 뒤는 실행하지 않는다"는 점을 이용해, **기본값**을 주는 패턴 `x = y or default`를 쓸 수 있다. `y`가 Falsy이면 `default`가 사용된다. 다만 `y`가 **0**이나 **""**처럼 Falsy이지만 "유효한 값"인 경우에도 `default`로 바뀌어 버리므로, "없을 때만 기본값"을 쓰려면 `x = y if y is not None else default`처럼 명시하는 것이 안전할 수 있다.

```python
name = ""
display = name or "anonymous"
print(display)
score = 0
result = score or 100
print(result)
```

실행 결과: `anonymous` / `100` (0이 Falsy라 100이 나옴 — 의도에 따라 주의)

### elif 체인과 읽기 순서

`if` → `elif` → … → `else`는 **위에서부터 순서대로** 조건을 검사한다. 처음으로 참이 되는 블록만 실행되고, 나머지 `elif`·`else`는 건너뛴다. 따라서 "더 구체적인 조건"을 위에, "넓은 조건"이나 `else`를 아래에 두면 읽기와 유지보수가 쉬워진다. `elif`가 많아지면 `match`(3.10+)로 바꾸는 것도 고려할 수 있다.

#### ④ 실습 포인트

- **if/elif/else**: 위에서부터 조건 검사, 처음 참인 블록만 실행.
- **삼항**: `a if c else b` — c가 참이면 a, 거짓이면 b. 표현식이라 대입·인자에 사용 가능.
- **short-circuit**: `and`는 앞이 거짓이면 뒤 미평가, `or`는 앞이 참이면 뒤 미평가.
- **`x = y or default`**: y가 Falsy면 default. **0·""**이 유효한 값이면 의도와 다를 수 있음.

## 주의사항 (Pitfalls)

- **`=`와 `==`**: 조건에 "같은지"를 넣을 때는 `==`를 쓴다. `if x = 1:`은 문법 오류다.
- **기본값과 0/빈 문자열**: `name = input() or "anonymous"`는 빈 입력에만 기본값을 주지만, `count = count or 0`은 count가 이미 0일 때도 0이므로 문제없다. 반면 "점수가 0일 수 있는데 기본 100을 주고 싶다"면 `score or 100`은 0을 100으로 바꿔 버리므로 부적절하다.
- **삼항의 중첩**: `a if c1 else b if c2 else d`처럼 쓰면 동작은 하지만 가독성이 떨어지므로, 복잡하면 일반 `if`/`elif`로 풀어 쓴다.

## Deep Insight

분기는 "한 시점에 한 갈래만 실행"한다는 점이 중요하다. `elif` 체인은 위에서부터 첫 번째로 참인 곳만 실행하고 끝난다. 삼항은 "값을 조건에 따라 고르는 것"이라, 문이 아닌 식으로 쓸 수 있어 함수 인자나 리스트 내부 등에 그대로 넣을 수 있다. `or`의 short-circuit은 "첫 번째 Truthy 또는 마지막 값"을 반환하므로, 기본값 패턴이 자연스럽게 나오지만, Falsy인 유효값이 있으면 의도와 달라질 수 있어 도메인에 맞게 선택해야 한다.

## 요약

- **if/elif/else**: 위에서부터 조건 검사, 처음 참인 블록만 실행. `else`는 선택.
- **삼항**: `a if c else b` — c 참이면 a, 거짓이면 b. 표현식.
- **short-circuit**: `and`/`or`는 결과가 정해지면 뒤를 평가하지 않음. `x = y or default`는 Falsy일 때 기본값이지만, 0·""이 유효하면 주의.
