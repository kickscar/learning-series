# 3.3 mapping: dict

## 개요

**dict(딕셔너리)**는 **키-값 쌍**으로 이루어진 매핑 타입이다. 키로 값을 조회(`d[k]`), 키가 없을 때 기본값으로 조회(`d.get(k, default)`), 항목 추가·수정(`d[k] = v`)을 할 수 있다. 키는 **hashable**이어야 한다. dict 생성·조회·수정과, 3.7+ 삽입 순서 보장, 3.9+ `dict | other`·`|=` 맛보기를 익히는 것이 이 단원의 목표다.

---

## 핵심 개념

### 생성과 리터럴

dict는 **키-값 쌍**의 집합이다. 리터럴은 `{"a": 1, "b": 2}`처럼 중괄호 안에 `키: 값`을 쉼표로 나열한다. `dict(a=1, b=2)`처럼 키워드 인자로도 만들 수 있는데, 이때 키는 식별자 규칙을 만족하는 문자열만 쓸 수 있다. 빈 dict는 `{}` 또는 `dict()`로 만든다. `{}`는 set이 아니라 dict이므로, 빈 set이 필요할 때는 `set()`을 써야 한다.

```python
d = {"name": "Kim", "age": 20}
print(d["name"])        # Kim
print(d.get("age"))     # 20
print(d.get("x", 0))    # 0 (키 없으면 기본값)
d["age"] = 21           # 수정
d["city"] = "Seoul"     # 추가
```

### 조회: d[k]와 d.get(k, default)

키로 값을 가져올 때 `d[k]`를 쓴다. 키 k가 dict에 **없으면** `KeyError`가 발생한다. 키가 있을지 없을지 모를 때는 `d.get(k)`를 쓰면, 없을 때 `None`을 반환한다. 기본값을 지정하려면 `d.get(k, default)`처럼 두 번째 인자를 주면, 키가 없을 때 그 default가 반환된다. 따라서 "키가 없으면 0으로 간주"하는 식의 로직을 한 줄로 쓸 수 있다.

### 추가와 수정

`d[k] = v`로 키 k에 값 v를 넣는다. k가 이미 있으면 그 키에 대응된 값만 v로 바뀌고, 없으면 새 항목이 추가된다. 같은 키는 한 번만 존재하므로, 같은 키에 계속 대입하면 "수정"이 되고 항목 개수는 늘지 않는다. 삭제는 `del d[k]` 또는 `d.pop(k)` 등으로 한다.

### 삽입 순서 (3.7+)

Python 3.7부터 dict는 **삽입한 순서**를 유지한다. 즉, 항목을 넣은 순서대로 키·값이 순회되고, `list(d.keys())`, `list(d.values())`도 그 순서를 따른다. 예전에는 순서가 보장되지 않았지만, 지금은 JSON처럼 "필드 순서가 의미 있는" 데이터를 dict로 다룰 때 예측 가능하게 동작한다.

### 키의 조건: hashable

dict의 키는 **hashable**이어야 한다. 즉, 수명 동안 값이 바뀌지 않고, 해시값을 계산할 수 있는 타입이어야 한다. int, float, str, tuple, frozenset 등 **불변** 타입은 키로 쓸 수 있다. list, set, dict처럼 **가변** 타입은 키로 쓸 수 없으며, 사용하면 `TypeError`가 난다. "키로 쓸 수 있는가?"를 판단할 때는 "이 타입이 불변인가?"를 먼저 생각하면 된다.

### 3.9+ 합치기와 comprehension

두 dict를 합쳐 새 dict를 만들 때 `d1 | d2`를 쓸 수 있다. 같은 키가 있으면 오른쪽(d2) 값이 남는다. `d |= other`는 d를 그 자리에서 갱신하는데, other의 키-값 쌍이 d에 추가되거나 덮어씌워진다. dict comprehension은 `{k: v for ...}` 형태로, 반복문으로 키-값 쌍을 만들어 dict를 한 번에 구성할 때 쓴다.

```python
a, b = {"a": 1, "b": 2}, {"b": 3, "c": 4}
print(a | b)   # {'a': 1, 'b': 3, 'c': 4}
squares = {x: x ** 2 for x in range(1, 4)}
print(squares)   # {1: 1, 2: 4, 3: 9}
```

이 실습에서는 **dict 조회·추가·수정과 합치기**의 개념을 복습한다.

#### ① `[]`로 조회

`script01.py`


실행 결과: `1`

`script02.py`

```python
d = {"a": 1, "b": 2}

```

실행 결과: `1` / `None` / `0`

#### ③ 추가·수정·삽입 순서

`script03.py`

```python
d = {}
d["one"], d["two"], d["three"] = 1, 2, 3
d["two"] = 20
print(list(d.keys()))
del d["one"]
print(d)
```

실행 결과: `['one', 'two', 'three']` / `{'two': 20, 'three': 3}`

#### ④ 합치기 `|`·comprehension

`script04.py`

```python
a = {"x": 1, "y": 2}
b = {"y": 10, "z": 3}
print(a | b)
squares = {i: i ** 2 for i in range(4)}
print(squares)
```

실행 결과: `{'x': 1, 'y': 10, 'z': 3}` / `{0: 0, 1: 1, 2: 4, 3: 9}`

#### ⑤ 실습 포인트

- `d[k]` → 키가 없으면 **KeyError**.
- `d.get(k)` → 없으면 `None`.
- `d.get(k, default)` → 없으면 default 반환.
- `d[k] = v`로 추가·수정. 3.7+ 삽입 순서 유지.
- 3.9+ `d1 | d2`로 합치기. dict comprehension `{k: v for ...}`.

## 주의사항 (Pitfalls)

- **키가 없을 때**: `d[k]`는 키가 없으면 `KeyError`. 키 존재 여부를 모를 때는 `d.get(k, default)` 사용.
- **키는 hashable**: list, set, dict처럼 **가변** 타입은 키로 쓸 수 없다. tuple, str, int 등 불변 타입은 키로 사용 가능.
- **`{}`는 빈 dict**: 빈 set이 아니다. 빈 set은 `set()`으로만 만든다.
- **`in`**: `k in d`는 "키 k가 d에 있는가"를 검사한다. 값이 아니라 키 기준이다.

---

## Deep Insight

dict는 해시 테이블 기반이라 키로 값을 찾는 것이 평균 O(1)에 가깝다. "키 → 값"이라는 한 방향 매핑이므로, 값으로 키를 찾으려면 순회하거나 역매핑을 따로 둬야 한다. 3.7+ 삽입 순서 보장 덕분에 JSON처럼 "필드 순서가 의미 있는" 데이터를 dict로 다룰 때 예측 가능한 순서로 순회·직렬화할 수 있다. `dict | other`는 두 매핑을 합칠 때 새 dict를 만들고, `|=`는 왼쪽 dict를 그 자리에서 갱신한다.

---

## 요약

- **dict**: 키-값 쌍. `{}`, `dict()`. `d[k]`, `d.get(k, default)`, `d[k] = v`. 키는 hashable이어야 한다.
- **조회**: 키 없음 시 `d[k]` → `KeyError`, `d.get(k, default)` → default 반환.
- **3.7+** 삽입 순서 유지. **3.9+** `d1 | d2`, `d |= other`. dict comprehension `{k: v for ...}`.
