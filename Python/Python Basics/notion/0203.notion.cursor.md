# 2.3 이름 사용 — 읽기와 쓰기

## 개요

이름을 **표현식**에서 쓰면, 그 이름에 붙어 있는 **값**이 사용된다. 같은 이름에 다시 **대입**하면, 그 이름은 새 값과 연결된다. 이름을 **읽는 것**과 **쓰는 것(대입)**을 구분하는 것이 이 단원의 목표다.

---

## 핵심 개념

- **읽기**: 표현식 안에서 이름을 쓰면, 그 이름에 대응된 값이 계산에 사용된다. 예: `print(a)`, `a + 1`, `len(s)`에서 `a`, `s`.
- **쓰기**: `이름 = 표현식` 대입문으로 그 이름에 새 값을 붙인다. 이후 그 이름을 읽으면 새 값이 나온다.
- **대입되지 않은 이름**: 아직 한 번도 대입되지 않은 이름을 읽으면 `NameError`가 난다.

---

## 코드 예제

```python
# 읽기: 이름에 붙어 있는 값이 사용된다.
a = 10
print(a)      # 10 — a를 "읽음"
b = a + 1     # 오른쪽에서 a를 읽어 11이 되고, b에 11이 붙음
print(b)      # 11
```

```python
# 쓰기: 같은 이름에 다시 대입하면 새 값으로 바뀐다.
x = 1
print(x)   # 1
x = 2      # x에 새 값을 붙임 (쓰기)
print(x)   # 2
```

```python
# 대입 없이 읽으면 NameError
# print(never_defined)   # NameError: name 'never_defined' is not defined
```

---

## 주의사항 (Pitfalls)

- **읽기 vs 쓰기**: `print(a)`는 a를 **읽는** 것. `a = 2`는 a를 **쓰는** 것(대입). 같은 이름이라도 문맥에 따라 읽기/쓰기가 다르다.
- **정의 전 사용**: 반복문·조건문 안에서 이름을 쓸 때, 그 이름이 그 전에 대입으로 정의되었는지 확인한다. 정의되지 않은 이름을 읽으면 `NameError`다.

---

## Deep Insight

한 이름은 한 시점에 하나의 값과만 연결된다. 같은 이름에 대입을 하면 "연결이 바뀌는" 것이지, 이전 값이 수정되는 것이 아니다. 따라서 표현식에서 이름을 읽을 때는 **그 시점에** 그 이름에 붙어 있는 값이 사용된다. 이 차이는 나중에 제어문·함수에서 이름이 어디서 정의되고 어디서 보이는지(스코프)를 이해하는 기초가 된다.

---

## 요약

- 이름을 **읽기**: 표현식에서 이름을 쓰면 그 이름에 붙은 값이 사용된다.
- 이름을 **쓰기**: 대입문으로 그 이름에 새 값을 붙인다. 같은 이름에 다시 대입하면 새 값으로 바뀐다.
- 대입되지 않은 이름을 읽으면 `NameError`가 난다.
