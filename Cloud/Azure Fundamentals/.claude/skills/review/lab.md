# Skill: review/lab

섹션 실습 draft 검토 체크리스트.

---

## 구조 체크

- [ ] `# [실습] lab{no}: {제목}` H1으로 실습 파트가 시작하는가 (이론/실습 경계 구분)
- [ ] `### 실습 목표`가 있는가
- [ ] `# 1. 전체 아키텍처`가 있는가
- [ ] `# 2. 사전 준비`가 있는가
- [ ] 이후 단계가 작업 내용에 맞게 명명된 H1으로 구성되는가
- [ ] `# {n}. 실행 및 결과 검증`이 있는가
- [ ] `# {n+1}. 자원 정리`가 있는가

---

## 규칙 체크

- [ ] ⚠️ 비용 주의 섹션이 없는가
- [ ] 헤딩에 볼드(`**`) 없는가
- [ ] 콘솔 경로 형식이 맞는가 (`CLAUDE.md > 플랫폼 상수`의 콘솔 경로 형식 기준)
- [ ] 설정값이 **`볼드+백틱`** 형식인가
- [ ] `[콘솔화면: ...]` 플레이스홀더가 모든 스텝에 있는가
- [ ] 전체 아키텍처 다이어그램에 해설이 있는가

---

## 내용 체크

- [ ] 실습 목표가 3개 이상인가
- [ ] 각 작업 단계 이름이 실습 내용을 반영하는가
- [ ] 자원 정리에 삭제할 리소스 목록이 구체적으로 명시되어 있는가

---

## 보고 형식

```
## 검토 결과: {ch_no}.{sec_no} 실습 파트 (lab{no})

OK:
- {항목}

수정 필요:
- {항목}: {이유}

기술 내용 확인 필요 (선택):
- {항목}
```
