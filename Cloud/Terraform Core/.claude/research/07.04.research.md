## 상세 조사

### 1. 변수 우선순위 — 공식 문서 기준 (높은 쪽 → 낮은 쪽)

```text
1. -var / -var-file (명령줄 나열 순서대로, 마지막 값 적용)
2. *.auto.tfvars / *.auto.tfvars.json (lexical 순서)
3. terraform.tfvars.json
4. terraform.tfvars
5. TF_VAR_<name> 환경 변수
6. variable 블록의 default 값
```

**02.04 draft와의 차이**: 02.04에서 `terraform.tfvars`와 `terraform.tfvars.json`을 하나로 묶어 "terraform.tfvars 파일"로 표기했다. 05.01에서는 `.json` 변형까지 정확히 분리 표기해야 한다.

**핵심**: `terraform.tfvars.json`이 `terraform.tfvars`보다 우선한다. 양쪽 모두 존재하면 **모두 로드**되며 충돌 변수는 `.json` 승.

### 2. `-var`과 `-var-file` 혼용

공식 문서: `-var and -var-file options on the command line in the order provided`

- 자유롭게 혼용 가능
- 왼쪽에서 오른쪽으로 순차 처리
- 같은 변수를 여러 번 지정하면 **마지막 값** 적용

```bash
# -var-file 먼저, -var 나중 → -var 승
$ terraform apply -var-file="base.tfvars" -var="instance_type=t3.large"

# -var 먼저, -var-file 나중 → -var-file 승
$ terraform apply -var="instance_type=t3.large" -var-file="override.tfvars"
```

이 동작은 02.04 draft의 "동일 우선순위, 나열 순서대로" 기술과 일치한다.

### 3. `*.auto.tfvars` 로드 규칙

- `*.auto.tfvars`와 `*.auto.tfvars.json` **모두** 자동 로드
- **lexical order** (사전순) — `a.auto.tfvars` → `b.auto.tfvars` → `z.auto.tfvars`
- `.auto.tfvars`와 `.auto.tfvars.json`이 혼재하면 함께 lexical 정렬되어 로드
- 뒤에 로드된 파일이 앞 파일의 같은 변수를 덮어씀

### 4. sensitive 변수 동작

**마스킹 범위:**

| 단계 | 동작 |
|------|------|
| `terraform plan` | `(sensitive value)` 표시 |
| `terraform apply` | `(sensitive value)` 표시 |
| `terraform output` | `<sensitive>` 표시 |
| `terraform output -json` | **평문 노출** (주의) |
| `terraform output -raw` | **평문 노출** (주의) |
| State 파일 | **평문 저장** |

**output의 sensitive**: output 블록에도 `sensitive = true` 설정 가능. `terraform output` 실행 시 `<sensitive>`로 표시되지만 `-json`이나 `-raw` 플래그 사용 시 평문 노출.

### 5. ephemeral 변수 (TF 1.10+)

- **도입 버전**: Terraform 1.10.0 (2024-11-27)
- **목적**: plan 파일과 state 파일에 값을 저장하지 않음
- **sensitive와 차이**: sensitive는 CLI 출력만 마스킹 (state에 평문 저장). ephemeral은 state/plan에 아예 저장 안 함
- **제한사항**: ephemeral 변수는 특정 context에서만 사용 가능 (일반 resource 인수에 직접 사용 불가)
- **관련 기능**: ephemeral resource (매 phase마다 재평가, state 미저장), `ephemeralasnull` 함수

**05.01 draft 작성 시 권장**: ephemeral은 sensitive와 대비하여 간단히 언급하되, 본격적 설명은 하지 않는다. "sensitive는 CLI 출력만 마스킹하고 state에는 평문 저장된다. TF 1.10부터 ephemeral 옵션이 추가되어 state/plan에 아예 저장하지 않을 수 있다" 수준.

### 6. `.tfvars` 파일 형식

```hcl
# terraform.tfvars (HCL 형식)
instance_type = "t3.micro"
allowed_cidrs = ["10.0.0.0/8"]
```

```json
// terraform.tfvars.json (JSON 형식)
{
  "instance_type": "t3.micro",
  "allowed_cidrs": ["10.0.0.0/8"]
}
```

`-var-file`로 지정하는 파일도 확장자가 `.json`이면 JSON으로 자동 파싱.

### 7. TF 1.14.x 변수 관련 변경사항

1.14.0~1.14.8 changelog 확인 결과, 변수 처리·우선순위·sensitive·tfvars 관련 변경사항 **없음**. 유일하게 관련된 항목:
- `terraform import` 시 workspace 변수 상속 개선 (variable set에서 상속된 변수 포함) — 05.01 범위 밖

## 주의사항

- **terraform.tfvars vs terraform.tfvars.json 공존**: 실무에서는 둘 중 하나만 사용하는 것이 관례. 05.01에서는 ".json 변형도 지원된다" 수준 언급 + 양쪽 공존 시 우선순위 표에 명시
- **sensitive state 저장**: 반드시 명시해야 할 보안 주의사항. "sensitive = true는 CLI 마스킹일 뿐, state에는 평문" 강조
- **`terraform output -json` 평문 노출**: lab02에서 sensitive output 확인 시 `-json` 플래그로 실제 값 확인하는 실습 가능
- **02.04 draft와 일관성**: 02.04에서 "-var과 -var-file은 동일 우선순위, 나열 순서대로"라고 기술. 05.01에서 이를 심화 설명하되 모순 없도록 유지
- **auto.tfvars lexical order**: `00-base.auto.tfvars` < `99-override.auto.tfvars` — 번호 접두사 패턴으로 의도적 순서 제어 가능. lab에서 시연 가능
