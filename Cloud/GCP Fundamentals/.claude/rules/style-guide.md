# Rule: style-guide

문서 스타일 공통 규칙. 모든 write skill에 적용.

---

## 헤딩 규칙

### 이론/개념 섹션

| 레벨 | 용도 | 형식 |
|------|------|------|
| H1 | 개념/토픽명 | `# {개념명}` — 번호 없음 |
| H2 | 하위 주제 | `## 1. {주제명}` — 번호 있음 |
| H3 | 세부 항목 | `### ① {항목}` — ①②③ 패턴 |

### 실습 섹션

| 레벨 | 용도 | 형식 |
|------|------|------|
| H1 | 작업 단계 | `# 1. {단계명}` — 번호 있음 |
| H2 | 콘솔 스텝 / 리소스 단위 | `## 1. {콘솔} > {서비스} > **{최종 화면}**` — 콘솔 이름은 `CLAUDE.md > 플랫폼 상수` 참조 |
| H3 | 세부 항목 | `### ① {항목}` 또는 `### 1. {항목}` |

- **헤딩에 볼드(`**`) 처리 안 함**
- H1 `# 전체 아키텍처`, `# 사전 준비`, `# 핵심 정리`, `# 참고 자료` 고정

---

## 문서 시작 규칙

- **챕터명/섹션명 헤딩 없이 시작** — Notion 타이틀이 대신
- 이론 섹션: 첫 H1 전에 개요 단락 1~3줄 (앞 섹션 환기 + 이번 섹션 예고)
- 실습 섹션: 설명 텍스트 1~2줄 + `### 실습 목표`

---

## 이미지 플레이스홀더

> 콘솔 이름은 `CLAUDE.md > 플랫폼 상수` 참조.

```
[콘솔화면: {콘솔} > {서비스} > {화면} > {핵심 포인트 설명}]
[이미지: {개념/구조 설명} — {독자가 봐야 할 포인트}]
```

- 콘솔 캡처가 필요한 모든 지점에 명시
- 플레이스홀더 뒤에 해설 1~2줄 추가 (선택)

---

## 설정값 / 확인값 표기

- 설정값: **`{값}`** (볼드 + 백틱)
- 리소스명: `{리소스명}` (백틱만)
- 필드명: 그대로 (볼드 없음)

예:
```
- Name: **`vm-lab05`**
- Region: **`asia-northeast3`**
```

---

## 콘솔 경로 표기

> 콘솔 경로 형식은 `CLAUDE.md > 플랫폼 상수` 참조.

```
{콘솔} > {서비스} > {하위 메뉴} > **{최종 화면}**
```

- `>` 구분자 사용
- 최종 화면(작업 대상)만 **볼드**
- 한글 메뉴명은 영문 병기 없이 그대로 사용 가능

## 리소스명 규칙

`{type}-{context}[-{suffix}]`

| 예시 | 설명 |
|------|------|
| `vm-lab05` | 섹션 lab 전용 VM |
| `vm-gallery` | Gallery VM |
| `vpc-gallery` | Gallery VPC |
| `subnet-pub-gallery` | Gallery Public Subnet |
| `subnet-pri-gallery` | Gallery Private Subnet |
