# 1. DB Subnet Group

## 1. RDS는 Subnet Group으로 배치 범위를 정한다

RDS는 "어떤 Subnet들에 DB를 둘 수 있는가"를 DB Subnet Group으로 지정한다. DB Subnet Group은 여러 Subnet의 묶음이며, 최소 2개 AZ에 걸친 Subnet을 요구한다.

### ① Private Subnet으로 구성한다

RDS는 인터넷에 직접 노출하지 않는다. 운영형 패턴은 다음이다.

- RDS: Private Subnet
- 애플리케이션(EC2/ECS): Private Subnet
- 외부 접근: ALB 같은 Public 엔드포인트를 통해서만

[이미지: 네트워크 구조 - ALB(Public) -> App(Private) -> RDS(Private) - 직접 DB 접근 없음]

이 구조는 "DB는 네트워크 경계로 보호한다"는 전제를 고정한다. DB는 Public IPv4를 갖지 않고, SG로 접근 대상을 제한한다.

---

# 2. DB Security Group 패턴

## 1. DB SG는 App SG만 허용한다

DB는 다음과 같이 설계한다.

- DB SG inbound: 3306 from App SG
- DB SG inbound: 3306 from 0.0.0.0/0 같은 설정은 금지에 가깝다

[이미지: AWS Console - EC2 - Security Groups - DB SG Inbound rules - Source를 App SG로 제한]

이 패턴은 "누구나 DB에 접근"이 아니라 "애플리케이션 계층만 DB에 접근"을 강제한다.

---

# 3. DB Instance 생성 포인트(MariaDB)

## 1. 엔진, 클래스, 스토리지, 인증 정보

DB Instance 생성 시 실습에서 중요한 선택지는 다음이다.

- Engine: MariaDB
- Engine version: **11.8.5**(Console에 동일 항목이 없으면 11.8.x 중 가장 가까운 패치)
- Instance class: 최소 사양(db.t3.micro 등, 계정/리전 정책에 따라 다름)
- Storage: 기본 gp3
- Master username/password: 안전하게 관리

[이미지: AWS Console - RDS - Create database 화면 - Engine version(11.8.5 또는 11.8.x)/Instance class/Storage 입력 포인트]

## 2. 네트워크 설정

실습에서 가장 중요한 구간은 네트워크다.

- VPC: Custom VPC
- DB subnet group: Private Subnet 기반
- Public access: No
- VPC security group: DB SG

[이미지: AWS Console - RDS - Create database - Connectivity - VPC/Subnet group/Public access/SG 확인 포인트]

이 설정이 틀리면 DB는 만들어져도 "접근이 안 되는 DB"가 된다. 다음 Section에서 EC2에서 연결할 것이므로, 여기서 Private 통신 경계를 정확히 만든다.

## 3. Parameter Group(개요)

Parameter Group은 엔진 설정(타임존, 문자셋 등)을 조정한다. 이 시리즈에서는 기본 Parameter Group을 사용하고, 존재와 위치만 이해한다.

---

# 핵심 정리

- DB Subnet Group은 RDS 배치 Subnet 범위를 지정하며, 최소 2개 AZ에 걸친 Subnet이 필요하다.
- RDS는 Private Subnet에 배치하고 Public access는 No로 둔다.
- DB SG는 3306을 App SG로부터만 허용하는 패턴이 표준이다.
- RDS 생성에서 가장 중요한 부분은 네트워크 설정(VPC, Subnet group, Public access, SG)과 **엔진 버전(MariaDB 11.8.5 기준)** 선택이다.

# [실습] lab20: RDS DB Instance 생성

Private Subnet 2개 이상으로 DB Subnet Group을 생성하고, MariaDB **11.8.5** 기준 DB Instance를 만든다. Public access를 No로 두고, DB Security Group을 구성해 "애플리케이션 계층에서만 접근 가능한 DB"를 만든다.

---

### 실습 목표

- DB Subnet Group을 생성한다(Private Subnet 2개 이상).
- DB SG를 생성하고 3306을 App SG에서만 허용한다.
- MariaDB DB Instance를 생성한다(Engine version **11.8.5** 기준, Public access No).
- Endpoint/Port를 확인한다.

⚠️ 비용 주의: RDS는 인스턴스 시간당 비용과 스토리지 비용이 발생한다. 학습 종료 시점에 RDS를 반드시 정리한다.

---

# 1. 전체 아키텍처

```mermaid
flowchart LR
  App[App EC2/ECS in Private Subnet] -->|3306| RDS[RDS(MariaDB 11.8.5) in Private Subnet]
  App --> AppSG[App SG]
  RDS --> DbSG[DB SG]
  AppSG -->|allowed| DbSG
```

이 실습은 "DB는 Private Subnet에 두고, SG 참조로 App 계층만 접근"하는 표준 경계를 만든다.

# 2. 사전 준비

- 리전: `ap-northeast-2 (Seoul)`
- Custom VPC 구성 완료
  - Private Subnet 2개 이상
- App SG 준비
  - Gallery 또는 앱 계층에 사용할 SG(예: `fundamentals-sg-app`)

# 3. 리소스 생성 및 설정 (생성 + 연결)

각 단계에서 AWS Console 화면 스냅샷을 반드시 명시한다.

## 1. DB Subnet Group 생성(Private Subnet)

설명: RDS가 배치될 Private Subnet 목록을 지정한다.

[이미지: AWS Console - RDS - Subnet groups - Create DB subnet group 화면 - VPC/Subnet 선택 포인트]

설정 포인트(예시):

- Name: **{db-subnet-group-name}**
- VPC: **{vpc-id}**
- Subnets: **{private-subnet-a-id}**, **{private-subnet-b-id}**

## 2. DB Security Group 생성(3306 from App SG)

설명: DB 접근을 App 계층으로만 제한한다.

[이미지: AWS Console - EC2 - Security Groups - Create security group - DB SG 생성 화면]
[이미지: AWS Console - EC2 - Security Groups - Inbound rules - MySQL/Aurora 3306 Source를 App SG로 지정]

설정 포인트(예시):

- Name: **{db-sg-name}** (예: `fundamentals-sg-db`)
- Inbound
  - MySQL/Aurora(3306) from `**{app-sg-id}**`

## 3. RDS DB Instance 생성(MariaDB)

설명: Private DB를 생성한다.

[이미지: AWS Console - RDS - Create database - Standard create 선택]
[이미지: AWS Console - RDS - Create database - Engine/MariaDB 및 Engine version(11.8.5) 선택]
[이미지: AWS Console - RDS - Create database - Credentials(Username/Password) 입력]
[이미지: AWS Console - RDS - Create database - Connectivity(VPC/Subnet group/Public access/SG) 설정]

설정 포인트(예시):

- Engine: MariaDB
- Engine version: **11.8.5**(또는 Console에서 선택 가능한 11.8.x)
- DB instance identifier: **{db-identifier}** (예: `fundamentals-db`)
- Master username: **{db-admin}** (`lab21`·Gallery에서 동일 계정명으로 참조)
- Master password: **{db-password}**
- VPC: **{vpc-id}**
- DB subnet group: **{db-subnet-group-name}**
- Public access: No
- VPC security group: `**{db-sg-id}**`
- Database port: 3306

⚠️ 주의:

- 생성에는 시간이 걸린다. Status가 Available이 될 때까지 기다린다.
