# Section Draft Template: AWS Fundamentals (이론/개념 구조)

이 템플릿은 AWS Fundamentals 시리즈의 Section draft 구조를 정의한다.

본 시리즈는 AWS Console 기반으로 Cloud Infrastructure를 이해하는 것을 목표로 하며,
개별 서비스 설명과 함께 **리소스 간 연결 구조와 실제 동작 흐름**을 중심으로 작성한다.

---

## 사용 방법 (중요)

- 이 파일은 작성자가 참고하는 **템플릿 설명서**이다.
- 실제 Section draft는 아래 템플릿 예시 중 하나를 선택하되, 문서가 `# 개요` 없이 시작하도록 **첫 번째 `# 1. {주요 개념/토픽 ...}`부터 작성**한다.
- Notion 페이지 타이틀은 `{Section name}`으로 두되, draft 본문은 **섹션명(문서 타이틀)로 시작하지 않는다.** 첫 줄은 **`# 1. {첫 개념/토픽}` H1**로 시작한다. 예: (X) `# 03 EC2 - Virtual Server / 02 Instance 생성과 접속`, (X) `# Instance 생성과 접속` (섹션명 반복), (O) `# 1. Instance 생성이 의미하는 것`
- `## 문서 구조 1`, `## 문서 구조 2`, `## 구조 선택 기준`, `## 다이어그램 규칙` 같은 메타 섹션은 **draft에 그대로 복사하지 않는다.**

## 구조 선택 기준

Section 작성 시 다음 기준으로 구조를 선택한다.

### 구조 1 사용

- 일반 개념 설명 (Cloud 개념, 네트워크 기본 개념 등)
- AWS 서비스가 아닌 범용 개념
- 연결 구조보다 개념 이해가 중요한 경우

예: Cloud Computing 개념, Region / AZ 개념, 기본 Networking 개념

### 구조 2 사용

- AWS 리소스를 다루는 경우
- 인프라 구성 요소 (VPC, Subnet, EC2, S3 등)
- 리소스 간 연결이 중요한 경우

예: VPC, Subnet, EC2, Security Group

---

## 다이어그램 규칙

- 연결 구조 설명 시 mermaid 사용을 우선한다
- 복잡한 구조는 반드시 시각적으로 표현한다
- 다이어그램 퀄리티를 확보하기 어렵다면 무리해서 mermaid로 그리지 않는다. Networking 주제는 이미지 + 상세 설명을 우선할 수 있다.
