# 3.4 set, frozenset

## 개요

**set**과 **frozenset**은 **중복이 없고 순서가 없는** 컬렉션이다. 멤버십 검사 `in`이 평균 O(1)로 빠르다. **set**은 가변(추가·삭제 가능), **frozenset**은 불변이다. set으로 중복 제거·멤버십 검사를 하고, `{1, 2} | {2, 3}` 같은 합집합 연산과 **빈 set**은 `set()`으로 만든다는 점을 익히는 것이 이 단원의 목표다.

---

## 핵심 개념

### set (집합)

set은 **중복이 없고 순서가 없는** 컬렉션이다. 같은 값을 여러 번 넣어도 하나만 남으므로, 리스트에서 중복을 제거할 때 `list(set(lst))`처럼 자주 쓴다. 리터럴은 `{1, 2, 3}`처럼 중괄호에 값을 쉼표로 나열한다. 가변이라 `add(x)`로 요소를 추가하고, `remove(x)`로 제거한다(없으면 `KeyError`). `discard(x)`는 x가 없어도 에러 없이 무시한다. set에 넣을 수 있는 요소는 **hashable**이어야 하므로, list나 dict는 set의 요소가 될 수 없고, tuple·str·int 등은 가능하다. 순서가 없기 때문에 `s[0]` 같은 인덱싱은 지원하지 않는다.

```python
s = {3, 1, 2, 1, 2}
print(s)          # 중복 제거, 순서 미보장
print(2 in s)     # True (멤버십 O(1))
s.add(4)
s.discard(1)
print(s)          # {2, 3, 4} 등
```

### frozenset (불변 집합)

frozenset은 **불변** set이다. `frozenset([1, 2, 3])`처럼 이터러블을 넘겨 만들어 쓴다. 한 번 만들면 추가·삭제가 불가능하다. 대신 dict의 **키**나 다른 set의 **요소**로 쓸 수 있다. 가변인 set은 키나 다른 set의 요소가 될 수 없지만, frozenset은 가능하다. set과 동일하게 `|`, `&`, `-`, `^` 등 집합 연산을 지원한다.

### 순서 없음과 멤버십 in

set·frozenset은 내부 순서를 보장하지 않는다. 출력하거나 순회할 때 순서에 의존하면 안 되고, 순서가 필요하면 `sorted(s)`로 리스트를 만들어 쓴다. 대신 "이 값이 있는가?"를 나타내는 **멤버십** 연산 `x in s`가 평균 **O(1)** 로 동작한다. list에서 `in`을 쓰면 앞에서부터 하나씩 찾으므로 O(n)인데, set은 해시 테이블로 구현되어 있어 "있는지" 검사가 매우 빠르다. 따라서 값이 많고 "포함 여부"를 자주 검사할 때는 list를 set으로 바꿔 두고 `in`을 쓰는 것이 효율적이다.

### 빈 set과 단일 요소 set

**빈 set**은 `{}`로 만들 수 없다. `{}`는 빈 **dict**이다. 빈 set이 필요하면 반드시 `set()`으로 만든다. 요소가 하나인 set은 `{1}`처럼 쓸 수 있다. `{-1}`은 "음수 리터럴 -1 하나를 가진 set"으로 해석된다. 튜플에서 `(1,)`처럼 단일 요소일 때 쉼표를 넣는 것과 비슷하게, `{1,}`처럼 쓰면 "한 개 요소 set"임이 더 분명해진다.

```python
empty = set()     # {}는 dict
a, b = {1, 2, 3}, {2, 3, 4}
print(a | b, a & b, a - b)   # 합집합, 교집합, 차집합
fs = frozenset([1, 2, 3])
d = {fs: "ok"}    # frozenset은 키 가능
```

## 실습 예제

### [실습] lab34

이 실습에서는 **set·frozenset과 집합 연산·멤버십**의 개념을 복습한다.

#### ① set·중복 제거·빈 set

`script01.py`

```python
lst = [1, 2, 2, 3, 3, 3, 4]
s = set(lst)
print(s)
print(type(set()), type({}))
```

실행 결과: `{1, 2, 3, 4}` (순서 미보장) / `<class 'set'> <class 'dict'>`

#### ② add·discard·remove

`script02.py`

```python
s = {1, 2, 3}
s.add(4)
s.discard(2)
print(s)
# s.remove(99)   # KeyError
```

실행 결과: `{1, 3, 4}`

#### ③ 집합 연산 | & - ^

`script03.py`

```python
a, b = {1, 2, 3}, {2, 3, 4}
print(a | b, a & b, a - b, a ^ b)
```

실행 결과: `{1, 2, 3, 4}` `{2, 3}` `{1}` `{1, 4}`

#### ④ frozenset·dict 키

`script04.py`

```python
fs = frozenset([1, 2, 3])
d = {fs: "ok"}
print(d[frozenset([1, 2, 3])])
# d[{1, 2}] = "x"   # TypeError: unhashable type: 'set'
```

실행 결과: `ok`

#### ⑤ 실습 포인트

- **빈 set**: `set()`만 가능. `{}`는 빈 dict.
- **중복 제거**: `set(이터러블)` → 순서 없음.
- **연산**: `|` 합집합, `&` 교집합, `-` 차집합, `^` 대칭차.
- **멤버십**: `in`은 set에서 평균 O(1).
- **frozenset**: 불변 → dict 키·다른 set 요소로 사용 가능.

## 주의사항 (Pitfalls)

- **`{}`는 빈 dict**: 빈 set이 아니다. 빈 set은 반드시 `set()`.
- **순서 없음**: set을 순회할 때 순서에 의존하면 안 된다. 정렬이 필요하면 `sorted(s)`로 리스트를 만들어 쓴다.
- **요소는 hashable**: list, set, dict는 set 요소로 쓸 수 없다. tuple, str, int 등은 가능.
- **`{-1}`**: `-1` 하나를 가진 set. `{-1,}`도 같은 set이다. `{1}`만으로는 set인지 구분이 명확하므로, 단일 요소일 때 `{1,}`처럼 쉼표를 쓰면 tuple과 대비해 "한 요소 set"임이 분명해진다.

---

## Deep Insight

set은 해시 테이블로 구현되어 있어 "이 값이 있는가?"를 평균 O(1)에 판단한다. list에서 `in`을 쓰면 앞에서부터 하나씩 보므로 O(n)이다. 따라서 "많은 값 중에 있는지 반복 검사"할 때는 list를 set으로 바꿔 두고 `in`을 쓰는 것이 효율적이다. frozenset은 불변이므로 dict의 키나 다른 set의 요소로 쓸 수 있고, set끼리 합집합·교집합을 할 때도 frozenset을 쓸 수 있다.

---

## 요약

- **set**: 중복 없음, 순서 없음. 가변. `{1, 2, 3}`, `add`/`remove`/`discard`. **빈 set**은 `set()`으로만 만든다.
- **frozenset**: 불변 set. `frozenset(...)`. dict 키·다른 set 요소로 사용 가능.
- **멤버십 `in`**은 평균 O(1). `{1, 2} | {2, 3}` 등 집합 연산. 단일 요소 set은 `{-1}`, `{1,}` 등으로 표현.
