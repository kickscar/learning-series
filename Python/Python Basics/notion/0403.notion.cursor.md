# 4.3 while 문과 조건 반복 

## 개요

**`while`**은 **조건이 참인 동안** 블록을 반복 실행한다. `for`가 이터러블에서 값을 꺼내며 도는 것과 달리, `while`은 "조건이 False가 될 때까지" 돌린다. 루프 안에서 **`break`**를 만나면 **그 자리에서 루프를 빠져나가고**, **`continue`**를 만나면 **나머지 블록을 건너뛰고 다음 반복**으로 넘어간다. `for`와 `while` 모두 루프에 **`else`**를 붙일 수 있는데, 이 `else`는 **`break`로 빠져나가지 않고** 루프가 정상적으로 끝났을 때만 한 번 실행된다. 루프 제어와 "검색 실패" 등에 쓰이는 **for-else·while-else** 패턴을 이해하는 것이 이 단원의 목표다.

---

## 핵심 개념

### while 루프

`while 조건:` 다음에 들여쓴 블록은 **조건이 참인 동안** 반복 실행된다. 매 반복 시작 시 조건을 평가하고, 참이면 블록을 실행한 뒤 다시 조건을 검사한다. 조건이 거짓이 되면 루프를 빠져나간다. 조건은 Truthy/Falsy로 평가되므로 `while n > 0:`처럼 bool 식을 쓰거나, "무한 루프"를 만들 때는 `while True:`를 쓰고 안에서 `break`로 탈출하는 패턴이 자주 쓰인다. 조건이 처음부터 거짓이면 블록은 한 번도 실행되지 않는다.

```python
n = 3
while n > 0:
    print(n, end=" ")
    n -= 1
print("done")
```

실행 결과: `3 2 1 done`

### break — 루프 탈출

**`break`**는 **가장 안쪽의 `for` 또는 `while`** 루프를 **즉시 종료**한다. 조건을 다시 보지 않고, 그 루프 다음 문장으로 실행이 넘어간다. "원하는 것을 찾았을 때"나 "오류/종료 조건을 만났을 때" 루프를 끝내는 데 쓴다. 중첩 루프에서는 **한 단계**만 빠져나가므로, 바깥 루프까지 끝내려면 플래그 변수나 예외 등 다른 수단이 필요할 수 있다.

```python
for x in [1, 2, 3, 4, 5]:
    if x == 3:
        break
    print(x, end=" ")
print("out")
```

실행 결과: `1 2 out`

### continue — 다음 반복으로

**`continue`**는 **이번 반복의 나머지 코드를 건너뛰고**, **다음 반복**으로 넘어간다. `for`에서는 다음 요소로, `while`에서는 다시 조건을 평가한다. "이번만 제외하고 계속 돌리고 싶을 때" 사용한다. `continue`를 쓰면 들여쓰기가 깊어지지 않게 조건을 뒤집어서 쓸 수 있는 경우도 있다.

```python
for x in [1, 2, 3, 4]:
    if x % 2 == 0:
        continue
    print(x, end=" ")
print()
```

실행 결과: `1 3`

### 루프의 else — break 없이 끝났을 때

`for`와 `while`에는 **`else`** 절을 붙일 수 있다. 이 `else` 블록은 **루프가 `break`로 빠져나가지 않고** 정상적으로 끝났을 때만 실행된다. 즉, `for`는 이터러블이 다 소진되었을 때, `while`은 조건이 거짓이 되어 나왔을 때, 그때만 `else`가 실행된다. "시퀀스에서 뭔가를 찾다가 **못 찾고 끝났을 때**" 처리하는 패턴에 잘 맞는다. 예: `for item in items: if item == target: break` 다음 `else: print("not found")`. `break`로 나가면 `else`는 실행되지 않는다.

```python
items = [1, 2, 3]
target = 5
for x in items:
    if x == target:
        print("found")
        break
else:
    print("not found")
```

실행 결과: `not found`

## 실습 예제

### [실습] lab44

이 실습에서는 **while, break, continue, 루프-else**의 개념을 복습한다.

#### ① while — 조건이 참인 동안

`script01.py`

```python
n = 0
while n < 3:
    print(n, end=" ")
    n += 1
print("end")
```

실행 결과: `0 1 2 end`

#### ② break — 조건 만족 시 탈출

`script02.py`

```python
s = 0
for x in [1, 2, 3, 4, 5]:
    s += x
    if s >= 6:
        break
print(s)
```

실행 결과: `6`

#### ③ continue — 짝수 건너뛰기

`script03.py`

```python
for i in range(5):
    if i % 2 == 0:
        continue
    print(i, end=" ")
print()
```

실행 결과: `1 3`

#### ④ for-else — 찾지 못했을 때

`script04.py`

```python
lst = [10, 20, 30]
for x in lst:
    if x == 20:
        print("found")
        break
else:
    print("not found")
```

실행 결과: `found`

같은 리스트에서 99를 찾으면 `break`가 실행되지 않아 `else`가 실행되고 `not found`가 출력된다.

#### ⑤ 실습 포인트

- **while**: 조건이 참인 동안 반복. 조건이 거짓이 되면 종료.
- **break**: 루프를 즉시 종료. **continue**: 이번 반복 나머지 건너뛰고 다음 반복으로.
- **루프-else**: **break로 나가지 않고** 루프가 끝났을 때만 실행. "검색 실패" 등에 활용.

## 주의사항 (Pitfalls)

- **무한 루프**: `while True:`에서 `break`를 빼먹거나 조건이 영원히 참이면 무한 루프가 된다. `while` 조건이 언제 거짓이 되는지, 또는 `break`가 반드시 올 경로가 있는지 확인한다.
- **else의 의미**: 루프의 `else`는 "if가 거짓일 때"가 아니라 **"break 없이 정상 종료했을 때"**다. 처음 보면 헷갈릴 수 있으니, "break로 나가면 else 실행 안 함"으로 기억하면 된다.
- **중첩 루프**: `break`는 가장 안쪽 루프만 빠져나간다. 바깥 루프까지 끝내려면 플래그(`found = True`)를 두거나, 예외를 쓰는 방식 등을 고려한다.

## Deep Insight

`for`는 "이터러블을 소진할 때까지", `while`은 "조건이 참인 동안"이라는 다른 기준으로 반복하지만, **break·continue·else**는 두 루프에 공통으로 적용된다. 루프-else는 "검색 실패"처럼 "한 번에 찾지 못하고 끝까지 갔을 때" 한 번만 실행할 로직을 `break`와 짝으로 두기 좋은 패턴이다. 레거시 코드와 도움말에서 자주 나오므로, 의미를 알아두면 읽기와 유지보수가 수월해진다.

---

## 요약

- **while**: 조건이 참인 동안 반복. 조건이 거짓이 되면 종료.
- **break**: 루프 즉시 탈출. **continue**: 이번 반복 건너뛰고 다음 반복으로.
- **루프-else**: **break로 나가지 않고** 루프가 끝났을 때만 실행. "못 찾았을 때" 처리에 사용.
