# 2.1 변수란 — 이름과 값의 대응

## 개요

**변수**는 값에 붙인 **이름**이다. 이름을 쓰면 그 이름에 대응된 값이 사용된다. "변수 = 이름, 값 = 데이터"라는 기본 개념을 익히는 것이 이 단원의 목표다.

---

## 핵심 개념

- **이름(식별자)**: 값에 접근할 때 사용하는 문자. Python 식별자 규칙을 따른다.
  - 영문자, 숫자, `_` 사용. 숫자로 시작 불가.
  - 대소문자 구분 (`name`과 `Name`은 다른 이름).
- **대입 전**: 이름이 아직 정의되지 않았으면 그 이름을 쓰면 `NameError`가 난다.
- **대입 후**: 그 이름으로 값에 접근할 수 있다.

---

## 코드 예제

```python
# 이름에 값을 붙이기 전에는 사용할 수 없다.
# print(x)   # NameError: name 'x' is not defined

x = 10
print(x)   # 10 — 이름 x에 대응된 값이 사용됨
```

```python
# 식별자 규칙: 영문자, 숫자, _ (숫자로 시작 X)
count = 1
user_name = "Kim"
# 2nd = 2   # SyntaxError
```

---

## 주의사항 (Pitfalls)

- **정의되지 않은 이름 사용**: 대입 없이 이름만 쓰면 `NameError`. 반드시 대입으로 이름을 정의한 뒤 사용한다.
- **오타**: `usr_name`과 `user_name`은 다른 이름이므로, 오타 시 `NameError` 또는 의도와 다른 값이 사용될 수 있다.

---

## Deep Insight

변수는 "값이 들어 있는 상자"가 아니라 **이름과 값의 대응 관계**다. 같은 이름에 다시 대입하면, 그 이름은 새 값과 대응된다(이전 값으로의 연결이 바뀐다). 이 관점은 대입문·이름 사용 단원에서 이어진다.

---

## 요약

- 변수 = 값에 붙인 이름. 이름을 쓰면 대응된 값이 사용된다.
- 식별자 규칙을 지키고, 사용 전에 반드시 대입으로 이름을 정의한다.
- 대입 전 이름 사용 → `NameError`.
