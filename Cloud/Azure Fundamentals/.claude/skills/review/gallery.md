# Skill: review/gallery

Gallery 실습 섹션 draft 검토 체크리스트.

---

## 구조 체크

- [ ] H1 타이틀 없이 설명 텍스트로 시작하는가
- [ ] `### 실습 목표`가 있는가
- [ ] `# 1. 전체 아키텍처`가 있는가
- [ ] `# 2. 사전 준비`에 이전 Lab/섹션 완료 조건이 있는가
- [ ] 각 리소스 단계가 `# {n}. {리소스명}` H1으로 구성되는가
- [ ] 각 단계에 `### 1. 설정값`과 `### 2. 참고`가 있는가
- [ ] `# {n}. 리소스 확인` + `# {n+1}. 애플리케이션 실행` + `# {n+2}. 실행 및 결과 검증` 흐름이 있는가
- [ ] `# {n+3}. 자원 정리`에 "유지" 원칙이 명시되어 있는가

---

## 규칙 체크

- [ ] ⚠️ 비용 주의 섹션이 없는가
- [ ] 설정값 형식이 맞는가: `` `{값이름}` (예: azure-fund-gallery-...) ``
- [ ] 참고 링크 형식이 맞는가: `[섹션 실습 lab{no}: {제목}]({링크})`
- [ ] 자원 정리에 Gallery 인프라 삭제 지시가 없는가

---

## 내용 체크

- [ ] 전체 아키텍처가 이번 Chapter 변경 부분을 강조하는가
- [ ] 설정값이 독자 환경 맞춤 가능하도록 `{값이름}` 형식으로 표기되어 있는가
- [ ] 리소스 검증/앱 실행 단계에 `[콘솔화면: ...]` 플레이스홀더가 있는가

---

```
## 검토 결과: {ch_no}.{sec_no} Gallery 실습

OK:

수정 필요:
- {항목}: {이유}

- {항목}
```
