# **개요**

본 Section에서는 Terraform을 실제로 실행하기 위한 로컬 환경을 준비한다. Terraform 학습은 개념 이해만으로 끝나지 않고, `terraform init`, `terraform plan`, `terraform apply`를 직접 실행해보는 과정으로 이어지기 때문에, CLI 설치와 인증 환경 구성이 가장 먼저 정리되어야 한다.

이 문서의 목적은 운영체제별 설치 명령을 장황하게 나열하는 데 있지 않다. 이미 개발 도구 설치 경험이 있는 학습자를 전제로, Terraform CLI를 실행 가능한 상태로 만들고 AWS 실습을 진행할 수 있는 최소한의 환경 조건을 명확히 정리하는 데 목적이 있다.

# **핵심 포인트**

- Terraform 학습의 실제 시작점은 CLI를 실행 가능한 상태로 만드는 것이다.
- Terraform은 로컬에서 실행되지만, provider를 통해 외부 플랫폼과 통신하므로 인증 정보 준비가 필요하다.
- 설치가 끝났는지 여부는 단순히 파일이 있는지가 아니라 `terraform version`과 같은 검증 명령으로 확인해야 한다.
- AWS 실습 전에는 Terraform CLI와 AWS credentials가 모두 준비되어야 한다.
- 이 Section의 목표는 “설치 완료”가 아니라 “실행 가능한 작업 디렉토리 준비”까지 포함한다.

# **실습 목표**

- Terraform CLI를 로컬 환경에 설치한다.
- `terraform version`으로 설치 상태를 검증한다.
- AWS provider 사용을 위한 인증 정보를 준비한다.
- Terraform 작업 디렉토리를 초기화할 기본 구조를 이해한다.
- 이후 `terraform init`을 실행할 수 있는 준비 상태를 만든다.

# **Terraform CLI가 필요한 이유**

Terraform은 GUI 중심 도구가 아니라 CLI 기반 도구다. 사용자는 HCL로 작성된 Terraform configuration을 기준으로 `terraform init`, `terraform plan`, `terraform apply`, `terraform destroy` 같은 명령을 실행한다.

즉, Terraform 학습에서 설치는 부수 작업이 아니라 실행 모델의 일부다. CLI를 직접 다뤄야만 provider 초기화, 변경 계획 생성, state 반영 같은 핵심 동작을 확인할 수 있다.

CloudFormation 경험이 있는 학습자라면 콘솔과 서비스 중심으로 인프라 변경을 인식해왔을 가능성이 있다. Terraform은 반대로 로컬 작업 디렉토리와 CLI 명령을 중심으로 동작한다는 점에서 시작 경험이 다르다.

# **설치 전 확인할 항목**

Terraform CLI를 설치하기 전에 다음 항목을 확인하는 것이 좋다.

| 항목 | 확인 이유 |
| --- | --- |
| 운영체제 | 설치 방식이 macOS, Linux, Windows에 따라 달라질 수 있다. |
| 터미널 환경 | Terraform은 터미널에서 실행되므로 기본 shell 환경이 필요하다. |
| PATH 설정 | `terraform` 명령을 어느 위치에서든 실행할 수 있어야 한다. |
| AWS 계정 및 권한 | 이후 AWS provider 실습을 위해 유효한 계정과 최소 권한이 필요하다. |
| 텍스트 편집기 또는 IDE | `.tf` 파일을 작성할 편집 도구가 필요하다. |

이 항목들은 기초적인 내용처럼 보이지만, 실제로는 “설치는 했는데 명령이 실행되지 않는다”거나 “provider 인증이 되지 않는다”는 문제의 대부분이 여기에서 발생한다.

# **Terraform 설치 방식**

Terraform은 일반적으로 다음 방식 중 하나로 설치한다.

| 방식 | 설명 |
| --- | --- |
| 공식 바이너리 다운로드 | HashiCorp가 제공하는 바이너리를 직접 내려받아 설치 |
| 패키지 매니저 사용 | Homebrew, apt, dnf, choco 등 운영체제 패키지 도구 사용 |
| 버전 관리 도구 사용 | tfenv 같은 도구로 버전 전환과 고정 관리 |

입문 단계에서는 공식 문서 또는 신뢰 가능한 패키지 매니저 기반 설치가 가장 단순하다. 중요한 점은 설치 방식보다 **설치 후 검증**이다.

Terraform Basics에서는 특정 운영체제별 상세 설치 명령보다 다음 원칙을 우선한다.

- 공식 경로로 설치한다.
- 버전을 확인한다.
- PATH에서 실행 가능한지 검증한다.
- 이후 실습에서 사용하는 버전 범위를 일관되게 유지한다.

# **설치 검증 : `terraform version`**

설치가 끝난 뒤 가장 먼저 확인해야 하는 명령은 다음이다.

```bash
terraform version
```

이 명령은 단순히 버전을 보여주는 것 이상의 의미를 가진다.

- 실행 파일이 정상 설치되었는지
- PATH에서 명령을 찾을 수 있는지
- 현재 환경에서 Terraform CLI가 동작 가능한지

즉, 설치 검증의 기준은 “설치했다”가 아니라 “명령이 정상적으로 실행된다”다.

필요하면 이후 학습의 일관성을 위해 특정 버전 범위를 정해둘 수 있다. 예를 들어 Section 문서나 예제에서 `>= 1.6.0` 같은 요구 버전을 명시하면, 실습 간 버전 차이로 생기는 혼선을 줄일 수 있다.

# **AWS 인증 환경 준비**

Terraform이 AWS resource를 다루려면 AWS provider가 AWS API와 통신할 수 있어야 한다. 이를 위해 로컬 실행 환경에는 유효한 AWS credentials가 준비되어 있어야 한다.

Terraform Basics에서 중요한 점은 인증 메커니즘 자체를 깊게 설명하는 것이 아니라, Terraform provider가 외부 플랫폼과 통신하려면 인증 정보가 필요하다는 사실을 이해하는 것이다.

일반적으로 다음 방식 중 하나를 사용할 수 있다.

| 방식 | 설명 |
| --- | --- |
| AWS CLI 기본 profile | `aws configure` 등으로 설정된 기본 profile 사용 |
| 환경 변수 | `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_REGION` 등 사용 |
| named profile | 특정 profile을 지정하여 사용 |
| 조직 표준 인증 방식 | SSO, role assume, credential process 등 |

입문 실습에서는 복잡한 조직 표준보다, Terraform provider가 인증 가능한 상태를 만드는 것이 우선이다. 다만 문서에는 실제 access key 값을 절대 포함하지 않는다.

> Terraform은 로컬에서 실행되지만, provider는 외부 플랫폼 API와 통신한다. 따라서 CLI 설치와 인증 준비는 분리된 문제가 아니라 하나의 실행 준비 과정이다.

# **작업 디렉토리와 기본 파일 구조**

Terraform은 특정 프로젝트 디렉토리 안의 `.tf` 파일을 기준으로 동작한다. 따라서 설치가 끝난 뒤에는 작업 디렉토리 개념도 함께 이해해야 한다.

입문 단계에서 필요한 최소 구조는 다음 정도로 충분하다.

```text
terraform-project/
└── main.tf
```

이후 학습이 진행되면 `variables.tf`, `outputs.tf`, `providers.tf`, `terraform.tfvars` 같은 파일로 구조가 확장될 수 있지만, 설치 직후 단계에서는 먼저 “Terraform은 작업 디렉토리 안의 configuration을 읽는다”는 개념을 잡는 것이 중요하다.

이 구조를 이해해야 다음 Section이나 첫 프로젝트 실습에서 `terraform init`이 왜 디렉토리 단위로 실행되는지 자연스럽게 연결된다.

# **실행 및 검증**

설치와 인증 준비가 끝났다면 다음 항목을 순서대로 확인한다.

## 1. Terraform 명령 실행 가능 여부

```bash
terraform version
```

정상 결과가 나온다면 CLI 자체는 준비된 것이다.

## 2. 작업 디렉토리 준비

```text
terraform-project/
└── main.tf
```

간단한 빈 디렉토리 또는 최소 configuration 파일을 둘 준비를 한다.

## 3. AWS 인증 준비 여부 확인

Terraform 실행 전 다음을 확인한다.

- 사용할 AWS 계정이 정해져 있는지
- provider가 참조할 region이 정해져 있는지
- credentials 또는 profile이 준비되었는지

## 4. 다음 Section 또는 실습으로의 연결 상태 확인

설치 단계의 최종 목표는 다음 질문에 “예”라고 답할 수 있는 상태다.

- `terraform` 명령이 터미널에서 실행되는가
- `.tf` 파일을 둘 디렉토리를 만들 수 있는가
- AWS provider가 참조할 인증 정보가 준비되어 있는가

# **검증 포인트**

- `terraform version`이 오류 없이 실행되는가
- Terraform CLI가 PATH에서 인식되는가
- AWS credentials를 어떤 방식으로 사용할지 정리됐는가
- 이후 `terraform init`을 실행할 작업 디렉토리 준비가 되었는가

# **정리**

이 Section에서는 Terraform 설치와 AWS 실습을 위한 기본 환경 구성을 정리했다. Terraform 학습에서 설치는 단순 준비 작업이 아니라, CLI 중심 실행 모델을 실제로 시작하기 위한 첫 단계다.

또한 Terraform은 로컬에서 명령을 실행하지만, provider를 통해 외부 플랫폼과 통신하므로 인증 환경 준비가 함께 필요하다는 점을 확인했다. 이후 Chapter부터는 HCL 작성, Terraform workflow, Terraform state 같은 핵심 개념을 실제 configuration과 함께 더 구체적으로 다루게 된다.
