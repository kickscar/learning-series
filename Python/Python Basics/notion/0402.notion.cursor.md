# 4.2 for문과 순회(iteration)

## 개요

`for x in it:`은 **이터러블(it)**에서 값을 하나씩 꺼내 `x`에 넣으며 반복한다. Python에서 `for`는 "시퀀스 인덱스"가 아니라 **이터러블 프로토콜**에 의존한다. `iter(it)`으로 **이터레이터**를 얻고, 그 이터레이터에 `next()`를 반복 호출하다가 **`StopIteration`**이 나면 루프가 끝난다. `for`가 시퀀스가 아닌 이터러블에 의존한다는 점과, `iter()`·`next()`의 기본 사용을 이해하는 것이 이 단원의 목표다.

---

## 핵심 개념

### for 루프가 하는 일

`for x in it:`을 실행하면 Python은 먼저 **`iter(it)`**을 호출해 **이터레이터**를 얻는다. 그 다음 루프마다 이터레이터에 **`next(이터레이터)`**를 호출해 다음 값을 받아 `x`에 넣고 블록을 실행한다. `next()`가 **`StopIteration`** 예외를 일으키면(더 줄 값이 없으면) 루프가 종료된다. 따라서 "반복 가능한 것"은 **`iter()`를 쓸 수 있는 객체**, 즉 **이터러블**이다. list, tuple, range, str, dict, set 등은 모두 이터러블이라서 `for`에 그대로 쓸 수 있다.

```python
for i in [1, 2, 3]:
    print(i, end=" ")
print()
for c in "ab":
    print(c, end=" ")
```

실행 결과: `1 2 3` / `a b`

### 이터러블과 이터레이터

**이터러블**은 `iter()`에 넘기면 **이터레이터**를 돌려주는 객체다. 이터레이터는 **`__next__`**를 갖고 있어서 `next(it)`를 호출할 때마다 "다음 값"을 반환하고, 더 없으면 `StopIteration`을 발생시킨다. `for`는 내부적으로 `iter(대상)` → 반복적으로 `next(이터레이터)` → `StopIteration`이 나면 종료라는 흐름이다. 리스트나 문자열처럼 **`__iter__`**를 제공하는 타입은 `iter()`에 넘기면 새 이터레이터가 만들어진다. 한 이터레이터는 **한 번만** 앞으로만 진행하며, 끝까지 가면 재사용할 수 없다. 새로 순회하려면 다시 `iter()`로 이터레이터를 만들어야 한다.

```python
it = iter([10, 20, 30])
print(next(it), next(it), next(it))
# next(it)   # StopIteration
```

실행 결과: `10 20 30`

### __iter__와 __next__

객체가 **이터러블**이 되려면 **`__iter__`** 메서드를 제공하면 된다. `iter(obj)`는 `obj.__iter__()`를 호출하고, 그 반환값이 이터레이터여야 한다. 이터레이터는 **`__next__`**를 구현해, `next(이터레이터)` 호출 시 다음 값을 반환하거나 `StopIteration`을 발생시킨다. 흔히 이터레이터는 **`__iter__`**도 구현하는데, `self`를 반환해 "자기 자신이 이터레이터"임을 만족시킨다. 시퀀스(list, str 등)는 예전에 **`__getitem__`**으로 인덱스 접근이 가능하면 이터레이터를 만들어 주는 fallback이 있지만, **`for`는 `__iter__`를 우선** 사용한다. 따라서 "순서가 있다"는 것만으로는 부족하고, **`__iter__`**가 있어야 이터러블로 인정된다.

### range, dict, enumerate

**`range(n)`**은 이터러블이다. 0부터 n 미만의 정수를 순서대로 내주는 이터레이터를 만든다. **`dict`**에 `for`를 쓰면 **키**만 순회한다. `for k in d:` → 키 순회. 키와 값을 함께 쓰려면 `for k, v in d.items():`를 쓴다. **`enumerate(이터러블)`**은 (인덱스, 요소) 쌍을 주는 이터러블이라, `for i, x in enumerate(lst):`처럼 인덱스와 값을 동시에 다룰 때 유용하다.

```python
for i in range(2, 5):
    print(i, end=" ")
print()
d = {"a": 1, "b": 2}
for k in d:
    print(k, end=" ")
print()
for i, v in enumerate(["x", "y"]):
    print(i, v)
```

실행 결과: `2 3 4` / `a b` / `0 x` `1 y`

## 실습 예제

### [실습] lab43

이 실습에서는 **for와 이터러블, iter(), next()**의 개념을 복습한다.

#### ① for로 list·str 순회

`script01.py`

```python
for x in [10, 20, 30]:
    print(x)
for c in "Hi":
    print(c)
```

실행 결과: `10` `20` `30` / `H` `i`

#### ② iter()와 next()

`script02.py`

```python
it = iter([1, 2, 3])
print(next(it), next(it), next(it))
```

실행 결과: `1 2 3`

#### ③ range·enumerate

`script03.py`

```python
for i in range(0, 6, 2):
    print(i, end=" ")
print()
for i, name in enumerate(["a", "b", "c"]):
    print(i, name)
```

실행 결과: `0 2 4` / `0 a` `1 b` `2 c`

#### ④ dict 순회 — keys, items

`script04.py`

```python
d = {"x": 1, "y": 2}
for k in d:
    print(k, end=" ")
print()
for k, v in d.items():
    print(k, v)
```

실행 결과: `x y` / `x 1` `y 2`

#### ⑤ 실습 포인트

- **for**는 `iter(대상)`으로 이터레이터를 얻고, **`next()`**를 반복하다 **`StopIteration`**에서 종료.
- **이터러블**: `iter()`에 넣을 수 있는 객체. **이터레이터**: `next()`로 다음 값을 주는 객체.
- **`iter()`·`next()`**로 수동 순회 가능. 이터레이터는 한 번만 진행, 재사용 불가.
- **range**, **dict**(키 순회), **enumerate**(인덱스·값)는 자주 쓰는 이터러블.

---

## 주의사항 (Pitfalls)

- **이터레이터 소진**: 한 이터레이터는 한 방향으로만 진행한다. 끝까지 가면 `next()`는 `StopIteration`만 낸다. 다시 순회하려면 `iter(원본)`으로 새 이터레이터를 만들어야 한다.
- **dict 순회**: `for x in d:`는 **키**만 순회한다. 값만 필요하면 `d.values()`, 키와 값이 필요하면 `d.items()`를 쓴다.
- **순회 중 수정**: list 등을 `for`로 돌리면서 그 list에 append/remove하면 반복 횟수나 요소가 바뀌어 예상과 다를 수 있다. 필요한 경우 복사본을 순회하거나 인덱스를 역순으로 쓰는 등으로 피한다.

## Deep Insight

`for`가 "인덱스 0, 1, 2, …"가 아니라 **이터레이터 프로토콜**에 의존한다는 점은 Python이 list·tuple뿐 아니라 파일, 제너레이터, 사용자 정의 타입까지 같은 `for` 문법으로 다룰 수 있게 한다. `__iter__`와 `__next__`만 구현하면 어떤 객체든 `for x in my_obj:`에 쓸 수 있다. 시퀀스는 "길이와 인덱스"가 있지만, 이터러블은 "다음 값이 있는가?"만 있으면 되어 무한 스트림이나 한 번만 소비하는 데이터에도 맞는다.

---

## 요약

- **for x in it:** → `iter(it)`으로 이터레이터 획득, **`next()`** 반복, **`StopIteration`** 시 종료.
- **이터러블**: `iter()` 가능. **이터레이터**: `next()`로 다음 값 반환. `for`는 **`__iter__`** 우선.
- **range**, **dict**(키), **enumerate**, **str** 등은 이터러블. `iter()`·`next()`로 수동 순회 가능.
