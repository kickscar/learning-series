## 콘솔 경로

### GCS 스테이징 버킷 생성 (JAR 배포용)

1. Google Cloud Console 홈
2. 탐색 메뉴 > Cloud Storage > **버킷**
3. 만들기 클릭
4. 버킷 이름 입력 (전 세계 고유) — 예: `gcp-fund-gallery-staging-{PROJECT_ID}`
5. 위치 유형: Region, `us-central1`
6. 스토리지 클래스: Standard
7. 액세스 제어: 균일한 버킷 수준 액세스
8. 만들기 클릭

### Cloud Shell — JAR 빌드 및 업로드

Cloud Shell은 Console 우측 상단 터미널 아이콘 클릭으로 열 수 있다.

```bash
# 저장소 클론
git clone https://github.com/kickscar/learning-series.git
cd learning-series/Cloud/Workloads/gallery-spring-boot

# JAR 빌드
./mvnw clean package -Dbuild.finalName=gallery -DskipTests

# GCS 업로드
gsutil cp target/gallery.jar gs://{BUCKET_NAME}/gallery.jar
```

### VM 생성 (Startup Script 포함)

1. Google Cloud Console > Compute Engine > **VM 인스턴스**
2. 인스턴스 만들기
3. 기본 설정: 이름, 리전/영역, Machine Type, Boot Disk
4. 고급 옵션 > **관리** 탭: 자동화 > 시작 스크립트 입력
5. 고급 옵션 > **네트워킹** 탭: Network Tag 입력

### VM 상태 확인

1. Google Cloud Console > Compute Engine > **VM 인스턴스**
2. `gcp-fund-gallery-vm` 클릭 → 세부정보
3. 직렬 포트 탭: Startup Script 실행 로그 확인

## 주요 설정

- VM Name: `gcp-fund-gallery-vm`
- Region: `us-central1` / Zone: `us-central1-a`
- Machine Type: `e2-medium`
- Boot Disk: Ubuntu 22.04 LTS, Balanced PD, 20GB
- Network Tags: `allow-ssh`, `allow-http`
- External IP: Ephemeral
- Startup Script: 아래 내용 입력

## Startup Script 전체 (startup-script.sh)

```bash
#!/bin/bash
set -e

BUCKET_NAME="gcp-fund-gallery-staging-PROJECT_ID"
APP_DIR="/app"
LOG_FILE="/var/log/gallery.log"

# 1. JDK 21 설치 (Adoptium Temurin)
apt-get update -y
apt-get install -y wget apt-transport-https gpg

wget -qO - https://packages.adoptium.net/artifactory/api/gpg/key/public | \
  gpg --dearmor | \
  tee /etc/apt/trusted.gpg.d/adoptium.gpg > /dev/null

echo "deb https://packages.adoptium.net/artifactory/deb $(awk -F= '/^VERSION_CODENAME/{print$2}' /etc/os-release) main" | \
  tee /etc/apt/sources.list.d/adoptium.list

apt-get update -y
apt-get install -y temurin-21-jre

# 2. JAR 다운로드 (GCS)
mkdir -p $APP_DIR
gsutil cp gs://$BUCKET_NAME/gallery.jar $APP_DIR/gallery.jar

# 3. GCE 인스턴스 이름 조회
INSTANCE_ID=$(curl -sf -H "Metadata-Flavor: Google" \
  http://metadata.google.internal/computeMetadata/v1/instance/name || echo "gallery-vm")

# 4. systemd 서비스 등록
cat > /etc/systemd/system/gallery.service << EOF
[Unit]
Description=Gallery Spring Boot App
After=network.target

[Service]
Type=simple
WorkingDirectory=$APP_DIR
ExecStart=/usr/bin/java -jar $APP_DIR/gallery.jar --app.runtime.instance-id=$INSTANCE_ID
Restart=on-failure
StandardOutput=append:$LOG_FILE
StandardError=append:$LOG_FILE

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable gallery
systemctl start gallery
```

## 조사 메모

- Startup Script는 VM 생성 시 **고급 옵션 > 관리 탭 > 자동화 > 시작 스크립트** 에 붙여넣는다
- GCS 버킷은 전 세계 고유해야 해서 Project ID를 suffix로 사용 권장
- VM의 기본 Service Account에 GCS 읽기 권한(Storage Object Viewer)이 있어야 gsutil 동작
  - 기본 Compute Engine SA는 `{PROJECT_NUMBER}-compute@developer.gserviceaccount.com`
  - 기본적으로 GCS 읽기 권한 포함되어 있음 (같은 프로젝트 버킷이면 동작)
- JDK 대신 JRE (temurin-21-jre)로 충분 — 실행만 하면 됨
- 앱 포트: 8080 (dev 프로파일 기본)
- Firewall Rule `allow-http`(TCP 8080)는 lab05에서 이미 생성됨 — 재사용
- `EC2InstanceIdentity`: GCE Metadata Server(`metadata.google.internal`)에서 instance/name 조회
  - IMDSv2 실패 시 `"local"` 폴백 → GCE metadata로 명시 주입
