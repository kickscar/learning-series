# curate/sync — SSoT 동기화 검증

## 목적

series-hierarchy.md (SSoT)와 실제 파일시스템을 비교하여 불일치를 탐지하고 수정한다.

## 실행 시점

- curate/workflow Phase 3 완료 후 자동
- 사용자 수동 요청 시

## 실행 단계

### Step 1: 기대 구조 추출

series-hierarchy.md에서 추출:
- 챕터 디렉토리 목록
- 섹션 디렉토리 목록
- README.md 파일 목록
- Draft 파일 목록 (`{ch_no}.{sec_no}.claude.notion.md`)

### Step 2: 실제 상태 수집

파일시스템 스캔:
- 실제 존재하는 디렉토리
- 실제 존재하는 README.md
- 실제 존재하는 draft 파일

### Step 3: Diff 생성

| 항목 | 설명 |
|------|------|
| 누락 디렉토리 | SSoT에 있으나 파일시스템에 없음 |
| 잉여 디렉토리 | 파일시스템에 있으나 SSoT에 없음 |
| 누락 README.md | 디렉토리 존재하나 README.md 없음 |
| Draft 파일명 불일치 | 번호 변경으로 인한 이름 불일치 |

### Step 4: 보고 및 승인 요청

Diff 결과를 사용자에게 보고하고 수정 승인 요청

### Step 5: 수정 실행

승인된 항목만 실행:
- 디렉토리 생성/삭제
- 파일 이름 변경
- 내용 있는 파일은 삭제 전 확인

### Step 6: 결과 보고

수정 결과와 최종 상태 보고

## 규칙

- 내용이 있는 파일을 삭제할 때는 반드시 사용자 확인
- Diff만 보고하고 승인 후 실행 (자동 수정 금지)
