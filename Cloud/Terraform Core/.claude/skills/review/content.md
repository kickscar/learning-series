# Skill: review/content

이론/개념 섹션 draft 검토 스킬.

---

## 체크리스트

### 1. 구조 준수

- [ ] 파일명: `{ch_no}.{sec_no}.claude.notion.md` 형식
- [ ] 최상위 헤딩: `# Ch{no}. {챕터명} / Sec{no}. {섹션명}`
- [ ] 필수 섹션 존재: 개요, 핵심 내용, 마무리
- [ ] 헤딩 계층 올바름 (h1 → h2 → h3, skip 없음)

### 2. 내용 완결성

- [ ] 개요: 섹션 목적과 학습 범위 명시
- [ ] 핵심 내용: 3개 이상 개념/메커니즘 설명
- [ ] 각 개념에 HCL 예제 또는 CLI 예제 포함
- [ ] 마무리: 핵심 정리 bullet + 다음 섹션 연결

### 3. TF Core 원칙

- [ ] AWS 서비스 설명 없음 (실습 수단으로만 언급)
- [ ] 클라우드 개념 설명 없음 (Terraform 도구에 집중)
- [ ] 설계 당위("왜 이렇게 나눠야 하는가") 없음 — Architecture & Design 영역

### 4. Terraform 버전 표기

- [ ] 버전 민감 기능에 `<!-- 요구 버전: 1.x.x+ -->` 어노테이션 있음
- [ ] 버전 범위가 plan 파일 "요구 버전" 컬럼과 일치

### 5. 코드 블록 규칙 (rules/code-blocks.md)

- [ ] HCL 코드 블록: ` ```hcl ` 언어 지정
- [ ] CLI 명령어: ` ```bash ` 언어 지정
- [ ] terraform output 결과: ` ```text ` 언어 지정
- [ ] 파일 트리: ` ```text ` 언어 지정
- [ ] 인라인 코드: 파일명·속성명·블록명은 `` ` `` 사용

### 6. 한국어 작성 규칙 (rules/korean-writing.md)

- [ ] 기술 용어 영문 유지 (번역 없음)
- [ ] 문체: `~다` 체 (존댓말 아님)
- [ ] 영문 용어 첫 등장 시 한글 병기 여부 (필요한 경우)

### 7. 스타일 가이드 (rules/style-guide.md)

- [ ] HCL 파일 참조: `main.tf`, `variables.tf` (백틱 사용)
- [ ] CLI 명령어 참조: `terraform init` (백틱 사용)

## 판정 기준

- **OK**: 모든 항목 통과 — writer에게 README 작성 진행 신호
- **수정 필요**: 1개 이상 항목 실패 — 위치와 이유 명시 후 writer 재실행 요청
- **기술 오류 의심**: 체크리스트 외 기술 내용 이상 발견 시 별도 flagging, 수정은 사용자 판단
