# Chapter 2: 변수 — 이름과 값 (basic 입문자)

## Chapter Definition

### 왜 필요한가

프로그램에서 **값을 담아 두고 다시 쓰기 위해** 이름을 붙이는 것이 변수다. 입문자 수준에서 "변수란 무엇인가", "대입으로 이름에 값을 붙인다", "이름을 통해 값을 읽고 쓴다"를 다룬다.

### 세부 주제

1. 변수란 — 이름과 값의 대응
2. 대입문 — `a = 1`, `a = b`
3. 이름 사용 — 읽기와 쓰기
4. 변수와 타입 — 값에 따른 타입 (구체 타입은 Ch3에서)
5. 여러 개 대입 — `a, b = 1, 2`, 튜플 언패킹 맛보기

### 간단 소개

변수의 정의, 대입문으로 이름에 값을 붙이는 방법, 이름을 통해 값을 읽고 쓰는 방법, 값의 타입과 변수의 관계, 여러 이름에 한 번에 대입하는 문법을 입문자 수준으로 다룬다.

## Subject Definition

### 2.1 변수란

- **개요**: 변수는 "값에 붙인 이름". 이름을 쓰면 그 이름에 대응된 값이 사용된다.
- **목표**: "변수 = 이름, 값 = 데이터"라는 기본 개념을 익힌다.
- **내용 요약**: 이름(식별자) 규칙. 대입 전에는 이름이 정의되지 않음(`NameError`). 대입 후에는 그 이름으로 값에 접근.

### 2.2 대입문

- **개요**: `이름 = 표현식`. 오른쪽 표현식을 계산한 결과를 왼쪽 이름에 붙인다.
- **목표**: `a = 1`, `a = b`, `a = a + 1` 같은 대입의 의미를 안다.
- **내용 요약**: 대입은 이름에 값을 붙이는 것. 입문 단계에서는 "a에 값이 들어간다" 수준으로 다룬다.

### 2.3 이름 사용

- **개요**: 이름을 표현식에서 쓰면, 그 이름에 붙어 있는 값이 사용된다. 같은 이름에 다시 대입하면 새 값으로 바뀐다.
- **목표**: 읽기와 쓰기(대입)를 구분한다.
- **내용 요약**: `print(a)`는 읽기. `a = 2`는 쓰기. 대입되지 않은 이름을 읽으면 `NameError`.

### 2.4 변수와 타입

- **개요**: 변수 자체에는 "타입"이 고정되지 않는다. 값에 타입이 있고, 변수는 어떤 값이든 가리킬 수 있다. Ch3에서 int, str, list 등 구체 타입을 다룬다.
- **목표**: "변수의 타입"보다 "값의 타입"이라는 관점을 안다.
- **내용 요약**: `a = 1` 후 `a = "hello"` 가능. `type(a)`는 현재 a가 가리키는 값의 타입.

### 2.5 여러 개 대입

- **개요**: `a, b = 1, 2`, `a, b = b, a`(스왑). 튜플·리스트 언패킹 맛보기.
- **목표**: 한 줄에서 여러 이름에 값을 붙이는 문법을 쓸 수 있게 한다.
- **내용 요약**: 좌변과 우변 개수 일치. `x, *rest = [1, 2, 3]`는 Ch3·Ch4 이후에 다룰 수 있음.

---

## Modern Key Concepts

- **이름 = 값 대응**: 변수는 "상자"가 아니라 이름-값 바인딩.
- **대입은 문(statement)**: `a = 1`은 표현식이 아니라 문. `a = b = 0` 같은 연쇄 대입 가능.
