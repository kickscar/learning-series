# 1.4 실행자: python / python3 / shebang / py launcher

## 개요

파이썬 코드를 실행하려면 **어떤 Python 실행 파일**을 사용할지 결정해야 한다. 시스템에 여러 버전의 파이썬이 설치되어 있거나, 가상환경을 사용할 때는 이 선택이 더 중요해진다. `python`과 `python3`의 차이, shebang으로 스크립트에 실행자를 지정하는 방법, Windows의 `py` launcher까지 이해하지 못하면 "왜 다른 환경에서는 되는데 여기서는 안 되지?"라는 혼란이 생긴다.

Linux/macOS에서는 `python`이 Python 2를 가리킬 수 있어, Python 3를 쓰려면 `python3`를 명시해야 한다. shebang `#!/usr/bin/env python3`는 `PATH`에서 `python3`를 찾아 실행하므로, 교차 플랫폼과 버전 선택에 유리하다. Windows에서는 `py` launcher로 `py -3.11`처럼 버전을 지정할 수 있다. 이 주제를 이해하면 가상환경 활성화 시 `python`이 venv를 가리키게 되는 흐름도 자연스럽게 이어진다.

---

## 핵심 개념

### python vs python3

Linux와 macOS에서는 전통적으로 **`python`**이 Python 2를, **`python3`**가 Python 3를 가리키도록 심볼릭 링크가 설정되어 있다. Python 2가 deprecated된 지금도, 일부 시스템에서는 `python`이 여전히 Python 2를 가리키거나, `python3`만 설치되어 있을 수 있다.

- **`python`**: 시스템에 따라 Python 2 또는 Python 3를 가리킬 수 있다. 최신 배포판에서는 `python`이 Python 3를 가리키는 경우가 늘고 있지만, 보장되지 않는다.
- **`python3`**: Python 3를 가리킨다. Python 3를 쓰려면 `python3`를 명시하는 것이 안전하다.

Windows에서는 보통 `python`만 설치되어 있고, `python3`는 별도로 없을 수 있다. 설치 경로에 따라 `python`이 Python 3를 가리킨다.

### shebang (해시뱅)

스크립트 파일의 **첫 번째 줄**에 `#!`로 시작하는 shebang(해시뱅)을 넣으면, 그 스크립트를 직접 실행할 때 어떤 인터프리터를 쓸지 지정할 수 있다.

**형식:**

```
#!/usr/bin/env python3
```

**`/usr/bin/env python3`를 쓰는 이유:**

- `env`는 `PATH` 환경 변수에서 `python3`를 찾아 실행한다. 따라서 **어디에 Python이 설치되어 있든** `PATH`에만 있으면 동작한다.
- `/usr/bin/python3`처럼 절대 경로를 쓰면, Python이 다른 경로에 설치된 시스템에서는 동작하지 않을 수 있다.
- 가상환경을 활성화한 상태에서는 `PATH`에 venv의 `python3`가 먼저 오므로, shebang `#!/usr/bin/env python3`로 실행해도 **venv의 Python**이 사용된다.

### py launcher (Windows)

Windows에서는 **`py` launcher**를 사용해 여러 버전의 Python 중 하나를 선택할 수 있다. `py`는 Python 설치 시 함께 설치되는 도구다.

- **버전 지정**: `py -3.11 script.py`, `py -3 script.py`
- **설치된 버전 확인**: `py -0`, `py -0p` (경로 포함)
- **가상환경과의 관계**: `py` launcher는 전역 Python 버전을 선택하는 도구다. 가상환경 활성화 후에는 `python` 명령이 venv를 가리키므로, `py` 대신 `python`을 쓰는 것이 일반적이다.

### 가상환경 활성화 시 python이 가리키는 것

가상환경(venv)을 **활성화**하면, `PATH` 환경 변수 앞에 venv의 `bin`/`Scripts` 디렉터리가 추가된다. 따라서 `python` 명령은 **venv의 Python**을 먼저 찾게 된다. `deactivate`를 실행하면 `PATH`가 원래대로 돌아가고, `python`은 다시 전역 Python을 가리킨다.

### python -m과 실행자

`python -m pip`, `python -m venv`처럼 `-m` 옵션을 쓸 때는 **어떤 `python`을 쓰는가**가 중요하다. 가상환경이 활성화되어 있으면 `python`이 venv의 Python을 가리키므로, `python -m pip install requests`는 venv에 패키지를 설치한다.

## 코드 예제

### 1) python vs python3 확인 (lab01)

```bash
# Linux/macOS
$ python --version
$ python3 --version

# 어느 경로를 쓰는지 확인
$ which python
$ which python3
```

**아하 모먼트**: 같은 시스템에서 `python`과 `python3`가 다른 실행 파일을 가리킬 수 있다. Python 3를 쓰려면 `python3`를 명시하는 것이 안전하다.

### 2) shebang으로 스크립트 실행 (lab02)

```python
# lab02/script.py
#!/usr/bin/env python3
import sys
print(f"Python: {sys.executable}")
print(f"Version: {sys.version}")
```

```bash
$ chmod +x script.py
$ ./script.py
```

**가상환경 활성화 후** `./script.py`를 실행하면 venv의 Python이 사용된다. shebang을 바꾸지 않아도 가상환경이 적용된다.

### 3) py launcher (Windows) (lab03)

```cmd
py -0
py -3.11 script.py
py -3.10 script.py
```

**아하 모먼트**: `py -3.11`처럼 버전을 지정하면, 여러 Python이 설치된 환경에서 원하는 버전을 선택할 수 있다.

### 4) 가상환경 활성화 전후 python 경로 (lab04)

```bash
# 활성화 전
$ which python

# venv 활성화
$ source .venv/bin/activate
(.venv) $ which python

# 비활성화
(.venv) $ deactivate
$ which python
```

**아하 모먼트**: 활성화 시 `python`이 venv의 Python을 가리킨다. `deactivate` 후에는 다시 전역 Python을 가리킨다.

### 5) python -m pip로 "어떤 Python"에 설치할지 명확히 (lab05)

```bash
# 전역 Python 3에 설치 (venv 비활성화 상태)
$ python3 -m pip install requests

# venv에 설치 (venv 활성화 상태)
$ source .venv/bin/activate
(.venv) $ python -m pip install requests
```

**아하 모먼트**: `python -m pip`는 "현재 `python`이 가리키는 환경"에 설치한다. venv 활성화 여부에 따라 설치 대상이 달라진다. `pip` 명령만 쓰면 어떤 `pip`인지 헷갈릴 수 있어, `python -m pip`를 쓰는 것이 권장된다.

## 주의사항 (Pitfalls)

### 1) python이 Python 2를 가리키는 경우

Linux/macOS의 일부 시스템에서는 `python`이 여전히 Python 2를 가리킨다. Python 2는 2020년에 지원이 종료되었으므로, Python 3를 쓰려면 `python3`를 명시해야 한다.

### 2) shebang에 절대 경로를 쓰는 것

`#!/usr/bin/python3`처럼 절대 경로를 쓰면, Python이 다른 경로에 설치된 시스템에서는 동작하지 않는다. `#!/usr/bin/env python3`를 쓰면 `PATH`에서 `python3`를 찾으므로, 설치 위치에 덜 의존한다.

### 3) Windows에서 python 명령이 없는 경우

Windows에서 `python`을 PATH에 추가하지 않고 설치한 경우, `python` 명령이 동작하지 않을 수 있다. 이때는 `py` launcher를 쓰거나, Python 설치 경로를 PATH에 추가해야 한다. 설치 시 "Add Python to PATH" 옵션을 체크하면 보통 해결된다.

### 4) 가상환경 활성화를 잊는 경우

venv를 만들기만 하고 활성화하지 않으면, `python`은 여전히 전역 Python을 가리킨다. `pip install`도 전역에 설치된다. 프로젝트 디렉터리에서 작업할 때는 **항상 venv를 활성화**하는 습관이 필요하다.

### 5) 실행 시점에 결정되는 실행자

파이썬은 **실행 시점**에 어떤 `python`을 쓰는지가 결정된다. shebang, `PATH`, venv 활성화가 이 선택에 영향을 준다. 스크립트를 실행할 때마다 현재 환경의 `PATH`가 적용된다.

## Deep Insight

### env의 역할

`#!/usr/bin/env python3`에서 `env`는 **현재 환경의 `PATH`**를 사용해 `python3`를 찾는다. 따라서:

- 시스템 Python이 `/usr/bin`에 있으면 `/usr/bin/python3`를 실행한다.
- venv가 활성화되어 있으면 `PATH`에 venv의 `bin`이 먼저 오므로, venv의 `python3`를 실행한다.
- `conda` 등 다른 환경 관리자를 쓰면, 그 환경의 `python3`를 실행한다.

shebang은 **스크립트 파일에 고정된 경로**를 쓰는 것이 아니라, **실행 시점의 `PATH`**를 활용한다는 점이 중요하다. 이 덕분에 가상환경, conda, pyenv 등 다양한 환경과 호환된다.

### PATH 검색 순서

`python` 명령을 실행하면, 셸은 `PATH`에 나열된 디렉터리를 **순서대로** 검색해 `python` 실행 파일을 찾는다. **먼저 찾은 것**을 실행한다. venv 활성화는 venv의 `bin`을 `PATH` **앞에** 추가하므로, venv의 `python`이 먼저 찾아진다.

### py launcher의 동작

Windows의 `py` launcher는 `py.ini` 설정 파일과 환경 변수(`PY_PYTHON` 등)를 참고해, 요청된 버전(예: `-3.11`)에 맞는 Python을 찾아 실행한다. `py -0`로 설치된 버전 목록을 볼 수 있고, `py -3.11`처럼 명시적으로 버전을 지정할 수 있다.
