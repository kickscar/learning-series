# 2.5 여러 값의 대입 — 스왑, 언패킹

## 개요

한 줄에서 **여러 이름**에 **여러 값**을 한 번에 붙일 수 있다. `a, b = 1, 2`처럼 좌변의 이름 개수와 우변의 값 개수를 맞추면 된다. 스왑(`a, b = b, a`)과 튜플·리스트 언패킹 맛보기를 통해 이 문법을 익히는 것이 이 단원의 목표다.

---

## 핵심 개념

- **형식**: `이름1, 이름2, ... = 값1, 값2, ...`. 좌변 이름 개수와 우변 값 개수가 **같아야** 한다.
- **실행 순서**: 오른쪽을 먼저 다 계산한 뒤, 왼쪽 이름들에 순서대로 붙인다. 그래서 `a, b = b, a`로 두 값을 서로 바꿀 수 있다(스왑).
- **튜플·리스트 언패킹**: 우변에 튜플이나 리스트를 두고, 좌변에 같은 개수의 이름을 두면 요소가 순서대로 이름에 붙는다. `*rest` 같은 나머지 묶음은 나중 단원에서 다룬다.

---

## 코드 예제

```python
# 여러 이름에 여러 값을 한 번에 붙인다.
a, b = 1, 2
print(a, b)   # 1 2

x, y, z = 10, 20, 30
print(x, y, z)   # 10 20 30
```

```python
# 스왑: 오른쪽을 먼저 계산하므로 a, b 값을 서로 바꿀 수 있다.
a, b = 1, 2
a, b = b, a
print(a, b)   # 2 1
```

```python
# 튜플·리스트 언패킹: 요소 개수와 이름 개수가 일치해야 한다.
t = (100, 200)
p, q = t
print(p, q)   # 100 200

lst = [7, 8, 9]
u, v, w = lst
print(u, v, w)   # 7 8 9
```

---

## 주의사항 (Pitfalls)

- **개수 불일치**: 좌변 이름 개수와 우변 값(또는 요소) 개수가 다르면 `ValueError`가 난다. 예: `a, b = 1, 2, 3` 또는 `a, b = [10]`.
- **스왑 시 순서**: `a, b = b, a`는 맞고, `a = b; b = a`처럼 두 줄로 나누면 두 이름이 같은 값만 가리키게 되어 스왑이 되지 않는다. 한 문장에서 여러 개 대입할 때 오른쪽이 먼저 계산된다는 점을 이용해야 한다.

---

## Deep Insight

여러 개 대입은 "오른쪽을 전부 계산한 뒤, 그 결과를 왼쪽 이름들에 순서대로 붙인다"는 하나의 문이다. 그래서 `a, b = b, a`에서 오른쪽 `b, a`가 먼저 (예: 2, 1)로 평가되고, 그 다음 왼쪽 `a`, `b`에 각각 붙기 때문에 값이 바뀐다. 나중에 `*rest`로 나머지 요소를 한 이름에 묶는 언패킹을 배우면 시퀀스나 반복 가능한 값을 더 유연하게 나눌 수 있다.

---

## 요약

- `이름1, 이름2, ... = 값1, 값2, ...`: 좌변과 우변 개수가 같아야 한다.
- 오른쪽을 먼저 계산하므로 `a, b = b, a`로 두 값을 스왑할 수 있다.
- 튜플·리스트를 우변에 두면 요소가 순서대로 좌변 이름에 붙는다(언패킹). 개수 불일치 시 `ValueError`.
