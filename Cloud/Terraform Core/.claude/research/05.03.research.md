# Research: 05.03 — Gallery: ALB + ASG

> 조사일: 2026-04-29
> 기준: Terraform >= 1.14.0, hashicorp/aws provider ~> 6.0
> Provider 최신 버전: 6.43.0 (2026-04-29 기준)

---

## 버전 요구사항

| 기능 | 최소 TF 버전 | 최소 Provider 버전 | 비고 |
|------|-------------|-------------------|------|

## Provider 6.x 주요 변경사항 (Breaking Changes)

### aws_launch_template

| 변경 | 내용 |
|------|------|
| **삭제** | `elastic_gpu_specifications` (Elastic Graphics EOL 2024-01) |
| **삭제** | `elastic_inference_accelerator` (Elastic Inference EOL 2024-04) |
| **타입 변경** | Nullable Boolean 속성들이 `0`/`1` 대신 `""`/`true`/`false` 필수 |

영향받는 Nullable Boolean 속성:
- `block_device_mappings.ebs.delete_on_termination`
- `block_device_mappings.ebs.encrypted`
- `ebs_optimized`
- `network_interfaces.associate_carrier_ip_address`
- `network_interfaces.associate_public_ip_address`
- `network_interfaces.delete_on_termination`
- `network_interfaces.primary_ipv6`

### aws_autoscaling_group

Provider 6.x에서 Breaking Change **없음**.

### aws_lb_target_group

| 변경 | 내용 |
|------|------|
| **타입 변경** | `preserve_client_ip`가 `true`/`false`/`""` 필수 (숫자 불가) |

### 전체 리소스 공통

Provider 6.x에서 대부분의 리소스에 `region` attribute가 추가되었다. 멀티 리전 관리를 위한 것으로, 단일 리전 실습에서는 영향 없음.

## AWS Provider 속성 — aws_launch_template

| 인수 | 필수 | 타입 | 설명 |
|------|------|------|------|
| `name` | Optional | `string` | Launch Template 이름. 생략 시 Terraform이 자동 생성. `name_prefix`와 충돌 |
| `name_prefix` | Optional | `string` | 고유 이름 접두사. `name`과 충돌 |
| `image_id` | Optional | `string` | AMI ID. SSM Parameter 참조 지원 (`resolve:ssm:parameter-name`) |
| `instance_type` | Optional | `string` | 인스턴스 타입. `instance_requirements`와 충돌 |
| `vpc_security_group_ids` | Optional | `list(string)` | SG ID 목록. `network_interfaces.security_groups`와 충돌 |
| `user_data` | Optional | `string` | **Base64 인코딩된** user data. 수동 인코딩 필요 |
| `update_default_version` | Optional | `bool` | 수정 시 default version 자동 업데이트. `default_version`과 충돌 |
| `default_version` | Optional | `number` | default version 직접 지정. `update_default_version`과 충돌 |

### iam_instance_profile 블록

```hcl
iam_instance_profile {
  arn  = aws_iam_instance_profile.web.arn   # name과 충돌
  # 또는
  name = aws_iam_instance_profile.web.name  # arn과 충돌
}
```

- `arn`과 `name` 중 하나만 지정
- 블록 형태 (문자열 아님)

### tag_specifications 블록

```hcl
tag_specifications {
  resource_type = "instance"    # "instance", "volume", "spot-instances-request" 등
  tags = {
    Name = "tf-core-gallery-instance-web"
  }
}
```

- 여러 `tag_specifications` 블록 선언 가능
- `resource_type`별로 태그 지정

## AWS Provider 속성 — aws_autoscaling_group

| 인수 | 필수 | 타입 | 설명 |
|------|------|------|------|
| `name` | Optional | `string` | ASG 이름. `name_prefix`와 충돌 |
| `name_prefix` | Optional | `string` | 고유 이름 접두사. `name`과 충돌 |
| `min_size` | **Required** | `number` | 최소 인스턴스 수 |
| `max_size` | **Required** | `number` | 최대 인스턴스 수 |
| `desired_capacity` | Optional | `number` | 목표 인스턴스 수 |
| `vpc_zone_identifier` | Optional | `list(string)` | Subnet ID 목록. `availability_zones`와 충돌 |
| `target_group_arns` | Optional | `set(string)` | ALB/NLB Target Group ARN 목록 |
| `health_check_type` | Optional | `string` | `"EC2"` 또는 `"ELB"` |
| `health_check_grace_period` | Optional | `number` | Health check 시작 전 대기 시간 (초, 기본값 300) |

### launch_template 블록

```hcl
launch_template {
  id      = aws_launch_template.web.id    # name과 충돌
  version = "$Latest"                      # "$Latest", "$Default", 또는 버전 번호
}
```

- `id`와 `name` 중 하나만 지정
- `version` 기본값: `"$Default"`
- `version = "$Latest"` 설정 시 instance refresh가 자동 트리거되지 않음에 주의

### tag 블록 (tags 맵 아님)

```hcl
tag {
  key                 = "Name"
  value               = "tf-core-gallery-instance-web"
  propagate_at_launch = true
}

tag {
  key                 = "Project"
  value               = "tf-core"
  propagate_at_launch = true
}
```

- **`tag` 블록** 형태 (단순 `tags` 맵이 아님)
- 여러 태그 = 여러 `tag` 블록
- `propagate_at_launch`: ASG가 생성하는 EC2 인스턴스에 태그 전파 여부
- Provider 6.x에서도 이 구조 유지, deprecated 아님

```hcl

  healthy_threshold   = 3              # 2~10, 기본값 3
```

## ASG + ALB 통합 패턴

### 방법 1: target_group_arns (인라인, 권장)

```hcl
resource "aws_autoscaling_group" "web" {
  # ...
  target_group_arns = [aws_lb_target_group.web.arn]
  health_check_type = "ELB"
}
```

- ASG가 인스턴스 Launch/Terminate 시 Target Group에 자동 등록/해제
- `aws_lb_target_group_attachment` **불필요** — ASG가 자체 관리
- 단일 리소스에서 관계를 표현하므로 코드가 간결

### 방법 2: aws_autoscaling_attachment (별도 리소스)

```hcl
resource "aws_autoscaling_attachment" "web" {
  autoscaling_group_name = aws_autoscaling_group.web.id
  lb_target_group_arn    = aws_lb_target_group.web.arn
}
```

- ASG와 Target Group 연결을 독립 리소스로 관리
- 모듈 간 분리가 필요할 때 유용

### 혼용 금지

> **같은 Target Group을 `target_group_arns`와 `aws_autoscaling_attachment` 양쪽에서 참조하면 충돌이 발생한다.** 둘 중 하나만 사용해야 한다.

### aws_lb_target_group_attachment와의 관계

- `aws_lb_target_group_attachment`는 **개별 인스턴스를 Target Group에 수동 등록**하는 리소스
- ASG 환경에서는 **사용하지 않는다** — ASG가 `target_group_arns`를 통해 자동으로 등록/해제를 관리
- 수동 등록한 인스턴스와 ASG가 관리하는 인스턴스가 충돌할 수 있음

## Launch Template user_data

### Base64 인코딩 필수

`aws_launch_template`의 `user_data`는 **Base64 인코딩된 문자열**을 요구한다. Provider가 자동 인코딩하지 않는다.

### 패턴 1: templatefile + base64encode (권장)

```hcl
resource "aws_launch_template" "web" {
  # ...
  user_data = base64encode(templatefile("${path.module}/userdata.sh.tpl", {
    app_port    = var.app_port
    db_endpoint = var.db_endpoint
  }))
}
```

### 패턴 2: filebase64 (정적 파일)

```hcl
resource "aws_launch_template" "web" {
  # ...
  user_data = filebase64("${path.module}/userdata.sh")
}
```

### aws_instance와의 차이

- `aws_instance`의 `user_data`는 Provider가 **자동 Base64 인코딩**한다
- `aws_launch_template`의 `user_data`는 **수동 인코딩 필수** — 이것이 핵심 차이

## 주의사항

### 1. Launch Template — 삭제된 인수 (Provider 6.x)

- `elastic_gpu_specifications` → 사용 불가 (Elastic Graphics 서비스 종료)
- `elastic_inference_accelerator` → 사용 불가 (Elastic Inference 서비스 종료)

### 2. Launch Template — user_data 인코딩

- `aws_instance`에서 `aws_launch_template`으로 전환 시 반드시 `base64encode()` 래핑 필요
- 이전 Gallery(Ch04)에서 `aws_instance.user_data`를 사용했다면 마이그레이션 시 주의

### 3. ASG tag 블록 — default_tags 미적용

- `aws_autoscaling_group`의 `tag` 블록은 Provider의 `default_tags`가 **자동 적용되지 않는다**
- ASG `tag` 블록에서 필요한 태그를 명시적으로 선언해야 한다
- `propagate_at_launch = true`로 EC2 인스턴스에 전파 가능
- 단, Launch Template의 `tag_specifications`로 인스턴스 태그를 관리하는 것이 더 깔끔할 수 있음

### 4. ASG health_check_type

- ALB 연동 시 `health_check_type = "ELB"` 권장
- `"EC2"` (기본값)는 EC2 상태만 확인 — 앱 레벨 health check 불가
- `"ELB"`로 설정하면 Target Group의 health check 결과를 ASG가 활용

### 5. ASG desired_capacity drift

- `desired_capacity`를 Terraform에서 관리하면, 수동 스케일링이나 Auto Scaling Policy에 의한 변경이 다음 `apply`에서 되돌려짐
- `ignore_changes`로 drift를 허용할 수 있음:

```hcl
lifecycle {
  ignore_changes = [desired_capacity]
}
```

### 6. Launch Template version과 ASG

- `version = "$Latest"`: Launch Template 변경 시 새 인스턴스에 자동 적용, 기존 인스턴스는 그대로
- `version = "$Default"`: `update_default_version = true`와 함께 사용해야 변경 반영
- Instance Refresh 없이는 기존 인스턴스가 교체되지 않음
