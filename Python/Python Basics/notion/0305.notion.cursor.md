# 3.5 기본 연산자

## 개요

**연산자**는 값이나 변수를 묶어 하나의 값으로 만드는 기호다. **산술**(`+`, `-`, `*`, `/`, `//`, `%`, `**`), **비교**(`==`, `!=`, `<`, `>`, `<=`, `>=`), **논리**(`and`, `or`, `not`), **할당**(`=`, `+=` 등)과 **우선순위**를 익혀, 일상적인 연산을 올바른 순서로 쓸 수 있게 하는 것이 이 단원의 목표다.

---

## 핵심 개념

### 산술 연산자

`+`, `-`, `*`는 덧셈·뺄셈·곱셈이다. 나눗셈 `/`는 **항상 실수**를 반환한다. 정수끼리 `10 / 3`을 해도 결과는 `3.333...`처럼 float다. 정수 몫만 필요하면 `//`(floor division), 나머지는 `%`를 쓴다. `10 // 3`은 3, `10 % 3`은 1이다. 거듭제곱은 `**`로, `2 ** 10`은 1024가 된다. 피연산자의 타입에 따라 결과 타입이 정해지며, 정수와 실수를 섞으면 결과는 float가 된다.

```python
print(10 / 3, 10 // 3, 10 % 3)   # 3.33... 3 1
print(2 ** 10)                    # 1024
```

### 비교 연산자

`==`는 같음, `!=`는 다름을 검사한다. 크기 비교는 `<`, `>`, `<=`, `>=`를 쓴다. 비교 결과는 항상 **bool**(`True` 또는 `False`)이다. 문자열·리스트 등 시퀀스도 같은 연산자로 "사전순" 또는 "요소별 비교"가 가능한 타입이 많다. 비교 연산자는 연속해서 쓸 수 있다: `a < b < c`는 "a가 b보다 작고 b가 c보다 작다"를 한 번에 표현한다.

### 논리 연산자

`and`, `or`, `not`은 논리 연산이다. `and`는 "둘 다 참일 때만 참", `or`는 "하나라도 참이면 참", `not`은 "참이면 거짓, 거짓이면 참"이다. **Short-circuit** 동작이 있다. `a and b`에서 a가 거짓이면 b는 평가하지 않고 거짓을 반환하고, `a or b`에서 a가 참이면 b는 평가하지 않고 a를 반환한다. 따라서 "앞이 거짓일 때만 뒤를 실행"하는 패턴을 조건문 없이 쓸 수 있지만, 뒤에 부수 효과가 있으면 평가 순서에 유의해야 한다. `and`/`or`의 반환값은 bool로 변환된 값이 아니라 "마지막으로 평가한 객체"일 수 있다(예: `3 or 0` → `3`).

### 할당 연산자

`=`는 대입이다. `+=`, `-=`, `*=` 등은 "현재 값을 읽고, 연산한 뒤, 그 결과를 다시 그 이름에 붙인다"는 의미다. `a += 1`은 `a = a + 1`과 같은 결과를 낸다. 이 연산자들은 **문(statement)**이므로 표현식 자리(예: 조건문의 조건)에는 쓸 수 없다. 대입과 "같음 비교"를 혼동하지 않도록, 비교할 때는 반드시 `==`를 쓴다.

### 연산자 우선순위

여러 연산자가 한 식에 있으면 **우선순위**에 따라 묶인다. 큰 흐름은 **산술 > 비교 > 논리 > 할당** 순이다. 예를 들어 `1 + 2 < 3 * 4`는 `(1 + 2) < (3 * 4)`로 계산되어 `3 < 12` → `True`가 된다. 논리 연산에서는 `not`이 `and`보다, `and`가 `or`보다 먼저 묶인다. 따라서 `not a and b`는 `(not a) and b`이고, `not (a and b)`가 아니다. 우선순위를 외우기보다는, 헷갈리면 **괄호 `()`**로 의도를 명시하는 것이 안전하다.

```python
print(1 + 2 < 3 * 4)   # True (산술 후 비교)
print((1 + 2) * 3)     # 9
a = 10
a += 3
a *= 2
print(a)               # 26
```

## 실습 예제

### [실습] lab35

이 실습에서는 **산술·비교·논리·할당 연산자와 우선순위**의 개념을 복습한다.

#### ① 산술 — `/` vs `//`·`%`·`**`

`script01.py`

```python
print(10 / 3, 10 // 3, 10 % 3)
print(2 ** 0, 2 ** 10)
```

실행 결과: `3.333... 3 1` / `1 1024`

#### ② 비교 — 결과는 bool

`script02.py`

```python
print(3 == 3, 3 != 2)
print(1 < 2 < 3)
print("apple" < "banana")
```

실행 결과: `True True` / `True` / `True`

#### ③ 논리 — and·or·not, 반환값

`script03.py`

```python
print(0 and 1, 0 or 1)
print(3 and 0, 3 or 0)
print(not 0, not "")
```

실행 결과: `0 1` / `0 3` / `True True`

#### ④ 할당 — `+=`·`*=`

`script04.py`

```python
a = 10
a += 3
a *= 2
print(a)
```

실행 결과: `26`

#### ⑤ 우선순위 — 산술 > 비교 > 논리

`script05.py`

```python
print(2 * 3 + 4, 2 + 3 * 4)
print(2 * (3 + 4))
print(not True or False)
print(not (True or False))
```

실행 결과: `10 14` / `14` / `False` / `False`

#### ⑥ 실습 포인트

- **산술**: `/`는 float, `//`는 몫, `%`는 나머지, `**`는 거듭제곱.
- **비교**: `==`·`!=`·`<`·`>`. 결과는 bool. `a < b < c` 가능.
- **논리**: `and`/`or`는 short-circuit. 반환값이 bool이 아닐 수 있음.
- **할당**: `+=`는 `a = a + 1`과 동일. 문(statement)이라 조건 자리에 불가.
- **우선순위**: 산술 > 비교 > 논리. 헷갈리면 `()` 사용.

## 주의사항 (Pitfalls)

- **`=` vs `==`**: `=`는 대입, `==`는 같음 비교. 조건문에서 "같은지" 검사할 때 `==`를 쓴다. `if x = 1:`은 문법 오류.
- **`/`와 `//`**: `/`는 실수 나눗셈(결과 float). 정수 몫만 필요하면 `//`, 나머지는 `%`.
- **논리값과 숫자**: `and`/`or`는 피연산자를 bool로 해석하지만, 반환값은 마지막으로 평가한 객체일 수 있다(예: `3 or 0` → `3`). 조건문에서는 "참/거짓"으로만 쓰면 된다.
- **연산자 우선순위**: `not a and b`는 `(not a) and b`다. `not (a and b)`가 아니다. 헷갈리면 괄호를 붙인다.

## Deep Insight

연산자는 타입에 따라 의미가 정해진다. `+`는 수에서는 덧셈, 문자열에서는 이어붙이기, list에서는 연결이다. 비교 연산자는 대부분의 타입에서 직관적인 순서를 정의하고, 없으면 `TypeError`가 난다. `and`/`or`의 short-circuit은 "앞만 보고 결과가 정해지면 뒤를 실행하지 않는다"는 의미라, 뒤에 부수 효과가 있을 때 순서에 유의해야 한다. 우선순위 표를 외우기보다는, 애매하면 괄호로 의도를 드러내는 것이 안전하다.

---

## 요약

- **산술**: `+` `-` `*` `/` `//` `%` `**`. `/`는 float, `//`는 몫.
- **비교**: `==` `!=` `<` `>` `<=` `>=`. 결과는 bool. **논리**: `and`, `or`, `not`. **할당**: `=`, `+=` 등.
- **우선순위**: 산술 > 비교 > 논리 > 할당. 헷갈리면 `()`로 순서를 명시한다.
