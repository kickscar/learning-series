# brief/workflow — 세션 복원 브리핑

## 목적

새 세션 시작 시 이전 작업 맥락을 복원하고 현재 상태를 브리핑한다.

## 실행 단계

### Step 1: CLAUDE.md 읽기

- 시리즈 역할, 범위, 언어 규칙, 현재 상태 확인
- 에이전트/스킬/규칙 목록 파악

### Step 2: context_bridge.md 읽기

- 마지막 세션 날짜
- 완료된 작업
- 미완료/진행 중 항목
- 사용자 요청 사항

### Step 3: context_bridge.md 정리

- 완료된 `[핸드오프]` 식별 (draft 파일 존재 여부로 판단)
- 반영 완료된 `[인사이트]` 식별 (해당 draft에 반영되었는지 확인)
- 해당 draft 작성 완료된 `[결정]`, `[톤]` 식별
- 정리 대상 목록을 사용자에게 보고 → 승인 후 삭제

### Step 4: .claude/draft/ 스캔

- 실제 존재하는 draft 파일 목록
- series-hierarchy.md의 기대 구조와 대조
- 누락/불일치 파일 식별

### Step 5: 브리핑 출력

채팅으로 다음 정보를 출력한다:

```
## 세션 복원 브리핑

### 마지막 세션
- 날짜: {date}
- 완료: {completed items}

### 미완료/진행 중
- {pending items}

### 현재 상태
- 전체 챕터: {n}개
- 작성 완료: {n}개 섹션
- 진행 중: {n}개 섹션
- 미시작: {n}개 섹션

### 다음 작업 제안
- {suggested next step}
```

## 규칙

- 채팅 전용 출력 (파일 수정 없음)
- context_bridge.md 미존재 시 "이전 세션 기록 없음" 보고
