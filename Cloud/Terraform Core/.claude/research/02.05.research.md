## 주의사항

### [치명적] var.az_num validation에서 data source 참조 불가

draft의 `variables.tf` lab02에서 다음 코드가 있다:

```hcl
variable "az_num" {
  validation {
    condition     = var.az_num >= 1 && var.az_num <= length(data.aws_availability_zones.available.names)
    error_message = "가용 영역 번호는 1부터 ${length(data.aws_availability_zones.available.names)} 사이여야 한다."
  }
}
```

**이 코드는 동작하지 않는다.** variable validation의 `condition`에서는 **자기 자신(`var.az_num`)만 참조할 수 있다.** 다른 변수, resource, data source, local 등은 참조할 수 없다. Terraform은 variable validation을 plan 생성 이전에 즉시 평가하므로, data source 값이 아직 존재하지 않는다.

**수정 방안:**

1. **validation에서 하드코딩된 범위 사용:**
   ```hcl
   variable "az_num" {
     type    = number
     default = 1

     validation {
       condition     = var.az_num >= 1 && var.az_num <= 4
       error_message = "az_num은 1~4 범위여야 한다."
     }
   }
   ```

2. **locals에서 런타임 검증 (precondition 패턴):**
   리소스의 `lifecycle > precondition`에서 data source 결과를 검증한다. 이 패턴은 Ch03 이후에서 다루므로, Ch02에서는 option 1이 적절하다.

3. **validation 제거하고 locals에서 안전 처리:**
   `local.selected_az`에서 인덱스 범위를 벗어나면 Terraform이 자체 오류를 발생시킨다. 명시적 validation이 없어도 안전하다.

**권장:** option 1 — `var.az_num >= 1 && var.az_num <= 4` 하드코딩. ap-northeast-2 리전의 AZ 수(4개)를 고정값으로 사용한다. 문서에서 "ap-northeast-2 리전은 4개의 AZ를 제공한다"고 명시하면 하드코딩의 이유가 자연스럽다.

### error_message에서 문자열 보간

draft의 `error_message = "가용 영역 번호는 1부터 ${length(...)} 사이여야 한다."` — data source 참조 자체가 불가하므로 이 보간도 동작하지 않는다. validation의 error_message에서 문자열 보간은 Terraform 1.9+에서 지원되지만, 참조 범위는 condition과 동일하게 자기 변수만 가능하다.

### default_tags에서 local/var 참조 — 제약 조건

provider 블록에서 expressions를 사용할 수 있지만, "Terraform이 configuration을 적용하기 전에 알 수 있는 값"만 참조 가능하다. 즉:
- `var.*` (input variable) — 가능
- `local.*` (다른 변수나 정적 값에만 의존하는 경우) — 가능
- resource/data source의 computed attribute — **불가능**

draft의 `default_tags`는 `local.project`(정적 문자열)과 `var.env`(input variable)만 참조하므로 문제없다.

### aws_subnets data source — aws_subnet_ids와의 차이

`aws_subnet_ids`는 deprecated이다 (AWS Provider 3.x). `aws_subnets`가 현행 대체 data source이다. draft에서 `aws_subnets`를 사용하는 것은 올바르다.

### data source plan 시점 확정 — 예외 상황

data source 값이 plan 시점에 항상 확정되는 것은 아니다. data source의 인수가 다른 resource의 computed attribute를 참조하면, 해당 resource가 apply된 후에야 data source가 평가된다. draft의 lab02에서는 모든 data source가 정적 값(default VPC, amazon owner 등)에만 의존하므로 plan 시점 확정이 맞다.

### Common data source 목록

draft에서 언급하는 data source 목록은 모두 현행 유효하다:
- `aws_ami` — 현행
- `aws_vpc` — 현행
- `aws_availability_zones` — 현행
- `aws_subnets` — 현행 (aws_subnet_ids 대체)
- `aws_iam_policy_document` — 현행
- `aws_caller_identity` — 현행 (이론 부분에서만 언급, lab에서 미사용)
- `terraform_remote_state` — 현행 (이론 부분에서만 언급)
