# 08.01 Research — Provider 버전과 lock 파일

> 조사일: 2026-05-01
> 대상: Terraform >= 1.14.0, hashicorp/aws ~> 6.0

---

## 1. required_providers

- source: `[HOSTNAME/]NAMESPACE/TYPE`. hostname 생략 시 `registry.terraform.io` 기본
- source 자체 생략 시 `hashicorp/<LOCAL_NAME>`으로 추정 (0.13 이전 호환)
- version 생략 시 모든 버전 허용 (비권장)

### 버전 제약 문법

| 연산자 | 의미 | 예시 |
|--------|------|------|
| `=` | 정확히 해당 버전 | `= 6.0.0` |
| `!=` | 해당 버전 제외 | `!= 6.1.0` |
| `>`, `>=`, `<`, `<=` | 비교 | `>= 6.0.0, < 7.0.0` |
| `~>` | pessimistic — 가장 오른쪽 컴포넌트만 증가 허용 | `~> 6.0` |

### `~>` 상세

| Constraint | 허용 범위 |
|------------|----------|
| `~> 6.0` | `>= 6.0, < 7.0` (minor 증가 허용) |
| `~> 6.1` | `>= 6.1, < 7.0` (minor 증가 허용, 하한 6.1) |
| `~> 6.1.0` | `>= 6.1.0, < 6.2.0` (patch만 허용) |

- 2-part: minor 변경 허용, major 차단
- 3-part: patch만 허용, minor도 차단
- Pre-release(`1.2.0-beta`)는 `=`로만 매칭

## 2. .terraform.lock.hcl

- 첫 `terraform init` 시 생성
- HCL 형식 (`.tf`가 아닌 이유: configuration이 아님)
- **provider만 추적. module 버전은 추적하지 않음**

### 내용

| 항목 | 설명 |
|------|------|
| version | 선택된 정확한 버전 |
| constraints | 선택 시 고려한 version constraint |
| hashes | 패키지 무결성 체크섬 |

### Hash 형식

| 형식 | 이름 | 설명 |
|------|------|------|
| `zh:` | Zip Hash | registry `.zip` 파일의 SHA256. 모든 플랫폼 해시 포함 |
| `h1:` | Hash Scheme 1 | 패키지 내용물 SHA256. 현재 플랫폼 것만 기본 포함 |

Default: registry 설치 시 `zh:` (전 플랫폼) + `h1:` (현재 플랫폼)

---

## 3. terraform init 동작

| 시나리오 | 동작 |
|----------|------|
| 첫 init (lock 없음) | constraint 내 최신 선택, lock 생성 |
| 이후 init (lock 있음) | lock 버전 사용, 더 새 버전 무시 |
| `init -upgrade` | constraint 내 최신으로 업데이트, lock 재작성 |
| lock 버전이 constraint 밖 | 에러, `-upgrade` 필요 |

---

## 4. VCS 권장사항

- `.terraform.lock.hcl`: **커밋한다** — 팀 동일 버전 보장, code review로 의존성 변경 추적
- `.terraform/`: **커밋하지 않는다** — provider binary 포함, 크기 큼

---

## 5. Provider 설치 소스

| Source | 설명 |
|--------|------|
| Direct (Registry) | 기본. registry에서 직접 다운로드 |
| Filesystem mirror | 로컬 디스크에서 설치 |
| Network mirror | HTTPS 서버에서 조회 |

### Plugin Cache

- `plugin_cache_dir` (CLI config) 또는 `TF_PLUGIN_CACHE_DIR` (환경변수)
- 디렉토리 미리 생성 필요
- 동시성 안전하지 않음
- 오래된 버전 자동 정리 없음

---

## 6. TF 1.14.x 변경사항

provider handling, lock file, init 동작에 대한 변경 **없음**. 이 시리즈에서 다루는 동작은 1.x 전반에 걸쳐 안정적.

---

## 7. 크로스 플랫폼 해시

```bash
terraform providers lock \
  -platform=linux_amd64 \
  -platform=darwin_amd64 \
  -platform=darwin_arm64
```

CI/CD에서 필수. 모든 대상 플랫폼의 해시를 미리 기록.
