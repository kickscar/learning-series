# 09.01 Research — fmt / validate / lint

> 조사일: 2026-05-01
> 대상: Terraform >= 1.14.0, hashicorp/aws ~> 6.0

---

## 1. terraform fmt

- HCL canonical format으로 재작성. 포맷팅만 하고 문법 검증은 안 함
- 대상: `.tf`, `.tfvars`, `.tftest.hcl`, `.tfmock.hcl`, `.tfquery.hcl`(1.14+)
- JSON 변형(`.tf.json`, `.tfvars.json`)은 수정하지 않음
- 기본: 현재 디렉토리만, 파일 덮어쓰기, 변경된 파일 목록 출력

### 주요 플래그

| Flag | 설명 |
|------|------|
| `-recursive` | 하위 디렉토리 포함 |
| `-check` | 수정 안 함. exit 0 = 정상, exit 3 = 포맷 필요 |
| `-diff` | 변경 diff 표시 |
| `-write=false` | 덮어쓰기 안 함 (stdout 출력) |

- 커스터마이즈 옵션 없음 (의도적)
- TF 버전 업그레이드 후 `fmt` 재실행 권장

**Source**: https://developer.hashicorp.com/terraform/cli/commands/fmt

## 2. terraform validate

- 문법, 참조, 타입, provider schema 검사
- API 호출 없음, state 접근 없음, 실제 인프라 검증 없음
- `terraform init` 필수 (provider schema 필요). `-backend=false`로 backend 스킵 가능
- `-json` 출력: version `"1.0"` (TF 1.1+)
- exit 0 = valid, exit 1 = errors

### 잡는 오류

- 누락 variable, 잘못된 타입, 잘못된 참조, 알 수 없는 attribute, 중복 선언, 잘못된 블록 구조

**Source**: https://developer.hashicorp.com/terraform/cli/commands/validate

## 3. tflint

- 플러그인 기반 linting 프레임워크
- 현재 v0.62.0 (2026-04). TF >= 1.0 지원
- 설치: `brew install tflint`, binary, Docker, GitHub Actions

### .tflint.hcl

```hcl
plugin "terraform" {
  enabled = true
  preset  = "recommended"
}

plugin "aws" {
  enabled = true
  version = "0.47.0"
  source  = "github.com/terraform-linters/tflint-ruleset-aws"
}
```

### AWS Ruleset

- 700+ rules. instance_type 유효성, deprecated 리소스, 태그 누락 등
- `--init`으로 플러그인 다운로드
- `--format`: default, json, checkstyle, junit, compact, sarif
- `--recursive`: 하위 디렉토리 포함

**Source**: https://github.com/terraform-linters/tflint, https://github.com/terraform-linters/tflint-ruleset-aws

## 4. 비교

| | fmt | validate | tflint |
|--|-----|----------|--------|
| 목적 | 포맷팅 | 문법+스키마 검증 | 린팅+베스트 프랙티스 |
| 포맷 검사 | ✓ | — | — |
| 문법 검사 | — | ✓ | ✓ (제한적) |
| 참조 검사 | — | ✓ | ✓ |
| 값 검사 | — | — | ✓ (instance_type 유효성 등) |
| init 필요 | — | ✓ | — (자체 `--init`) |
| AWS 자격증명 | — | — | — |
| 파일 수정 | ✓ | — | — |
| CI exit code | 3 (`-check`) | 1 | 2 |

---

## 5. CI/CD 순서

```
fmt -check → validate → tflint → plan
```

1. fmt: 가장 빠름, init 불필요
2. validate: 문법/참조 에러 잡기
3. tflint: provider 규칙 검사
4. plan: 실제 실행 계획 (자격증명 필요)

## 6. TF 1.14.x 변경

- `fmt`이 `.tfquery.hcl` 파일도 처리 (1.14+)
- `validate`에 `-query` 플래그 추가
- fmt/validate 핵심 동작 변경 없음
