# Research: 03.05 — Gallery: Azure VM 기본 배포

> 조사일: 2026-04-05
> 기준: Azure Portal (영문)

---

## 콘솔 경로

### Resource Group 생성
1. Azure Portal 홈
2. 검색창 → "Resource groups"
3. Create
4. Basics 탭: Subscription, Resource group name, Region

### VM 생성 (Custom Data 포함)
1. Azure Portal 홈
2. 검색창 → "Virtual machines"
3. Create → Azure virtual machine
4. Basics 탭: Project details, Instance details, Administrator account, Inbound port rules
5. Networking 탭: Virtual network, Subnet, Public IP (Create new), NIC network security group
6. **Advanced 탭: Custom data** — cloud-init 스크립트 붙여넣기 (plaintext, base64 자동 처리)
7. Review + create

### NSG 인바운드 규칙 추가 (VM 생성 후)
1. Azure Portal → Virtual machines → {vm-name}
2. Networking → Network settings
3. Inbound port rules → Add inbound port rule

### VM 상태 / Public IP 확인
1. Azure Portal → Virtual machines → {vm-name} → Overview

### 리소스 목록 확인
1. Azure Portal → Resource groups → {rg-name} → Overview (Resources 탭)

## 주요 설정 필드

### VM 생성 — Basics 탭
- Virtual machine name
- Region
- Availability options
- Security type
- Image
- Size
- Authentication type
- Username
- SSH public key source / Key pair name
- Public inbound ports / Select inbound ports

### VM 생성 — Networking 탭
- Virtual network (자동 생성 또는 기존 선택)
- Subnet
- Public IP → Create new (Name, SKU, Assignment)
- NIC network security group (None / Basic / Advanced)

### VM 생성 — Advanced 탭
- **Custom data**: cloud-init 스크립트 텍스트 입력
  - `#cloud-config` 헤더로 시작하는 YAML 형식
  - Azure Portal에서 base64 인코딩 자동 처리
  - Ubuntu 24.04 LTS는 cloud-init 기본 지원

### NSG 인바운드 규칙 추가
- Source / Source port ranges
- Destination / Destination port ranges
- Protocol
- Action
- Priority (낮을수록 우선)
- Name

## 조사 메모

- **Custom data vs User data**: Custom data는 첫 부팅 1회만 실행. User data는 VM 재시작 후에도 접근 가능한 메타데이터 엔드포인트용. cloud-init 실행은 Custom data 사용.
- **cloud-init 완료 확인**: `cloud-init status` 명령어로 `done` 상태 확인. 빌드 시간(Maven 의존성 다운로드) 포함 5~10분 소요 예상.
- **Key pair 파일**: VM 생성 시 `key-gallery.pem` 다운로드. 이후 SSH 접속 및 배포 확인에 사용.
- **NSG 자동 생성**: Networking 탭에서 Basic 선택 시 `vm-gallery-nsg` 이름으로 자동 생성. SSH 22 허용 규칙 포함. 8080은 VM 생성 후 별도 추가.
