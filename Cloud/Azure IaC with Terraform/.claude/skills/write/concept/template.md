# 이론/개념 섹션 템플릿

## Structure 1 — 일반 개념

IaC 개요, 인증 개념, 네이밍 규칙, 프로젝트 구조 등 범용 개념에 사용한다.

```
{개요 문단 — 이 섹션에서 다루는 내용을 1~2문장으로}

# {개념명 1}

## 1. {소주제}

### ① {세부 항목}

{설명}

### ② {세부 항목}

{설명}

## 2. {소주제}

{설명 + mermaid 또는 코드 예시}

# {개념명 2}

## 1. {소주제}

{설명}

# 핵심 정리

- {핵심 포인트 1}
- {핵심 포인트 2}
- {핵심 포인트 3~5}

# 참고 자료

- [{문서명}]({URL}) — {출처}
```

# {메커니즘명}

## 1. 동작 원리

{mermaid flowchart로 흐름 시각화}

{2~4줄 설명}

{코드 예시 또는 구조 설명}

## 3. 실무 관점

{실무에서 어떻게 쓰이는지, 주의사항}

## 공통 규칙

- 문서 시작에 챕터/섹션 제목 넣지 않음
- 개요 문단으로 시작
- H1 번호 없음, H2 아라비아 번호, H3 원숫자
- 헤딩에 볼드 금지
- 영문 기술 용어 첫 등장 시 한국어 병기
- `hcl`, `bash`, `text` 등 코드 블록 언어 필수
