# 3.2 sequence: list, tuple, range

## 개요

**시퀀스(sequence)**는 순서가 있는 컬렉션이다. 인덱스로 한 요소에 접근(`a[i]`), 구간을 잘라 쓰기(`a[i:j]`), 길이(`len(a)`), 그리고 `for x in a`로 순회할 수 있다. **list**(가변), **tuple**(불변), **range**(정수 구간)의 리터럴·생성·수정과, `*` 언패킹 맛보기를 익히는 것이 이 단원의 목표다.

---

## 핵심 개념

### list (리스트)

list는 **순서가 있고 가변인** 시퀀스다. 리터럴은 대괄호로 `[1, 2, 3]`처럼 쓴다. 인덱스로 요소에 접근(`a[i]`)하거나, `a[i] = x`로 그 자리의 값을 바꿀 수 있다. 끝에 추가할 때는 `append(x)`, 끝 요소를 꺼낼 때는 `pop()`을 쓴다. 슬라이싱 `a[i:j]`는 "i 이상 j 미만" 구간을 **새 리스트**로 만들어 반환하므로, 원본을 바꾸지 않고 일부만 복사해 쓸 수 있다. 길이는 `len(a)`로 구한다.

```python
lst = [10, 20, 30]
lst[1] = 99
lst.append(40)
print(lst)   # [10, 99, 30, 40]
print(lst[1:3])   # [99, 30] — 새 리스트, 원본 변경 없음
```

### tuple (튜플)

tuple은 **순서가 있고 불변인** 시퀀스다. 리터럴은 `(1, 2, 3)` 또는 괄호 없이 `1, 2, 3`으로 쓴다. 한 번 만들면 요소를 추가·삭제·대입할 수 없다. 인덱싱과 슬라이싱은 가능한데, 슬라이싱 결과는 새 tuple이 된다. 함수에서 여러 값을 한 번에 반환할 때(`return a, b`) 실제로 반환되는 것이 tuple이라서, 여러 반환값을 다룰 때 자주 쓰인다. 요소가 하나뿐일 때는 `(1,)`처럼 쉼표를 넣어야 tuple로 인식된다.

### range (범위)

range는 **정수 구간**을 나타내는 불변 시퀀스다. `range(n)`은 0 이상 n 미만, `range(start, stop)`은 start 이상 stop 미만, `range(start, stop, step)`은 step만큼 건너뛰는 구간이다. range는 0, 1, 2, …를 미리 전부 만들어 두는 것이 아니라, 인덱싱이나 순회할 때 그때그때 값만 만든다. 그래서 `range(1000000)`처럼 큰 구간도 메모리를 거의 쓰지 않는다. 실제 리스트가 필요하면 `list(range(n))`으로 변환한다.

### 인덱싱과 슬라이싱

시퀀스의 i번째 요소는 `a[i]`로 접근한다. 인덱스는 0부터 시작하고, `a[-1]`처럼 음수는 "끝에서부터"를 의미한다(예: -1은 마지막). `a[i:j]`는 인덱스 i 이상 j **미만** 구간을 잘라 **새 시퀀스**로 만든다. i, j를 생략하면 "처음"/"끝"으로 해석된다. `a[i:j:k]`는 step k로 건너뛰며 자른다. 슬라이스는 범위가 비어 있으면 빈 시퀀스를 돌려주지만, 인덱스가 범위를 벗어나면 `IndexError`가 난다.

```python
a = [10, 20, 30, 40]
print(a[0], a[-1])    # 10 40
print(a[1:3])         # [20, 30]
print(a[::2])         # [10, 30]
first, *rest = [1, 2, 3, 4]
print(first, rest)    # 1 [2, 3, 4]
```

### 언패킹 맛보기

시퀀스의 앞 몇 개를 이름에 붙이고, **나머지 전체**를 한 이름에 묶을 수 있다. `first, *rest = [1, 2, 3, 4]`라고 쓰면 `first`는 1, `rest`는 `[2, 3, 4]`가 된다. `*rest`는 "나머지 요소들을 리스트로 묶는다"는 의미다. 좌변 이름 개수와 우변 요소 개수가 맞지 않아도, `*`가 있는 이름이 나머지를 받으므로 유연하게 쓸 수 있다.

## 실습 예제

### [실습] lab32

이 실습에서는 **list, tuple, range와 인덱싱·슬라이싱·언패킹**의 개념을 복습한다.

#### ① list — 수정·append·pop

`script01.py`

```python
lst = [10, 20, 30, 40, 50]
lst[2] = 99
print(lst)
lst.append(60)
lst.pop()
print(lst, len(lst))
```

실행 결과: `[10, 20, 99, 40, 50]` / `[10, 20, 99, 40, 50] 5`

#### ② tuple — 인덱싱·슬라이싱·불변

`script02.py`

```python
t = (100, 200, 300)
print(t[1], t[-1])
print(t[1:])
# t[0] = 0   # TypeError
```

실행 결과: `200 300` / `(200, 300)`

#### ③ range — 구간·list 변환

`script03.py`

```python
r = range(1, 11, 2)
print(list(r))
print(r[0], r[-1])
```

실행 결과: `[1, 3, 5, 7, 9]` / `1 9`

#### ④ 인덱싱·슬라이싱 — 구간·step

`script04.py`

```python
a = [10, 20, 30, 40, 50]
print(a[0], a[-1])
print(a[1:4])
print(a[::2])
```

실행 결과: `10 50` / `[20, 30, 40]` / `[10, 30, 50]`

#### ⑤ 언패킹 — `*rest`

`script05.py`

```python
first, *rest = [10, 20, 30, 40, 50]
print(first, rest)
a, b, *mid, z = [1, 2, 3, 4, 5]
print(a, b, mid, z)
```

실행 결과: `10 [20, 30, 40, 50]` / `1 2 [3, 4] 5`

#### ⑥ 실습 포인트

- **list**: `a[i] = x`, `append`, `pop`. 슬라이스는 새 리스트.
- **tuple**: 인덱싱·슬라이싱 가능, 요소 대입 불가.
- **range**: `list(range(...))`로 리스트화. 인덱싱 가능.
- **슬라이스**: `[i:j]`는 i 이상 j 미만. `[::k]`는 step k.
- **언패킹**: `*rest`는 나머지를 리스트로 묶음.

## 주의사항 (Pitfalls)

- **tuple 요소 변경**: tuple 자체는 불변이지만, 그 안에 list 같은 가변 객체가 있으면 그 객체의 내용은 바꿀 수 있다. tuple이 "바꿀 수 없는 참조의 나열"이라고 보면 된다.
- **range는 리스트가 아님**: `range(5)`는 `[0,1,2,3,4]`를 바로 만든 게 아니라, 필요할 때 값만 만든다. 리스트가 필요하면 `list(range(5))`.
- **슬라이스 vs 인덱스**: `a[i]`는 하나의 요소(정수 등), `a[i:j]`는 시퀀스(리스트/튜플). 범위를 벗어나도 슬라이스는 빈 시퀀스를 돌려주지만, 인덱스는 `IndexError`가 난다.
- **빈 tuple**: `()` 또는 `tuple()`. 요소가 하나면 `(1,)`처럼 쉼표 필요.

---

## Deep Insight

시퀀스는 "순서가 있는" 것이 공통점이고, **가변(list)**과 **불변(tuple, range)**이 다르다. list는 in-place 수정이 가능해 스택·큐처럼 쓰고, tuple은 수정이 불가해 키·함수 반환값처럼 고정된 묶음을 표현할 때 적합하다. range는 반복문에서 `for i in range(n):`처럼 쓰일 때 메모리를 아끼고, 슬라이싱·인덱싱 규칙(`a[-1]`, `a[:]`)은 나중에 문자열·바이트 등 다른 시퀀스에서도 동일하게 적용된다.

---

## 요약

- **list**: 가변 시퀀스 `[ ... ]`. 인덱싱·슬라이싱·`len`, `append` 등으로 생성·수정.
- **tuple**: 불변 시퀀스 `( ... )`. 요소 변경 불가. 인덱싱·슬라이싱·`len`.
- **range**: 정수 구간 `range(start, stop, step)`. 리스트가 아니라 이터레이션·인덱싱 시 값 생성. `a, *rest = it`로 언패킹 가능.
