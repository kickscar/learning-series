## Subject Definition

### 5.1 이름과 객체

- **개요**: "객체"란 값이 살아 있는 메모리 상의 대상. 이름은 객체를 가리킨다(참조). 대입은 "상자에 넣기"가 아니라 "이름표를 객체에 붙이기". 객체의 개념을 명시적으로 다룬다.
- **목표**: "상자" 대신 "이름표→객체" 모델을 내재화한다.
- **내용 요약**: `a = [1,2]; b = a`에서 `a`와 `b`가 같은 리스트 객체를 가리킴. 대입은 바인딩.

### 5.2 객체로서의 타입

- **개요**: `type`도 객체. `type(obj)`는 그 객체의 클래스(타입). `isinstance` vs `type(x)==T`. 상속 고려.
- **목표**: 타입 검사는 `isinstance` 권장. `type(x) is C`는 서브클래스 배제.
- **내용 요약**: `isinstance(x, (A, B))`. `typing`·`abc`와의 연결.

### 5.3 불변/가변

- **개요**: `id()` 관찰로 "객체가 바뀌었는가" 확인. `hash()`, hashable과 불변성. tuple of mutable caveat.
- **목표**: immutable vs mutable, dict 키·set 멤버가 hashable이어야 하는 이유.
- **내용 요약**: `x += 1`에서 int는 새 객체, list는 같은 객체 in-place. `tuple` 안에 `list`면 해시 불가.

### 5.4 `id`, `is`, `==`

- **개요**: `id(obj)`는 객체 식별자. `a is b`는 `id(a)==id(b)`. `==`는 `__eq__`에 의한 동등성.
- **목표**: `is None`만 `is` 사용, 나머지는 `==`. `id()`의 의미와 한계.
- **내용 요약**: `x is None`, `x is not None` 권장. `is`로 리터럴·작은 정수 비교는 구현 의존 비권장.

### 5.5 정수 캐시 (-5~256)

- **개요**: CPython이 -5~256 정수를 인터닝. 이 범위 리터럴은 같은 `id`를 가질 수 있음.
- **목표**: "왜 `100 is 100`이 True일 수 있나" 설명. 구현 의존·문서화되지 않은 동작 강조.
- **내용 요약**: `id(100) == id(100)` 가능. 범위 밖은 `is`로 비교하지 말 것.

### 5.6 별칭과 공유

- **개요**: `a = b`로 같은 객체를 가리키는 별칭. 함수에 list 등을 넘기면 참조가 전달되어 함수 내 수정이 호출자에 반영됨.
- **목표**: 함수·루프에서 부작용을 피하는 법. 객체가 그래프처럼 공유될 수 있음을 직관화.
- **내용 요약**: `def f(l): l.append(1)`과 호출자 리스트 변경. 루프에서 `obj = {}` 빼먹는 버그.

### 5.7 얕은/깊은 복사

- **개요**: `copy.copy`(얕은)는 최상위만 복사, 중첩 가변은 공유. `copy.deepcopy`는 재귀 복사. `list(l)`, `dict(d)`, `[:]`는 얕은 복사.
- **목표**: 언제 얕은/깊은이 필요한지 구분. `deepcopy` 순환 참조 이슈.
- **내용 요약**: `l2 = list(l1)` vs `l2 = l1`. `d2 = d1.copy()`. `copy.deepcopy`의 `memo`.

### 5.8 `__slots__`, 크기 (Deep)

- **개요**: `__slots__`로 인스턴스 `__dict__` 제거해 메모리 절약. `sys.getsizeof`로 객체 크기 추정(직접 바이트만, 내부 참조 제외).
- **목표**: 객체 내부 구조 직관. `__dict__`가 없으면 동적 속성 추가 불가. Ch10과 연계.
- **내용 요약**: `class C: __slots__ = ('a','b')`. `getsizeof` 함정.

### 5.9 연산자와 특수메서드

- **개요**: `+` → `__add__`, `==` → `__eq__`, `@` → `__matmul__`. `__radd__`, `__iadd__`. `or`/`and` short-circuit.
- **목표**: 연산자가 내부적으로 어떤 메서드를 부르는지 파악. Ch10에서 자신만의 타입에 연산자 정의로 연결.
- **내용 요약**: `is`는 메서드 없음. `and`/`or`는 bool로 변환하지 않고 객체를 그대로 반환할 수 있음.

### 5.10 `|` for types (3.10+)

- **개요**: `int | None`으로 union 타입 표현. `Union[int, str]` 대체. `Optional[T]` = `T | None`.
- **목표**: 3.10+ 타입 구문. Ch6 Type Hints, Ch10 `Protocol`과 연결.
- **내용 요약**: `def f(x: int | None) -> str | None`. `list[int]`, `dict[str, int]` (3.9+).

## Modern Key Concepts

- **`x is None`**, **`x is not None`**: 타입 체커·PEP 8 친화.
- **`int | None`**, **`list[int]`**: 3.10+ 내장 제네릭·union.
- **가변 기본인자 해법**: Ch6 인자 모델에서 `None`+`if l is None: l = []` 패턴.
- **`@dataclass(slots=True)`** (3.10+): Ch10에서 상세화.
