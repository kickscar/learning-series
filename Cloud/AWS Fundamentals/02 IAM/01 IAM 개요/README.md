# 02 IAM / 01 IAM 개요

IAM(Identity and Access Management)의 목적과 인증(Authentication)/인가(Authorization) 흐름을 정리한다. User, Group, Policy, Role이 어떻게 연결되는지 큰 그림을 잡고, Root User 대신 IAM User로 작업해야 하는 이유를 확정한다.

## What you will learn

- IAM이 AWS 리소스 접근 제어에서 맡는 역할
- 인증(Authentication)과 인가(Authorization)의 차이와 흐름
- User/Group/Policy/Role의 연결 관계와 역할
- Root User 최소 사용, IAM User 기반 작업, MFA의 중요성
