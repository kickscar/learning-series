# Research: 02.02 — provider 블록

> 조사일: 2026-04-15
> 기준: Terraform 1.14.x, hashicorp/aws provider >= 6.0

## 검증 결과

| 항목 | 상태 | 비고 |
|------|------|------|
| Terraform Core와 Provider 분리 구조 | ✓ 정확 | Core는 언어/계획, Provider는 API 호출. 분리된 바이너리. 공식 문서 일치 |
| terraform 블록 + required_providers 구조 | ✓ 정확 | source, version 인수 유효 |
| source 생략 시 "hashicorp/{type}" 기본값 | ✓ 정확 | 공식 문서 Provider Requirements 페이지에서 확인 |
| provider 블록 인수: region, profile, access_key/secret_key | ✓ 정확 | AWS Provider 문서에서 확인. access_key/secret_key는 여전히 지원되나 비권장 |
| default_tags에서 local.* 참조 | ✓ 정확 | 허용됨. 단, local 값이 apply 전에 known이어야 함. draft의 `local.project = "tf-core-lab02"` 같은 리터럴 기반 local은 문제없음 |
| tags vs tags_all 동작 | ✓ 정확 | tags = 리소스 직접 선언 태그, tags_all = default_tags + tags 합산. plan에서 둘 다 표시. 리소스 tags가 같은 키일 경우 리소스 값이 우선 |
| `~> 6.0` 의미 | ✓ 정확 | ">= 6.0, < 7.0" — 두 부분(MAJOR.MINOR) 지정 시 rightmost(MINOR)가 증가 허용. 공식 문서 예시: `~> 1.1`은 1.2, 1.10 허용, 2.0 불허 |
| `~> 6.1` 의미 | ✗ 수정필요 | draft에 "6.1.x 계열"로 기술. **실제로는 ">= 6.1, < 7.0"이다.** 두 부분 버전에서 rightmost가 MINOR이므로 6.2, 6.3, ... 모두 허용. "6.1.x 계열"이 되려면 `~> 6.1.0` (세 부분)이어야 함 |
| `= 6.1.0` 의미 | ✓ 정확 | 정확히 해당 버전만 |
| `>= 6.0` 의미 | ✓ 정확 | 6.0 이상 |
| `>= 6.0, < 7.0` = `~> 6.0` 동일 효과 | ✓ 정확 | 공식 문서에서 확인 |
| required_version 동작 | ✓ 정확 | TF CLI 버전 제약. 불일치 시 init/plan 단계에서 차단 |
| terraform init 흐름 설명 | ✓ 정확 | Registry 조회 → .terraform/ 다운로드 → .terraform.lock.hcl 기록 |
| .terraform.lock.hcl 구조 | ✓ 정확 | version, constraints, hashes 기록. VCS에 커밋 권장 |
| terraform init -upgrade 동작 | ✓ 정확 | lock 파일의 버전을 constraint 범위 내 최신으로 갱신 |
| lock 파일 존재 시 constraint 변경 → init 오류 | ✓ 정확 | locked version과 새 constraint 충돌 시 오류 발생. -upgrade로 해결 |
| Provider alias 문법 | ✓ 정확 | alias가 없는 provider가 기본값. 리소스에서 `provider = aws.us_east` 형식 |

## 주의사항

- **`~> 6.1` 설명 수정 필수**: draft 라인 114에서 `~> 6.1`을 "6.1.x 계열"로 설명하는데 이는 부정확하다. 공식 문서 기준: 두 부분 버전(`~> MAJOR.MINOR`)에서는 MINOR가 rightmost이므로 MINOR이 증가 허용된다. 따라서 `~> 6.1` = `>= 6.1, < 7.0`. "6.1.x 계열"로 제한하려면 세 부분(`~> 6.1.0`)을 사용해야 한다. 수정안:

  | 표현식 | 의미 |
  |--------|------|
  | `~> 6.0` | >= 6.0, < 7.0 (마이너 버전 증가 허용) |
  | `~> 6.1` | >= 6.1, < 7.0 (마이너 버전 증가 허용) |
  | `~> 6.1.0` | >= 6.1.0, < 6.2.0 (패치 버전만 증가 허용) |

- **default_tags의 제약 조건 명시 권장**: `default_tags`에서 `local.*`을 참조할 수 있지만, 해당 local이 apply 전에 known인 값이어야 한다. computed resource attribute를 참조하는 local은 사용 불가. draft의 `local.project = "tf-core-lab02"` 같은 리터럴 값은 문제없으므로 현재 lab 코드는 정확하지만, 이론 섹션에서 이 제약을 한 줄 언급하면 좋다.
