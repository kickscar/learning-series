# Research: 03.02 — VM 생성 및 기본 구성

> 조사일: 2026-04-05
> 기준: Azure Portal (영문), learn.microsoft.com 공식 문서

---

## 콘솔 경로

### Resource Group 생성

1. Azure Portal 상단 검색창에 **"Resource groups"** 입력
2. Services 아래 **Resource groups** 선택
3. 상단 **+ Create** 버튼 클릭
4. Basics 탭에서 설정:
   - **Subscription**: 사용할 구독 선택
   - **Resource group**: 이름 입력 (예: `myResourceGroup`)
   - **Region**: 배포 리전 선택
5. **Review + create** → **Create** 클릭

> VM 생성 흐름 내에서 Resource Group을 인라인으로 신규 생성하는 방법도 있음:
> Basics 탭 > Project details > Resource group 필드 > **"Create new"** 링크 클릭

---

### VM 생성

1. Azure Portal 상단 검색창에 **"Virtual machines"** 입력
2. Services 아래 **Virtual machines** 선택
3. **Virtual machines** 목록 페이지 상단 **+ Create** 클릭 → 드롭다운에서 **Azure virtual machine** 선택
4. **Create a virtual machine** 페이지가 열림. 탭 순서:
   - **Basics** → **Disks** → **Networking** → **Management** → **Monitoring** → **Advanced** → **Tags** → **Review + create**

**[Basics 탭]**

5. **Project details** 섹션:
   - Subscription: 구독 선택
   - Resource group: 기존 선택 또는 **Create new** 로 신규 생성
6. **Instance details** 섹션:
   - Virtual machine name: VM 이름 입력
   - Region: 배포 리전 선택
   - Availability options: **No infrastructure redundancy required** 선택 (학습/실습 환경)
   - Security type: **Standard**
   - Image: **Ubuntu Server 24.04 LTS - x64 Gen2** 선택 (또는 22.04 LTS)
   - VM architecture: x64
   - Size: **See all sizes** 클릭 → B2s (Standard_B2s) 검색/선택
7. **Administrator account** 섹션:
   - Authentication type: **SSH public key**
   - Username: `azureuser` (기본값)
   - SSH public key source: **Generate new key pair** → Key pair name 입력
8. **Inbound port rules** 섹션:
   - Public inbound ports: **Allow selected ports**
   - Select inbound ports: **SSH (22)** 체크

**[Networking 탭]**

9. **Network interface** 섹션:
   - Virtual network: 자동 생성 또는 기존 VNet 선택
   - Subnet: 자동 생성 또는 기존 서브넷 선택
   - Public IP: **(new)** 선택 → 옆 **Create new** 클릭하여 SKU/Assignment 설정
     - SKU: **Standard** (Basic은 2025-09-30 retired)
     - Assignment: **Static**
   - NIC network security group: **Basic** 또는 **Advanced**
     - Basic 선택 시 → Configure network security group 드롭다운 표시
     - Advanced 선택 시 → 기존 NSG 연결 또는 **Create new** 로 신규 NSG 생성
10. **Load balancing** 섹션: None (기본값 유지)

**[Management 탭]** (참고용)

- **Boot diagnostics**: Enable with managed storage account (기본값으로 활성화됨)

**[Review + create]**

11. 상단 배너에 **Validation passed** 확인
12. 리소스 요약 및 예상 비용 확인 후 **Create** 클릭
13. **Generate new key pair** 팝업 → **Download private key and create resource** 클릭 (`.pem` 파일 다운로드)
14. 배포 완료 후 **Go to resource** 클릭

---

### VM 상태 확인 / NSG 확인

**VM 상태 확인:**
1. **Virtual machines** 목록에서 해당 VM 클릭
2. **Overview** 페이지:
   - 상단 Status 표시 (Running / Stopped / Deallocated)
   - Essentials 섹션: Public IP address, Private IP address, OS, Size, Location 등 확인

**NSG 확인 (VM에서 직접):**
1. VM Overview 페이지 왼쪽 사이드바
2. **Networking** 섹션 → **Network settings** 클릭
3. **Network Interface** 카드 또는 **Application security groups** 탭에서 연결된 NSG 확인
4. NSG 이름 클릭 → NSG 리소스 페이지로 이동
5. **Inbound security rules** 탭에서 현재 규칙 목록 확인
6. **+ Add** 클릭으로 신규 인바운드 규칙 추가 가능

**NSG 직접 접근 경로:**
1. Portal 검색창 → **"Network security groups"** 입력
2. 해당 NSG 선택 → **Inbound security rules** 또는 **Outbound security rules** 탭

## 주요 설정 필드

### Basics 탭

**Project details 섹션**
- `Subscription`: Azure 구독 선택. 여러 구독이 있을 경우 드롭다운으로 선택
- `Resource group`: VM을 배치할 리소스 그룹. Create new 로 인라인 생성 가능

**Instance details 섹션**
- `Virtual machine name`: VM 식별자. 영숫자, 하이픈 조합, 최대 15자 (Windows) / 64자 (Linux)
- `Region`: VM이 배포될 Azure 리전. 선택 리전에 따라 사용 가능한 Size가 달라짐
- `Availability options`: No infrastructure redundancy required / Availability zones / Availability set 중 선택
- `Security type`: Standard / Trusted launch virtual machines / Confidential virtual machines 중 선택
- `Image`: OS 이미지 선택. "See all images" 링크로 Marketplace 전체 탐색 가능
- `VM architecture`: x64 / Arm64 (이미지에 따라 자동 설정)
- `Size`: vCPU, RAM, 월 예상 비용 표시. "See all sizes" 링크로 전체 Size 검색 가능

**Administrator account 섹션**
- `Authentication type`: SSH public key (권장) / Password
- `Username`: VM 관리자 계정 이름 (기본값: azureuser)
- `SSH public key source`: Generate new key pair / Use existing key stored in Azure / Use existing public key
- `Key pair name`: 새 키 페어 이름 지정 (Generate new key pair 선택 시)

**Inbound port rules 섹션**
- `Public inbound ports`: None / Allow selected ports 중 선택
- `Select inbound ports`: SSH (22), HTTP (80), HTTPS (443), RDP (3389) 중 복수 선택 가능
  - 이 설정은 VM 생성 시 NSG를 자동 생성하고 해당 포트의 인바운드 규칙을 추가함

---

### Networking 탭

**Network interface 섹션**
- `Virtual network`: VM이 연결될 VNet. Create new 로 인라인 생성 가능 (기본: 이름-vnet 형태로 자동 생성)
- `Subnet`: VNet 내 서브넷 선택. 기본: default (10.0.0.0/24)
- `Public IP`: (new) 상태로 자동 생성되거나 None 선택. Create new 클릭 시:
  - **Name**: Public IP 리소스 이름
  - **SKU**: Standard (Basic은 retired)
  - **Tier**: Regional / Global
  - **Assignment**: Static / Dynamic (Standard SKU는 Static만 지원)
  - **Routing preference**: Microsoft network / Internet
- `NIC network security group`:
  - **None**: NSG 미생성
  - **Basic**: 인라인으로 간단한 NSG 규칙 지정 (Configure network security group 드롭다운 표시)
  - **Advanced**: 기존 NSG 연결 또는 Create new 로 신규 NSG 생성
- `Delete NIC when VM is deleted`: VM 삭제 시 NIC 함께 삭제 여부 (기본값: 체크됨)
- `Enable accelerated networking`: 고성능 네트워킹 활성화 (Size에 따라 지원 여부 다름)

**Load balancing 섹션**
- `Load balancing options`: None / Azure load balancer / Application gateway 중 선택

---

### Management 탭 (Boot diagnostics 관련)

- `Boot diagnostics`: Enable with managed storage account (기본 활성화) / Enable with custom storage account / Disable
  - 관리형(Managed) storage account 사용 권장: 별도 Storage Account 생성 불필요
  - VM 생성 후 확인 경로: VM 왼쪽 사이드바 > **Help** 섹션 > **Boot diagnostics**

## NSG 규칙 구조 (인바운드 규칙 추가 시 필드)

| 필드 | 설명 |
|------|------|
| Source | Any / IP Addresses / Service Tag / Application security group |
| Source port ranges | * (Any) 또는 특정 포트 번호 |
| Destination | Any / IP Addresses / Service Tag 등 |
| Service | Custom / SSH / HTTP / HTTPS 등 프리셋 |
| Destination port ranges | 22, 80, 443 등 |
| Protocol | Any / TCP / UDP / ICMP |
| Action | Allow / Deny |
| Priority | 100 ~ 4096 (낮을수록 먼저 처리) |
| Name | 규칙 이름 |

**NSG 기본 인바운드 규칙 (자동 생성, 삭제 불가):**

| 이름 | 우선순위 | 동작 |
|------|----------|------|
| AllowVNetInBound | 65000 | VNet 내부 트래픽 허용 |
| AllowAzureLoadBalancerInBound | 65001 | Azure Load Balancer 허용 |
| DenyAllInbound | 65500 | 나머지 모든 인바운드 차단 |

## VM과 NSG 연결 구조

```
Public Internet
       │
       ▼
  [Public IP]  ─── Standard SKU, Static
       │
       ▼
  [NIC (Network Interface Card)]
       │
       ├── NSG (NIC 레벨에 연결) ← VM 생성 시 "NIC NSG" 옵션
       │         └── Inbound rules: SSH (22) Allow 등
       │
       ▼
  [VM Instance]
       │
  (VNet / Subnet 내에 위치)
       │
  서브넷 레벨 NSG도 별도 연결 가능
```

- NSG는 NIC 레벨 또는 Subnet 레벨에 연결 가능
- VM 생성 시 Networking 탭에서 설정하면 NIC에 NSG가 연결됨
- Standard SKU Public IP는 기본적으로 인바운드 트래픽 차단 → NSG로 명시적 허용 필수

## 조사 메모

### B2s 선택 관련 주의사항

- `Standard_B2s`는 B-series v1으로 **previous-gen (Capacity limited)** 상태
- Retirement 공식 발표됨 (은퇴일: 2028-11-15)
- Azure Portal에서 Size 선택 시 여전히 표시되나, 일부 리전에서 용량 제한으로 생성 실패 가능
- 학습/실습 목적으로는 사용 가능하나, **문서 내 "추후 Bsv2 계열로 마이그레이션 권장" 언급 필요**
- 대안: `Standard_B2s_v2` (Bsv2-series) — 동일한 가격대, 더 나은 성능

### Public IP SKU 관련 주의사항

- Basic SKU는 **2025년 9월 30일 완전 은퇴** (일반 Public IP 기준)
- 현재 Azure Portal VM 생성 Networking 탭에서 Public IP 생성 시 **Standard SKU만 표시**
- Standard SKU는 **Static 할당만** 지원 (Dynamic 지원 안 함)
- 과거 Basic SKU는 Static/Dynamic 모두 지원했으나 현재 해당 없음
- Standard SKU Public IP는 "Secure by default" 모델 — 기본적으로 인바운드 차단. **NSG 없이는 SSH 접속 불가**

### VM 생성 탭 흐름 (현재 Portal UI)

- Basics → Disks → **Networking** → Management → Monitoring → Advanced → Tags → Review + create
- 이전 UI에서는 "Management" 탭 안에 부팅 진단 설정이 있었으나 현재도 동일
- Networking 탭에서 NSG를 "Basic"으로 설정하면 Basics 탭에서 선택한 inbound port rules가 그대로 반영됨
- "Advanced" NSG 선택 시 기존 NSG 연결 또는 신규 NSG 생성 선택 가능

### VM 상태 확인 경로

- Overview 페이지 **Status** 필드: Running / Stopped (deallocated) / Stopped / Starting 등
- 왼쪽 사이드바 **Help** > **Boot diagnostics**: 부팅 화면 스크린샷 및 시리얼 로그 확인
- 왼쪽 사이드바 **Networking** > **Network settings**: 연결된 NIC 및 NSG, 유효 보안 규칙 확인

### NSG Flow Logs 관련 참고

- NSG flow logs는 2027-09-30 retired 예정
- 대체: Virtual Network flow logs (vNet flow logs)
- 문서 내 flow logs 설명 포함 시 이 전환 사항 언급 권장

### 한국어 Portal 메뉴명

- Azure Portal 한국어 설정 시 대부분 영문 용어의 직역 사용
- "만들기(Create)", "검토 + 만들기(Review + create)", "개요(Overview)" 등
- 기술 용어(NSG, NIC, VNet, SKU 등)는 한글 Portal에서도 영문 약어 그대로 사용되는 경우 많음
- 실제 한국어 Portal UI는 사용자 설정에 따라 다를 수 있어 영문 기준으로 설명 후 괄호 안에 한글 병기 권장
