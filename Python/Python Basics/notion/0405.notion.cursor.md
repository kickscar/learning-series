# 4.5 반복문의 else 절

## 개요

`for`와 `while`에 붙는 **`else`** 절은 **`break`로 빠져나가지 않고** 루프가 끝났을 때만 실행된다. "조건이 거짓이어서"가 아니라, "**break를 타지 않고** 정상 종료했을 때"라는 점이 핵심이다. 도움말과 레거시 코드에서 자주 나오는 패턴이므로, "검색 실패" 등 루프-else의 의도를 해석하고 필요 시 활용할 수 있게 하는 것이 이 단원의 목표다.

---

## 핵심 개념

### else가 실행되는 조건

**`for`** 루프의 `else`는 **이터러블이 모두 소진되어** 루프가 끝났을 때 실행된다. 단, **`break`로 한 번이라도 빠져나갔다면** `else`는 실행되지 않는다. **`while`** 루프의 `else`는 **조건이 거짓이 되어** 루프를 빠져나왔을 때 실행된다. 마찬가지로 **`break`로 나갔다면** `else`는 실행되지 않는다. 즉, "루프를 **정상적으로** 끝까지 돌았을 때만" `else`가 실행된다. "정상 종료"란 `for`는 다음 항목이 없어서, `while`은 조건이 False가 되어서 나온 경우다. **`continue`**는 `else` 실행 여부에 영향을 주지 않는다. `continue`로 건너뛰기만 할 뿐, 루프를 "탈출"하지는 않기 때문이다.

```python
for x in [1, 2, 3]:
    if x == 2:
        break
else:
    print("else ran")
print("done")
```

실행 결과: `done` (break로 나갔으므로 else 미실행)

```python
for x in [1, 2, 3]:
    pass
else:
    print("else ran")
```

실행 결과: `else ran`

### "검색 실패" 패턴

"시퀀스에서 조건을 만족하는 항목을 찾고, **없으면** 별도 처리"를 할 때 루프-else가 잘 맞는다. 루프 안에서 조건을 만족하면 **`break`**로 빠져나가고, **한 번도 break하지 않았다**는 것은 "못 찾았다"는 뜻이므로 `else`에서 "not found"나 예외를 발생시키면 된다. `for item in items: if item == target: break` 다음 `else: raise NotFound` 같은 형태다. 플래그 변수(`found = False` … `if not found:`)를 쓰지 않아도 되어 코드가 짧아진다.

```python
items = [10, 20, 30]
target = 25
for x in items:
    if x == target:
        print("found", x)
        break
else:
    print("not found")
```

실행 결과: `not found`

### while-else — "조건이 False가 되어 나왔을 때"

`while 조건:`에서 **조건이 거짓이 되어** 루프를 빠져나오면, 그때 `else` 블록이 실행된다. "조건이 참인 동안" 돌다가 "조건이 거짓이 되었다"는 것은, `while`이 **정상적으로** 끝났다는 의미다. 반대로 **`break`**로 나갔다면 "조건과 무관하게 중간에 탈출"한 것이므로 `else`는 실행되지 않는다. 따라서 `while`-else도 "break로 나가지 않고 조건이 False가 되어 종료했을 때만" 실행된다는 점에서 `for`-else와 동일한 규칙이다.

```python
n = 0
while n < 3:
    n += 1
    if n == 2:
        break
else:
    print("loop ended normally")
print("n =", n)
```

실행 결과: `n = 2` (break로 나갔으므로 else 미실행)

### 왜 헷갈리는가

다른 언어에서는 `else`가 보통 "if가 거짓일 때"를 의미한다. 루프의 `else`는 "**break 안 했을 때**"를 의미하므로, 직관과 다르게 느껴질 수 있다. "루프가 **완료**되었을 때" 또는 "**break되지 않고** 끝났을 때"로 기억하면 된다. 도움말과 오래된 코드에서 자주 등장하므로, 의미를 한 번 정리해 두면 읽기와 유지보수에 도움이 된다.

## 실습 예제

### [실습] lab46

이 실습에서는 **루프-else가 실행되는 조건과 "검색 실패" 패턴**을 복습한다.

#### ① for-else — break 없이 끝났을 때만

`script01.py`

```python
for x in [1, 2, 3]:
    if x == 10:
        break
else:
    print("else: no break")
print("done")
```

실행 결과: `else: no break` / `done`

#### ② for-else — break로 나가면 else 미실행

`script02.py`

```python
for x in [1, 2, 3]:
    if x == 2:
        break
else:
    print("else ran")
print("done")
```

실행 결과: `done` (else 블록은 실행되지 않음)

#### ③ 검색 실패 시 else에서 처리

`script03.py`

```python
def find(items, target):
    for x in items:
        if x == target:
            return x
    else:
        return None

print(find([1, 2, 3], 2))
print(find([1, 2, 3], 5))
```

실행 결과: `2` / `None`

(실제로는 `return`으로 나가도 break와 마찬가지로 루프를 벗어나므로, 위 함수에서는 `else`가 "못 찾고 끝났을 때"만 실행된다. `return x`는 break처럼 루프를 끝내고, `else`의 `return None`은 break 없이 루프가 끝났을 때만 실행된다.)

#### ④ while-else

`script04.py`

```python
n = 0
while n < 2:
    n += 1
else:
    print("while-else ran")
print("n =", n)
```

실행 결과: `while-else ran` / `n = 2`

#### ⑤ 실습 포인트

- **for-else**: 이터러블이 **다 소진**되어 루프가 끝났을 때만 실행. **break**로 나가면 실행 안 함.
- **while-else**: **조건이 False**가 되어 루프가 끝났을 때만 실행. **break**로 나가면 실행 안 함.
- **검색 실패**: "찾으면 break, 못 찾고 끝나면 else" 패턴으로 not found 처리.

## 주의사항 (Pitfalls)

- **return과 else**: 루프 안에서 **`return`**으로 함수를 빠져나가면, 그 역시 "break처럼" 루프를 종료하는 것이므로 해당 루프의 `else`는 실행되지 않는다. "break 없이 루프가 끝났을 때"만 else가 실행된다는 점은 동일하다.
- **중첩 루프**: 안쪽 루프의 `break`는 안쪽 루프만 빠져나가므로, 안쪽 루프의 `else`는 "안쪽에서 break 안 했을 때"만 실행된다. 바깥 루프의 else와는 별개다.
- **가독성**: 팀에서 루프-else를 잘 모르면 "break 안 했을 때"라고 주석을 달아 두는 것이 좋다.

---

## Deep Insight

루프-else는 "루프가 **완료**되었는가, 아니면 **중간에 break**했는가"를 한 번에 표현한다. 플래그(`found = False` … `if not found:`)를 쓰지 않아도 되어, "찾기 실패" 같은 의도가 명확해진다. Python만의 문법이라 처음 보면 낯설 수 있지만, 한 번 규칙을 익혀 두면 레거시 코드와 공식 문서를 읽을 때 빠르게 해석할 수 있다.

---

## 요약

- **for-else** / **while-else**: 루프가 **break 없이** 정상 종료되었을 때만 `else` 실행.
- **for**: 이터러블 소진 시 else. **while**: 조건 False가 되어 나왔을 때 else.
- **검색 실패 패턴**: 찾으면 break, 못 찾고 끝나면 else에서 not found 처리.
