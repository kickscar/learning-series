# Research: 03.02 — VM 생성 및 기본 구성

> 조사일: 2026-04-06
> 기준: Google Cloud Console (CLAUDE.md > 플랫폼 상수 참조)

## 콘솔 경로

### lab05: VM 생성

1. Google Cloud Console 홈
2. 탐색 메뉴 > Compute Engine > **VM 인스턴스**
   (또는 검색창 "Compute Engine" 입력)
3. 인스턴스 만들기 클릭
4. 구성 탭:
   - 이름 입력
   - 리전 / 영역 선택
   - 머신 구성: 시리즈(E2) > 머신 유형(e2-medium)
   - 부팅 디스크: 변경 > OS 이미지 선택, 디스크 유형, 크기 설정
   - ID 및 API 액세스: 기본 서비스 계정 유지
   - 방화벽: HTTP/HTTPS 트래픽 허용 체크박스
   - 고급 옵션 > 네트워킹 탭: 네트워크 태그 입력, 외부 IP 설정

5. 만들기 클릭

### lab05: Firewall Rule 생성

1. Google Cloud Console 홈
2. 탐색 메뉴 > VPC 네트워크 > **방화벽**
   (또는 검색창 "방화벽 규칙")
3. 방화벽 규칙 만들기 클릭
4. 구성:
   - 이름
   - 네트워크: default
   - 트래픽 방향: 인그레스
   - 일치 시 작업: 허용
   - 대상: 지정된 대상 태그
   - 대상 태그: allow-ssh / allow-http
   - 소스 IPv4 범위: 0.0.0.0/0
   - 프로토콜 및 포트: TCP 포트 지정

5. 만들기 클릭

### VM 상태 확인

1. Google Cloud Console > Compute Engine > **VM 인스턴스**
2. VM 이름 클릭 → VM 세부정보 페이지
3. 로그 탭 또는 직렬 포트 출력 버튼 클릭

## 주요 설정 필드

- Name: VM 인스턴스 이름 (소문자, 숫자, 하이픈)
- Region: GCP 리전 (예: `us-central1`)
- Zone: 가용 영역 (예: `us-central1-a`)
- Machine Series: E2, N2, C2 등
- Machine Type: e2-micro, e2-medium, e2-standard-2 등
- Boot Disk Image: Ubuntu 22.04 LTS, Debian 11 등
- Boot Disk Type: Standard, Balanced, SSD
- Boot Disk Size: GB 단위 (기본 10GB)
- Network Tags: 콤마 구분 없이 Enter로 추가 (예: `allow-ssh`, `allow-http`)
- External IP: 없음 / 임시 / 고정 주소 예약

## 조사 메모

- VM 생성 UI는 2023년 이후 "새 VM 인스턴스 만들기" 레이아웃으로 변경됨
  - 이전: 단일 페이지 스크롤 / 현재: 섹션별 확장형 (구성, 스토리지, 네트워킹, 보안 탭)
  - 네트워크 태그는 "고급 옵션 > 네트워킹" 탭에 위치
- External IP 기본값은 Ephemeral(임시) — VM 중지/시작 시 IP가 변경됨
- Firewall Rule은 VM이 아닌 VPC에 속함 → VPC 네트워크 > 방화벽 메뉴에서 생성
- allow-ssh (TCP 22), allow-http (TCP 8080) 두 개의 Firewall Rule 별도 생성
- 직렬 포트 출력(Serial Port Output)은 VM 세부정보 > 직렬 포트 탭 또는 로그 탭에서 확인
