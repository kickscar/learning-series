# 4.6 bool 값 규칙

## 개요

`if x:`처럼 조건 자리에 값을 하나만 두면, Python은 그 값을 **참처럼 쓰일 수 있는지(Truthy)** 아니면 **거짓처럼 쓰일 수 있는지(Falsy)** 로 판단한다. 이때 내부적으로 `bool(x)`가 호출되며, 객체에 따라 `__bool__` 또는 `__len__`이 사용된다. `if x`의 실제 조건 평가 과정과 `__bool__`·`__len__`의 관계를 이해하는 것이 이 단원의 목표다.

---

## 핵심 개념

### 조건과 bool(x)

`if x:`나 `while x:`처럼 조건 위치에 식을 하나만 두면, Python은 그 식의 결과를 **참/거짓**으로 해석한다. 이때 `bool(해당_객체)`가 호출된다. `bool(x)`가 `True`를 반환하면 조건이 참, `False`를 반환하면 거짓이 되어 분기나 루프가 그에 따라 동작한다. 따라서 "어떤 값이 조건에서 어떻게 평가되는가?"는 **`bool()`에 넣었을 때 무엇이 나오는가**와 같다.

```python
print(bool(1), bool(0))
print(bool("hello"), bool(""))
print(bool([1, 2]), bool([]))
```

실행 결과: `True False` / `True False` / `True False`

### __bool__과 __len__

객체가 **참/거짓**으로 평가될 때, Python은 먼저 그 객체의 `__bool__` 메서드가 있는지 본다. `__bool__`이 있으면 그 반환값을 쓴다. `__bool__`이 없으면 **`__len__`**을 사용한다. `__len__`이 0을 반환하면 거짓, 0이 아니면 참으로 본다. 따라서 "길이"가 있는 컨테이너형은 **길이가 0이면 Falsy**, 0이 아니면 Truthy가 된다. 사용자 정의 클래스에서 `def __bool__(self): return self.data != []`처럼 `__bool__`을 정의하면, `if my_obj:`의 결과를 직접 제어할 수 있다.

```python
class HasLen:
    def __init__(self, n):
        self.n = n
    def __len__(self):
        return self.n

print(bool(HasLen(0)), bool(HasLen(3)))
```

실행 결과: `False True`

### Falsy인 값들

다음 값들은 `bool(x)`가 `False`를 반환하므로 **Falsy**다. 조건문에서 "비어 있다"·"없다"를 짧게 쓸 때 자주 쓰인다.

- **None**
- 숫자 **0**, **0.0** (정수·실수 0)
- 빈 시퀀스·컨테이너: **`""`**, **`[]`**, **`()`**, **`{}`**, **`set()`**
- `False` 자체

이 외에는 대부분 Truthy로 취급된다. 예를 들어 `" "`(공백 한 글자), `[0]`, `{"": 0}`처럼 "비어 보이지만 실제로는 길이가 있는" 값은 Truthy다.

```python
falsy = [None, 0, 0.0, "", [], (), {}, set()]
for x in falsy:
    print(bool(x), end=" ")
# False False False False False False False False
```

### if not items vs if len(items) == 0

"컨테이너가 비어 있는가?"를 검사할 때 **`if not items:`**가 **`if len(items) == 0:`**보다 선호된다. 의미가 같고, `not items`가 더 짧으며, `__len__`이 없는 이터러블에 대해서도 `__bool__`/`__len__` 규칙으로 동작할 수 있기 때문이다. 다만 "길이 0"과 "Falsy"가 항상 같은 것은 아니다. 사용자 타입이 `__bool__`을 정의하면 `__len__`과 무관하게 참/거짓이 정해진다. 일반적인 list·str·dict 등에서는 `if not items:`로 "비어 있음"을 검사하면 된다.

## 실습 예제

### [실습] lab41

이 실습에서는 **Truthy/Falsy와 `bool()`·`__len__`**의 개념을 복습한다.

#### ① bool() — 숫자·문자열·컨테이너

`script01.py`

```python
print(bool(0), bool(1), bool(-1))
print(bool(""), bool(" "))
print(bool([]), bool([0]))
```

실행 결과: `False True True` / `False True` / `False True`

#### ② Falsy 목록 — 조건에서의 사용

`script02.py`

```python
for x in [None, 0, "", [], {}]:
    if x:
        print("Truthy:", x)
    else:
        print("Falsy:", repr(x))
```

실행 결과: 각각 Falsy로 출력 (None, 0, '', [], {})

#### ③ if not x — "비어 있음" 검사

`script03.py`

```python
items = []
if not items:
    print("empty")
items = [1]
if not items:
    print("empty")
else:
    print("has items")
```

실행 결과: `empty` / `has items`

#### ④ 실습 포인트

- `if x:`에서 실제로는 **`bool(x)`**가 호출된다.
- **Falsy**: `None`, `0`, `0.0`, `""`, `[]`, `()`, `{}`, `set()`, `False`.
- **Truthy**: 위가 아닌 값(공백 문자열 `" "`, `[0]` 등 포함).
- `__bool__`이 있으면 그 결과 사용, 없으면 **`__len__`이 0이면 False**.
- "비어 있음" 검사는 **`if not items:`** 권장.

---

## 주의사항 (Pitfalls)

- **0과 "빈 값"**: `0`은 Falsy이므로 `if count:`는 "count가 0이면 거짓"이다. "0이 유효한 값"인 경우에는 `if count is not None:` 등으로 구분해야 할 수 있다.
- **빈 문자열 vs 공백**: `""`는 Falsy, `" "`는 Truthy. "공백만 있는 문자열"을 비어 있다고 보려면 `if not s.strip()` 등을 쓴다.
- **`if len(x) == 0`**: `__len__`이 없는 객체는 `len(x)`에서 `TypeError`가 날 수 있다. 일반적인 "비어 있음" 검사는 `if not x:`가 안전하다.

---

## Deep Insight

Truthy/Falsy 규칙은 "조건을 짧게 쓸 수 있게" 하기 위한 것이다. `if items:`는 "항목이 있으면", `if not items:`는 "비어 있으면"으로 읽히고, `__bool__`·`__len__`에 의해 일관되게 동작한다. 사용자 타입에 `__bool__`을 두면 "이 객체를 조건에 쓸 때 무엇을 기준으로 할지"를 명시할 수 있어, 도메인에 맞는 읽기 좋은 조건문을 만들 수 있다.

---

## 요약

- `if x:`에서는 **`bool(x)`**가 호출된다. `__bool__`이 있으면 그 결과, 없으면 **`__len__`이 0이면 False**.
- **Falsy**: `None`, `0`, `0.0`, `""`, `[]`, `()`, `{}`, `set()`, `False`. 나머지는 Truthy.
- "비어 있음" 검사는 **`if not items:`**를 쓰면 된다.
