# Skill: status/report

언제든 호출 가능한 전체 진행 현황 보고 스킬.

---

## 목적

시리즈 전체 진행률, 섹션별 draft 완성 상태, 다음 작업 대상을 한눈에 파악한다.

---

## 실행 순서

### Step 1 — 구조 파악

읽기 대상:
1. `.claude/plan/series-hierarchy.md` — 전체 챕터/섹션 목록 + Lab 번호 맵 + Gallery 맵

파악 내용:
- 챕터 수, 전체 섹션 수 (Gallery 포함), Lab 수, Gallery 수
- 섹션별 기대 draft 파일명 (`{ch_no}.{sec_no}.claude.notion.md`)

---

### Step 2 — draft 파일 스캔

스캔 대상:
- `.claude/draft/` 디렉토리 내 파일 목록

비교:
- series-hierarchy.md 기준 전체 섹션 수 대비 실제 존재하는 draft 파일 수
- 파일명으로 어느 챕터/섹션인지 매핑

---

### Step 3 — 현황 보고 출력

아래 형식으로 대화 출력:

```
## 현황 보고 — {시리즈명}

### 전체 진행률
- 전체 {N}개 섹션 중 {M}개 draft 완료 ({P}%)
- Lab {총수}개 | Gallery {총수}개

### 챕터별 진행률

| Chapter | 섹션 수 | draft 완료 | 진행률 |
|---------|--------|-----------|--------|
| 01 ... | N | M | P% |
| ...    |   |   |    |

### 섹션별 draft 상태

| Ch | Sec | 제목 | Lab | 이론 | 실습 | 비고 |
|----|-----|------|-----|------|------|------|
| 01 | 01  | ...  |  —  |  ✓  |  —  |      |
| ...                                              |

- 이론: draft 파일 존재 여부 (✓ / —)
- 실습: draft 파일 내 `# [실습]` 섹션 존재 여부 (✓ / — / N/A)
- N/A: 이론 전용 섹션 (Lab 없음)

### 다음 작업 대상
- {현재 미완성 섹션 중 첫 번째 — 챕터 순서 기준}
```

---

## 출력 규칙

- 대화로만 출력. 파일 생성/수정 없음.
- 이론/실습 완성도 구분: draft 파일 존재 = 이론 ✓, `# [실습]` 포함 = 실습 ✓
- 보고 후 "다음 작업 대상"으로 바로 넘어가도 되는지 확인 질문으로 마무리
