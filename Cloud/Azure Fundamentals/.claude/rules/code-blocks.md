# Rule: code-blocks

코드 블록 작성 규칙.

---

## 언어 지정 필수

모든 코드 블록에 언어를 명시한다.

| 언어 | 사용 상황 |
|------|----------|
| `bash` | 터미널 명령어, Shell 스크립트 |
| `yaml` | 설정 파일, 인프라 템플릿 |
| `json` | API 응답, 설정 값 |
| `mermaid` | 다이어그램 |
| `text` | **기본값** — 언어가 없는 모든 코드 블록, 출력 결과, 로그, ASCII 다이어그램 |

---

## 명령어와 출력 예시

명령어와 출력을 하나의 블록에 포함. `# 출력 예` 주석으로 구분.

> CLI 도구명은 `CLAUDE.md > 플랫폼 상수` 참조.

```bash
$ {cli} {명령어} --{옵션} {값}

# 출력 예
[
  {
    "name": "{리소스명}",
    ...
  }
]
```

- 프롬프트 `$` 표시
- 생략 부분: `...(생략)...`
- 강조할 라인: 주석 없이 별도 설명 텍스트로

---

## 인라인 코드

백틱(`` ` ``)으로 감싼다.

- 리소스명: `rg-gallery`
- 경로/파일: `/etc/nginx/nginx.conf`
- 명령어 단어: `{cli} vm start`
- 포트/값: `8080`, `Korea Central`

---

## 설정값 블록 (코드 블록 아님)

실습의 콘솔 설정값은 코드 블록이 아니라 목록으로 표기.

```text
**설정:**

- Resource group: **`rg-lab03`**
- Location: **`Korea Central`**
- VM name: **`vm-lab03`**
```
