# Research: 01.01 — IaC와 Terraform 포지셔닝

> 조사일: 2026-04-15
> 기준: Terraform 1.14.x, hashicorp/aws provider >= 6.0

## 검증 결과

| 항목 | 상태 | 비고 |
|------|------|------|
| IaC 개념 (선언형 vs 절차형) | ✓ 정확 | 설명 구조와 비교표 정확 |
| Terraform을 "오픈소스 IaC 도구"로 표현 | ✗ 수정필요 | 2023년 8월 BSL 1.1 전환 이후 오픈소스가 아님. "source-available" 또는 단순히 "IaC 도구"로 수정 필요 |
| BSL 라이선스 변경 설명 | ⚠ 주의 | draft에 BSL 변경 사실은 언급하나, 현재 상태가 불명확. BSL(BUSL 1.1)은 소스 공개이나 OSI 인증 오픈소스가 아님. 내부 인프라 관리 용도는 제한 없이 사용 가능. 상용 SaaS로 제공 시 제한. 4년 경과 소스는 자동 MPL 2.0 전환 |
| OpenTofu를 "CNCF 프로젝트"로 표현 | ✓ 정확 | 2025년 4월 CNCF Sandbox 수준으로 수락. Linux Foundation 거버넌스 하에 운영. 최신 버전 v1.11.6 (2026-04-08) |
| `required_providers` 문법: `source = "hashicorp/aws"` | ✓ 정확 | v0.13 이후 표준 문법, 1.14.x에서 동일 |
| `version = "~> 5.0"` 표기 | ⚠ 주의 | 문법 자체는 정확하나, draft 본문에서 `~> 5.0`으로 예시. 시리즈 기준이 aws provider >= 6.0이므로 `~> 6.0`으로 통일 권장. 단, 01.01은 소개 섹션이므로 일반적 예시로 유지해도 무방 — 01.02 lab에서 `~> 6.0` 사용 |
| Provider 생태계 "수백 개" 표현 | ⚠ 주의 | 2026년 기준 Terraform Registry에 4,800+ Provider 등록. "수백 개"는 과소 — "수천 개" 또는 "4,000개 이상"이 정확 |
| CloudFormation vs Terraform 비교표 | ✓ 정확 | 핵심 차이점 정확. "비용" 항에서 "오픈소스 무료" 표현은 BSL 수정과 연동 필요 |
| 비교표 "오픈소스 무료 (Terraform Cloud는 유료)" | ⚠ 주의 | BSL 전환 반영 필요. "CLI 무료 (Terraform Cloud/Enterprise는 유료)" 정도로 수정 |
| 시리즈 기준 버전 "Terraform 1.10.x" 표기 | ✗ 수정필요 | draft 133행에 "이 시리즈는 Terraform 1.10.x 기준"이라 기재. CLAUDE.md와 다른 draft는 모두 1.14.x 기준. 오타 수정 필요 |
| Ansible 예시 코드 | ✓ 정확 | 절차형 설명 목적의 예시로 적절 |

## 주의사항

- **"오픈소스" 표현 일괄 점검**: draft 96행 "HashiCorp가 개발한 오픈소스 IaC 도구다"를 수정해야 한다. BSL은 소스를 공개하지만 OSI 인증 오픈소스 라이선스가 아니다. "HashiCorp가 개발한 IaC 도구다" 또는 라이선스 섹션에서 별도 설명하는 방식이 정확하다.
- **버전 오타**: draft 135행 "Terraform 1.10.x" → "Terraform 1.14.x"로 수정 필요.
- **Provider 수 표현**: "수백 개"를 "수천 개"로 갱신. Registry에 4,800+ Provider 등록(2026년 기준).
- **비교표 비용 항목**: "오픈소스 무료" → "CLI 무료" 또는 "소스 공개/무료(상업적 경쟁 서비스 제공 시 제한)"로 수정 권장.
