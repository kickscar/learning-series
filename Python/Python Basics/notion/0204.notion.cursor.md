# 2.4 변수와 타입 — 값에 따른 타입

## 개요

변수 **자체**에는 "타입"이 고정되어 있지 않다. **값**에 타입이 있고, 변수는 어떤 값이든 붙일 수 있다. "이 변수의 타입"이 아니라 **"지금 이 이름에 붙어 있는 값의 타입"**이라는 관점을 익히는 것이 이 단원의 목표다.

---

## 핵심 개념

- **타입은 값에 있다**: 이름은 그저 값과 연결될 뿐이다. 같은 이름에 정수, 문자열, 리스트 등 서로 다른 종류의 값을 차례로 붙여도 된다.
- **`type(값)`**: 현재 그 값의 타입을 알려 준다. `type(a)`는 **지금** a에 붙어 있는 값의 타입을 반환한다.
- **타입에 따른 연산**: 값의 종류(타입)에 따라 쓸 수 있는 연산이 다르다. 구체적인 타입(int, str, list 등)과 연산은 다음 장에서 다룬다.

---

## 코드 예제

```python
# 같은 이름에 서로 다른 타입의 값을 붙여도 된다.
a = 1
print(type(a))   # <class 'int'>

a = "hello"
print(type(a))   # <class 'str'>

a = [1, 2, 3]
print(type(a))   # <class 'list'>
```

```python
# type(x)는 "변수 x의 타입"이 아니라 "지금 x에 붙어 있는 값의 타입"이다.
x = 42
print(type(x))   # <class 'int'>
x = 3.14
print(type(x))   # <class 'float'>
```

---

## 주의사항 (Pitfalls)

- **"변수의 타입"이라는 말**: Python에서는 "변수 타입"보다 **"값의 타입"**이 맞는 표현이다. 변수는 타입이 바뀌는 것이 아니라, 다른 타입의 값을 붙일 수 있을 뿐이다.
- **타입에 맞지 않는 연산**: 값의 타입에 따라 쓸 수 있는 연산이 정해져 있다. 예를 들어 문자열과 정수를 `+`로 더하려 하면 `TypeError`가 날 수 있다. 타입과 연산은 자료형 단원에서 다룬다.

---

## Deep Insight

이름은 타입을 갖지 않고, 값만 타입을 갖는다. 그래서 `a = 1` 다음에 `a = "hello"`가 가능하다. "a의 타입이 int에서 str로 바뀌었다"기보다 "a가 처음에는 int 값과, 나중에는 str 값과 연결되었다"로 보는 것이 Python의 모델에 맞다. 이 관점은 나중에 타입 힌트나 다형성을 다룰 때도 그대로 이어진다.

---

## 요약

- 변수 자체에는 타입이 고정되지 않는다. 값에 타입이 있다.
- 같은 이름에 서로 다른 타입의 값을 붙일 수 있다. `type(a)`는 지금 a에 붙어 있는 값의 타입을 반환한다.
- "변수의 타입"보다 "값의 타입"으로 생각하는 것이 맞다.
