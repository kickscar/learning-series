# 4.4 match-case 문과 패턴 매칭

## 개요

**`match`** / **`case`**는 Python 3.10에서 도입된 **구조 패턴 매칭** 문법이다. 값 하나를 여러 **패턴**과 순서대로 비교하고, 처음으로 맞는 `case` 블록만 실행한다. 리터럴(`1`, `"ok"`), 타입(`case int(x):`), 시퀀스(`case [a, b]:`), 매핑(`case {"k": v}:`) 등으로 분기할 수 있고, **`|`**(OR 패턴), **`as`**(이름 붙이기), **가드**(`case x if x > 0:`)로 조건을 더 세밀히 줄 수 있다. `if`-`elif` 대체와 타입·구조 분해를 `match`로 표현할 수 있게 하는 것이 이 단원의 목표다.

---

## 핵심 개념

### match의 기본 형태

`match 식:` 다음에 **`case 패턴:`** 블록들을 나열한다. 식을 평가한 값이 **위에서부터** 각 `case`의 패턴과 맞는지 검사하고, **처음으로 매칭되는** 블록만 실행한 뒤 `match`를 빠져나간다. 따라서 더 구체적인 패턴을 위에, 넓은 패턴이나 **`case _:`**(와일드카드)를 아래에 두는 것이 보통이다. `_`는 "그 외 모든 값"에 매칭되므로 fallback으로 쓴다. 어떤 `case`에도 맞지 않으면 아무 블록도 실행되지 않으며, **`MatchError`**는 발생하지 않는다(명시적으로 처리하지 않으면 그냥 끝난다).

```python
def describe(x):
    match x:
        case 1:
            return "one"
        case 2 | 3:
            return "two or three"
        case _:
            return "other"

print(describe(1), describe(2), describe(10))
```

실행 결과: `one two or three other`

### 리터럴과 타입 패턴

**리터럴**은 값이 그 숫자·문자열과 **같을 때** 매칭된다. `case 1:`, `case "ok":`처럼 쓴다. **타입 패턴**은 값이 해당 타입일 때 매칭하고, 그 값을 이름에 묶을 수 있다. `case int(n):`은 "정수면 매칭하고 그 값을 `n`에 넣는다"는 의미다. `case str(s):`는 문자열일 때 `s`에 바인딩. 타입이 맞지 않으면 그 `case`는 건너뛰고 다음을 본다. 따라서 `case int(x):` 다음에 `case float(x):`를 두면 정수는 위에서, 실수는 아래에서 처리할 수 있다.

```python
match 42:
    case int(n):
        print("int", n)
match 3.14:
    case float(x):
        print("float", x)
```

실행 결과: `int 42` / `float 3.14`

### 시퀀스·매핑 패턴

**시퀀스 패턴**으로 리스트·튜플 구조를 분해할 수 있다. `case [a, b]:`는 길이 2인 시퀀스에 매칭하고 첫 번째·두 번째 요소를 `a`, `b`에 넣는다. `case [h, *t]:`는 "첫 요소는 `h`, 나머지는 `t`(리스트)"로 묶는다. **매핑 패턴**은 `case {"key": v}:`처럼 키가 있고 그 값을 `v`에 바인딩한다. 키는 리터럴이어야 하며, 추가 키가 있어도 매칭된다. 구조가 맞지 않으면(길이 다름, 키 없음 등) 그 `case`는 건너뛴다.

```python
match [1, 2, 3]:
    case [a, *rest]:
        print(a, rest)
match {"x": 10, "y": 20}:
    case {"x": vx}:
        print("x", vx)
```

실행 결과: `1 [2, 3]` / `x 10`

### as, | (OR), 가드

**`as`**로 매칭한 값을 다른 이름에 붙일 수 있다. `case int(x) as n:`처럼 쓰면 타입 패턴으로 매칭한 뒤 그 값을 `n`이라는 이름으로도 쓴 수 있다(문법은 `case ... as n:` 형태). **`|`**는 **OR 패턴**이다. `case 1 | 2 | 3:`은 값이 1, 2, 3 중 하나일 때 매칭된다. **가드**는 `case` 뒤에 **`if 조건`**을 붙여, 패턴이 맞아도 조건이 거짓이면 그 `case`를 건너뛴다. `case int(x) if x > 0:`은 "정수이고 `x > 0`일 때만" 매칭된다. 가드에서 바인딩된 이름(예: `x`)을 쓸 수 있다.

```python
match 2:
    case 1 | 2 | 3:
        print("small")
match 5:
    case int(x) if x > 0:
        print("positive", x)
```

실행 결과: `small` / `positive 5`

## 실습 예제

### [실습] lab45

이 실습에서는 **match/case 리터럴·타입·시퀀스·와일드카드**의 개념을 복습한다.

#### ① 리터럴·OR 패턴

`script01.py`

```python
def code(n):
    match n:
        case 1:
            return "one"
        case 2 | 3:
            return "two or three"
        case _:
            return "other"

print(code(1), code(2), code(5))
```

실행 결과: `one two or three other`

#### ② 타입 패턴

`script02.py`

```python
def describe(v):
    match v:
        case int(x):
            return f"int {x}"
        case str(s):
            return f"str {s!r}"
        case _:
            return "other"

print(describe(42), describe("hi"))
```

실행 결과: `int 42` / `str 'hi'`

#### ③ 시퀀스 패턴 [a, *rest]

`script03.py`

```python
match [10, 20, 30]:
    case [first, *rest]:
        print(first, rest)
```

실행 결과: `10 [20, 30]`

#### ④ 가드 if

`script04.py`

```python
match 3:
    case int(x) if x > 5:
        print("big")
    case int(x) if x > 0:
        print("small positive", x)
```

실행 결과: `small positive 3`

#### ⑤ 실습 포인트

- **match 식:** 위에서부터 **case 패턴:**과 비교, **처음 매칭**만 실행.
- **리터럴**: `case 1:`, `case "ok":`. **타입**: `case int(x):`로 매칭·바인딩.
- **시퀀스**: `case [a, b]:`, `case [h, *t]:`. **매핑**: `case {"k": v}:`.
- **`case _:`**: 와일드카드(fallback). **`|`**: OR 패턴. **가드**: `case ... if 조건:`.

---

## 주의사항 (Pitfalls)

- **버전**: `match`/`case`는 **Python 3.10+**에서만 사용 가능하다. 이전 버전에서는 문법 오류가 난다.
- **순서**: 더 구체적인 패턴을 위에 두어야 한다. `case _:`를 맨 위에 두면 항상 그만 매칭되어 아래 `case`는 도달하지 않는다.
- **바인딩 이름**: 같은 `match` 안에서 서로 다른 `case`에 같은 이름을 써도 괜찮다. 각 블록은 독립된 스코프에 가깝게 동작한다. 가드에서는 해당 `case`에서 바인딩된 이름만 사용할 수 있다.

---

## Deep Insight

`match`는 "값의 모양"으로 분기하는 것이다. `if x == 1`이 아니라 `case 1:`로 쓰고, 타입과 구조를 한 번에 검사·분해할 수 있어서, 복잡한 `if`-`elif` 체인을 읽기 쉬운 형태로 바꿀 수 있다. 리스트·딕셔너리 구조를 `case [a, *t]:`, `case {"k": v}:`로 풀어 쓰면 "이런 형태일 때 이렇게 처리"가 명확해진다. 3.10+ 코드베이스에서는 점점 보편화되는 문법이므로, 조건 분기와 구조 분해가 겹치는 경우 `match`를 우선 고려하면 좋다.

---

## 요약

- **match/case** (3.10+): 식을 패턴과 위에서부터 비교, **처음 매칭되는 case**만 실행.
- **리터럴** `case 1:`, **타입** `case int(x):`, **시퀀스** `case [h, *t]:`, **매핑** `case {"k": v}:`, **와일드카드** `case _:`.
- **`|`** OR 패턴, **가드** `case ... if 조건:`. 구체적인 패턴을 위에 두고 `_`는 fallback으로.
