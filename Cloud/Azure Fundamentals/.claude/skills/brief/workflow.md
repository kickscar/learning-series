# Skill: brief/workflow

## 목적

세션 시작 시 시리즈 작업 상태를 파악하고 브리핑한다.
범용 설계 — CLAUDE.md + series-hierarchy.md에서 시리즈 맥락을 읽어 동작한다.

---

## 실행 순서

### Step 1 — 시리즈 상태 파악

읽기 대상 (순서대로):
1. `CLAUDE.md` — 시리즈 역할, 범위, 언어 규칙
2. `.claude/plan/series-hierarchy.md` — 전체 챕터/섹션 구조 + 작업 상태 테이블

파악 내용:
- 총 챕터 수 / 총 섹션 수 / 총 Lab 수
- 각 챕터 상태 (`계획확정` / `작업중` / `완료`)

---

### Step 2 — 이전 세션 기록 읽기

읽기 대상:
- `.claude/memory/context_bridge.md`

파악 내용:
- 마지막 세션 날짜
- 완료된 작업
- 미완료 인계 항목
- 주요 결정사항

파일이 없거나 비어 있으면: "이전 세션 기록 없음"으로 처리하고 계속 진행.

---

### Step 3 — 실제 작업물 스캔

스캔 대상:
- `.claude/draft/` 디렉토리 내 파일 목록

비교:
- `series-hierarchy.md` 기준 예상 draft 파일 수
- 실제 존재하는 draft 파일 수 및 파일명 목록

---

### Step 4 — 브리핑 출력

아래 형식으로 대화 출력:

```
## 세션 브리핑 — {시리즈명}

### 마지막 세션 주요 내용
- {context_bridge.md 최근 항목 요약}

### 미완료 / 다음 작업 추천
- {미완료 항목 또는 다음으로 작성할 섹션}

### 미해결 이슈
- {있을 경우 표시, 없으면 생략}
```

---

### Step 5 — 현황 보고

`status/report` 스킬 실행 → 전체 진행률 + 섹션별 draft 상태 출력

---

## 출력 규칙

- 대화로만 출력. 파일 생성/수정 없음.
- Step 4 브리핑 → Step 5 현황 보고 순서로 출력.
- 현황 보고 후 다음 행동을 제안하며 마무리:
  `"{다음 작업} 이어서 진행할까요?"`
