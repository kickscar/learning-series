## 주의사항

### 값 전달 우선순위 — draft 수정 필요

draft의 우선순위 표에 오류가 있다. 공식 문서 기준 정확한 순서:

| 우선순위 | 방법 |
|---------|------|
| 1 (최고) | `-var` 및 `-var-file` 플래그 (명령행에 나열된 순서대로, 나중 것이 우선) |
| 2 | `*.auto.tfvars` / `*.auto.tfvars.json` 파일 (사전순, 나중 파일이 우선) |
| 3 | `terraform.tfvars` / `terraform.tfvars.json` 파일 |
| 4 | 환경 변수 (`TF_VAR_이름`) |
| 5 (최저) | `default` 값 |

**draft와의 차이점:**

1. `-var`와 `-var-file`은 **같은 우선순위**다. 둘 다 "command-line options"으로 묶이며, 명령행에 나열된 순서에서 나중 것이 우선한다. draft에서 `-var`(1순위)와 `-var-file`(2순위)를 분리한 것은 부정확하다.
2. `*.auto.tfvars`가 `terraform.tfvars`보다 높은 우선순위다. draft에서 `*.auto.tfvars`(3순위)가 `terraform.tfvars`(4순위)보다 위로 나열한 것은 맞지만, 실제 공식 문서에서도 auto.tfvars가 terraform.tfvars보다 높으므로 순서 자체는 맞다.
3. 환경 변수(`TF_VAR_`)가 `terraform.tfvars`보다 **낮은** 우선순위다. draft에서 환경 변수(5순위)를 `terraform.tfvars`(4순위) 아래에 둔 것은 맞다.

**결론:** draft의 1~2순위를 합쳐서 "CLI 플래그(`-var`, `-var-file`) — 명령행 순서대로, 나중 것 우선"으로 수정해야 한다.

### validation에서 `can()` 함수

draft에서 `can()` 사용 예시는 없지만, 공식 문서에서 권장하는 패턴이다. 예: `condition = can(regex("^ami-", var.image_id))`. 필요시 lab03에 추가 가능하나, 현재 `contains()`, 비교연산자, `length()` 조합으로 충분하다.

### `terraform output -json` sensitive 값 주의

`-json` 또는 `-raw` 플래그 사용 시 sensitive output도 평문으로 출력된다. draft의 sensitive output 설명에서 이 점을 보충하면 좋다.

### Ephemeral Values (TF 1.10+)

Terraform 1.10부터 ephemeral values가 도입되었다. State/plan 파일에 저장되지 않는 값으로, sensitive output의 한계를 보완한다. 이 시리즈 범위에서는 다루지 않아도 되지만, 참고로 기록한다.
