# 01 Cloud와 AWS 시작하기 / 02 AWS Global Infrastructure

AWS의 Global Infrastructure(Region, Availability Zone, Edge Location)를 인프라 관점에서 정리한다. Region 선택이 성능/규정/서비스 가용성/비용을 동시에 결정한다는 기준을 세우고, 본 시리즈의 실습 기준 Region을 고정한다.

## What you will learn

- AWS 서비스 카테고리를 "아키텍처 블록" 관점으로 묶는 방법
- Region/AZ/Edge Location의 역할과 차이
- Global Infrastructure가 Multi-AZ, Subnet 배치, CDN으로 이어지는 연결 구조
- Region 선택 기준(Latency/Compliance/Service Availability/Cost)
- 본 시리즈의 실습 기준 Region(`ap-northeast-2 (Seoul)`)과 일관성 원칙
